# Context Rules

**ВАЖНО:** При поднятии контекста и исследовании кодовой базы игнорировать следующие папки:

```
docs/plans/     - Планы реализации (исторические)
docs/images/    - Скриншоты для документации
docs/reviews/   - Код ревью (исторические)
docs/ideas/     - Черновики идей
docs/works/     - Рабочие/аналитические документы
```

**Основные файлы документации для контекста (в порядке приоритета):**

1. **docs/QUICK_START.md** — **НАЧИНАТЬ ОТСЮДА** (2-3 минуты на полный контекст)
2. docs/INDEX.md — главный индекс документации
3. docs/project-overview.md — обзор проекта
4. docs/cheatsheet.md — быстрые команды и hotkeys
5. docs/rust-modules.md — Rust модули
6. docs/vue-components.md — Vue компоненты
7. docs/architecture.md — архитектура
8. docs/KEY_DECISIONS.md — архитектурные решения

**Приоритет при исследовании кода:**
1. Сначала прочитать **QUICK_START.md** для общего понимания
2. Исходный код (`src-tauri/src/`, `src/`)
3. Основная документация (выше)
4. Планы и скриншоты - только по явному запросу

**QUICK_START.md содержит:**
- Проект за 30 секунд
- Ключевые файлы и их назначение
- Как это работает (схемы)
- Горячие клавиши
- Где что искать при проблемах
- Основные модули и их связи
- Event system
- TTS providers (OpenAI, Silero, Local, Fish Audio)
- Key patterns
