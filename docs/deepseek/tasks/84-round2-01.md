# Task 84-round2-01: Доработки окна управления воспроизведением (после runtime-теста)

Окно управления (playback-control) теперь открывается корректно (план 84 round1 — белый квадрат
ушёл, UI рисуется с кнопками). Но runtime-тест выявил 3 проблемы. Чини каждую.

## Баг 1: Нет хоткея в панели настроек горячих клавиш
**Файлы:** `src/components/HotkeysPanel.vue`, `src/types/settings.ts`.
Бэкенд уже имеет `playback_control_window` в HotkeySettings (round1). Но:
- **Фронтовая панель** `HotkeysPanel.vue` рендерит хоткеи **поштучно вручную** (main_window,
  sound_panel — строки 252-342 шаблона). **Нет строки для playback-control.**
- `saveHotkey()` (строки 126-152) и `resetToDefault()` (154-169) — case только для main_window/
  sound_panel, **нет playback_control_window**.

### Что сделать
1. В `src/types/settings.ts` — `HotkeySettingsDto`: добавить поле `playback_control_window: HotkeyDto`
   (если его нет — проверь, бэкенд уже отдаёт его).
2. `HotkeysPanel.vue` шаблон: добавить блок «Управление воспроизведением» по образцу sound_panel
   (строки 298-342) — с иконкой (напр. `PlayCircle` или `MonitorPlay` из lucide-vue-next),
   `hotkeys.playback_control_window`, `startRecording('playback_control_window')`,
   `resetToDefault('playback_control_window')`.
3. `saveHotkey()`: добавить `else if (name === 'playback_control_window') { hotkeys.value.playback_control_window = hotkey }`.
4. `resetToDefault()`: то же — case для playback_control_window.
5. Добавить иконку в импорты (PlayCircle/MonitorPlay).

## Баг 2: Окно нельзя перетаскивать + нет кнопки закрытия
**Файл:** `src-playback/PlaybackControlApp.vue`.
Окно `decorations: false` → нет системной рамки. Сейчас `data-tauri-drag-region` есть на
`.window-header` (строка 75), но:
- Перетаскивание может не работать по всему header (проверь, что data-tauri-drag-region на
  корневом элементе header, не блокируется дочерними элементами status-badge).
- **Нет кнопки закрытия ×** — пользователь не может закрыть окно.

### Что сделать
1. Добавить кнопку закрытия в `.window-header` (рядом с title/status-badge):
   ```vue
   <button class="close-btn" @click="closeWindow" title="Закрыть">✕</button>
   ```
   ```ts
   import { getCurrentWindow } from '@tauri-apps/api/window'
   async function closeWindow() {
     await getCurrentWindow().hide()   // скрыть, не уничтожать (окно переиспользуется)
   }
   ```
   (hide, не close — окно создаётся из конфига один раз, переиспользуем через show/hide.)
2. Убедиться, что `data-tauri-drag-region` работает: повесить на весь `.window-header`, а
   status-badge и close-btn имеют `pointer-events`/не блокируют drag (кнопка по умолчанию
   перехватывает клик — это ок, остальная область header тянет окно).
3. CSS `.close-btn` — стиль по образцу других close-кнопок проекта, CSS-vars
   (`--color-text-muted`, hover `--color-danger`).

## Баг 3: Окно не реагирует на воспроизведение + история не обновилась
**Файлы:** `src-playback/PlaybackControlApp.vue`, `src-tauri/src/playback.rs` (диагностика).
Пользователь отправил текст на TTS → воспроизвелось, но окно управления **не показало реакцию**
(статус остался Idle, очередь пуста), и **история фраз не обновилась**.

### Диагностика
- Бэкенд эмитит события: `playback.rs:189` `app.emit("playback-started", ...)`, `playback.rs:293`
  `app.emit("queue-changed", ())`, и др. (paused/resumed/stopped/finished).
- `PlaybackControlApp.vue:53-60` подписан на них через `listen('playback-started', () => fetchState())`.
- `fetchState()` (22-28) → `invoke('get_playback_state')`.

**Гипотезы (проверить):**
1. **События не доходят до окна playback-control.** `app.emit()` в Tauri 2 рассылает всем окнам,
   но возможно webview playback-control не зарегистрирован как получатель, или событие эмитится
   ДО того, как окно смонтировалось (если окно было скрыто, WebView мог не инициализировать
   listener). Проверить: эмитить ли через `window.emit_to("playback-control", ...)` явно, или
   `app.emit` достаточно. Посмотри, как soundpanel получает события (он работает).
2. **`get_playback_state` возвращает устаревшее/пустое.** Проверь команду `get_playback_state`
   в `commands/` — она читает текущее состояние playback_manager. Если состояние не обновляется
   в менеджере — окно покажет Idle. Посмотри, обновляется ли playback_manager при speak_text.
3. **История фраз не обновилась** — отдельный симптом. `record_history` вызывается из `speak()`
   (InputPanel, план 83). Проверь, что `record_history` реально записывает (HistoryManager.
   record_phrase). Возможно пользователь смотрит на историю фраз в главном окне — она может
   не обновляться реактивно (нужен reload). Это отдельный баг от окна управления.

### Что сделать
1. **Диагностировать** (через логи/DevTools): добавить временный `console.log` в
   `PlaybackControlApp.vue` listeners — приходят ли события. Если нет →
2. Проверить `get_playback_state` команду — возвращает ли актуальное состояние.
3. Если события `app.emit` не доходят до скрытого/только-что-показанного окна — использовать
   `emit_to("playback-control", ...)` в playback.rs ИЛИ эмитить при show (refresh после show).
   Минимальный фикс: при `show_playback_window` — после show дёрнуть
   `window.emit("refresh-state", ())` или вызвать `fetchState` через emit, чтобы окно
   обновило состояние сразу при показе.
4. История фраз — проверить `record_history` → record_phrase. Если не пишется — отдельный баг.
