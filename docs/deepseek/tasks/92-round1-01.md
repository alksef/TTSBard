### 1. Новый файл `src-tauri/src/tabs.rs`
Скопируй структуру с `history.rs` (RwLock из parking_lot, spawn_save в потоке, read_or_default).

```rust
use parking_lot::RwLock;
use serde::{Deserialize, Serialize};
use std::fs;
use std::path::PathBuf;

const MAX_TABS: usize = 50;
const MAX_TAB_TEXT_LEN: usize = 100_000;

#[derive(Debug, Clone, Serialize, Deserialize, Default)]
pub struct EditorTab {
    #[serde(default)]
    pub id: String,
    #[serde(default)]
    pub title: String,
    #[serde(default)]
    pub text: String,
}

#[derive(Debug, Clone, Serialize, Deserialize, Default)]
pub struct TabsData {
    #[serde(default)]
    pub active_id: String,
    #[serde(default)]
    pub tabs: Vec<EditorTab>,
}

pub struct TabManager {
    path: PathBuf,
    data: RwLock<TabsData>,
}

fn spawn_save(path: PathBuf, data: TabsData) {
    std::thread::spawn(move || {
        if let Ok(content) = serde_json::to_string_pretty(&data) {
            let _ = fs::write(&path, content);
        }
    });
}

impl TabManager {
    pub fn new(path: PathBuf) -> Self {
        let _ = fs::create_dir_all(path.parent().unwrap_or(&path));
        let data = fs::read_to_string(&path)
            .ok()
            .and_then(|c| serde_json::from_str::<TabsData>(&c).ok())
            .unwrap_or_default();
        TabManager { path, data: RwLock::new(data) }
    }

    pub fn load_all(&self) -> TabsData {
        self.data.read().clone()
    }

    pub fn save_all(&self, mut data: TabsData) {
        // ВАЛИДАЦИЯ (урок SECURITY из review-018):
        // отбросить вкладки сверх MAX_TABS; обрезать text до MAX_TAB_TEXT_LEN.
        if data.tabs.len() > MAX_TABS {
            data.tabs.truncate(MAX_TABS);
        }
        for t in &mut data.tabs {
            if t.text.len() > MAX_TAB_TEXT_LEN {
                t.text.truncate(MAX_TAB_TEXT_LEN);
            }
        }
        // если active_id не среди tabs — сбросить на первый (или пусто)
        if !data.active_id.is_empty()
            && !data.tabs.iter().any(|t| t.id == data.active_id)
        {
            data.active_id = data.tabs.first().map(|t| t.id.clone()).unwrap_or_default();
        }
        let mut guard = self.data.write();
        *guard = data.clone();
        drop(guard);
        spawn_save(self.path.clone(), data);
    }
}

/// Путь к tabs.json (по образцу history::history_paths).
pub fn tabs_path() -> std::io::Result<PathBuf> {
    let dir = dirs::config_dir()
        .ok_or_else(|| std::io::Error::new(std::io::ErrorKind::NotFound, "config dir"))?
        .join("ttsbard");
    fs::create_dir_all(&dir)?;
    Ok(dir.join("tabs.json"))
}
```
- Проверь: `dirs::config_dir()` используется в `history.rs:342` — `%APPDATA%\ttsbard` на Windows. ОК.

async function flushSave() {
  if (saveTimer) { clearTimeout(saveTimer); saveTimer = null }

## Поведение для самопроверки (end-to-end, мысленно)
1. Старт → `initTabs()` → `get_tabs` → если tabs.json пуст/отсутствует → дефолтный «Текст 1».
2. Пользователь создаёт 2 вкладки, набирает текст → deep-watch → debounced `save_tabs` через 500мс.
3. Закрытие приложения → `onCloseRequested` → prevent → `flushTabsSave` → destroy.
4. Рестарт → `get_tabs` → 2 вкладки + активная восстановлены.

Если `initTabs` ещё не отработал, а пользователь уже набирает — `scheduleSave` не сработает (guard
`isHydrated`), но текст в `tabs.value` есть (in-memory); после hydrate последующие правки сохранятся.
При flush при выходе — сохранится всё текущее состояние regardless. Это допустимо.
