# Task 90-round2-01 (итерация по ревью Claude, Round 2)

Продолжение плана 90. Round 0 (task `90-round1-01.md`) выполнен DeepSeek и
проверен Claude в `docs/deepseek/reviews/90-round1-claude-review.md`. Найден
**1 блокирующий баг** (визуальная регрессия). Сборка зелёная: `vue-tsc` 0,
`cargo check` green.

## Цель — починить BLOCKER-1: скруглённые углы сломаны

В Round 0 фон перенесён на `body` (`body { background: var(--bg) }`), у которого
**нет** `border-radius`. Окно `transparent:true` + `decorations:false`, поэтому
раньше скругление 16px работало за счёт прозрачных углов вне `border-radius`.
Теперь `body` закрашивает всю прямоугольную площадь → углы стали **квадратными**
(регрессия против оригинала, где `body` был `transparent`).

## Что сделать (предпочтительный вариант): фон только на карточке

Фон должен быть **только на `.playback-window`** (у неё есть `border-radius:16px`),
а не на `body`. Карточку растянуть на всю площадь окна. Тогда углы вне радиуса
остаются прозрачными (скругление работает), а фон заполняет всю площадь
(пустоты нет).

### `src-playback/PlaybackControlApp.vue`

1. **Глобальный `<style>`:**
   - Вернуть `body { background: transparent }` (вместо `var(--bg)`).
   - Оставить `html, body { height: 100vh; margin: 0; }`.
   - Добавить `body { display: flex; }` — чтобы карточка-ребёнок растянулась по
     высоте на 100vh. Сохранить остальные стили body: `font-family`, `color`,
     `user-select: none`, `overflow: hidden`.

2. **Scoped `<style>` → `.playback-window`:**
   - Растянуть на всю площадь окна: добавить `width: 100%;` и `height: 100%;`
     (или `height: 100vh;`). Это гарантирует: карточка-фон = площадь окна во
     ВСЕХ состояниях (даже если автосайз на мгновение отстанет).
   - **Сохранить:** `border-radius: 16px`, `border: 1px solid var(--border)`,
     `padding: 12px`, `display: flex`, `flex-direction: column`, `gap: 12px`.
   - `min-height: 150px` можно оставить (не мешает).

3. **`<script setup>`:**
   - Удалить функцию `syncBodyBackground()` и **оба её вызова** (в `onMounted`
     и в обработчике `playback-appearance-update`). Фон теперь держится только
     на карточке через `:style="overlayStyle"`, который уже привязан к
     `.playback-window` (строка с `<div ref="playbackCard" class="playback-window" :style="overlayStyle">`).
   - **НЕ трогать** `resizeToFit()`, `playbackCard` ref, `watch`, импорт
     `LogicalSize`, константы `MIN_H`/`MAX_H`, все вызовы `resizeToFit()`.
     Автосайз остаётся как есть — он по-прежнему нужен для адаптивной высоты окна.

## Критерии приёмки (итерации)

1. **Скругление работает:** в запущенном окне углы скруглены 16px, НЕ квадратные.
   (Трассировка: вне `border-radius:16px` пиксели прозрачны — фон только на
   карточке.)
2. **Пустота устранена:** карточка-фон заполняет всю площадь окна во всех
   состояниях (Idle пусто / есть очередь+недавние / переходы между ними).
3. **Автосайз работает** как в Round 0 (высота под контент, 150–600, ширина 350).
4. Тёмная/светлая темы: фон корректный в обеих (`overlayStyle` = `var(--bg)` +
   opacity).
5. `npx vue-tsc --noEmit` — 0 errors.
6. Drag (`.window-header`), кнопки управления, реактивность — без регрессий.

## Не трогать

- `resizeToFit()`, template ref, `watch`, константы, импорты для resize.
- Команды воспроизведения, DTO `PlaybackStateDto`, конфиг Tauri
  (`tauri.conf.json`), настройки позиции окна (`playback_window.rs`,
  `config/windows.rs`).

## Контекст

В оригинале `body` был `transparent`, и скругление работало именно потому, что
карточка со скруглением рисовалась поверх прозрачного окна. Round 0 нарушил это,
положив фон на прямоугольный `body`. Возврат фона на карточку + растяжение
карточки на всю площадь окна чинит и пустоту, и скругление одновременно.
