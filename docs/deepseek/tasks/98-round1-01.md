## Фикс

### 1. Backend — дефолт ON (`src-tauri/src/config/settings.rs`)
`EditorSettings` (~строка 445-457). Сейчас:
```rust
#[serde(default)]
pub struct EditorSettings {
    #[serde(default)] pub quick: bool,
    #[serde(default)] pub ai: bool,
    #[serde(default)] pub ai_completion: bool,
    #[serde(default)] pub spellcheck_enabled: bool,        // ← default false
    #[serde(default)] pub spellcheck_source: SpellSource,
}
```
Сделать явный `impl Default for EditorSettings` с `spellcheck_enabled: true` и
`spellcheck_source: SpellSource::Offline` (остальные поля — как были: false).
**Важно:** `#[serde(default)]` на struct означает «при отсутствующем поле использовать Default».
Явный impl Default даст `spellcheck_enabled: true` для новых конфигов. Существующие конфиги с
явным `spellcheck_enabled: false` НЕ изменятся (serde прочитает значение). Проверить, есть ли
уже `derive(Default)` — если есть, заменить на явный impl (как делали для AiDeepSeekSettings в
плане 93).

⚠️ **Проверить другие поля:** `quick`, `ai`, `ai_completion` — их дефолт должен остаться `false`.
Не сломать. Если у EditorSettings уже есть derive(Default) и он где-то используется — оставить
поведение совместимым.

### 2. Frontend — тумблер в «Редактор»
**Добавить в `src/components/settings/SettingsEditor.vue`** чекбокс орфографии (по образцу
существующего «Быстрый редактор»):
```ts
const spellcheckEnabled = computed(() => editorSettings.value?.spellcheck_enabled ?? true)
async function toggleSpellcheck() {
  try {
    const newValue = !(editorSettings.value?.spellcheck_enabled ?? true)
    await invoke('set_editor_spellcheck_enabled', { enabled: newValue })
    emit('show-message', 'Настройка сохранена')
  } catch (e) {
    const errorMessage = e instanceof Error ? e.message : String(e)
    emit('show-message', 'Ошибка переключения орфографии: ' + errorMessage)
  }
}
```
