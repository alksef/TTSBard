# Stage 30, round 1, task 01

## Цель

Реализовать явный sample-rate контракт DeepFilterNet в backend audio pipeline.

## Файлы

- `src-tauri/src/audio/effects.rs`

## Требования

1. Сохранить вариант A из `docs/stage/30-deepfilternet-resampling-invariant.md`: при успешном DeepFilterNet output остаётся на фактическом sample rate модели (сейчас 48 kHz), без обратного ресемплинга.
2. Изменить `apply_enhance()` так, чтобы она возвращала samples вместе с фактическим sample rate результата. Не оставлять контракт только в виде `Vec<f32>`.
3. В `apply_effects()` использовать возвращённый sample rate при создании итогового `AudioPcm`; убрать безусловное присваивание `sample_rate = 48000`.
4. При ошибке enhancement с `fail_on_enhance_error = false` сохранить исходные samples и исходный sample rate. При `true` вернуть ошибку.
5. Не менять порядок обработки: `DeepFilterNet → Signalsmith Stretch → DSP → playback`.
6. Сохранить interleaved layout, число каналов и конечность `f32`. Не добавлять export policy и обратный ресемплинг в этой задаче.
7. Расширить unit-тесты в `effects.rs`: проверить успешный enhancement на входах 16_000, 22_050, 24_000, 44_100 и 48_000 Hz; sample rate результата должен быть 48_000 Hz, длительность около 0.2 s, channels/layout и finite output сохраняются. Добавить короткий mono-сигнал, stereo/interleaved сигнал и проверки disabled enhancement/fallback.
8. Если прямой вызов DeepFilterNet делает тесты слишком тяжёлыми или нестабильными, вынести малый тестируемый helper для контракта/метаданных, но обязательно оставить хотя бы один интеграционный тест реального `apply_effects()` с enhancement.

## Проверка

- `cargo test --manifest-path src-tauri/Cargo.toml audio::effects::tests`

Не изменять другие файлы и не делать commit.
