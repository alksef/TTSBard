# Task 112 / Round 1 / Iteration 01 — синхронизировать голос Silero перед кешированием

## Контекст

При сохранении результата TTS голос для `PhraseEntry.voice` и cache key берётся из `settings.tts.telegram.current_voice_id` в `src-tauri/src/commands/mod.rs`. В `src-tauri/src/commands/telegram.rs` команда `telegram_get_current_voice` получает актуальный голос из Telegram, но сейчас только возвращает его вызывающему коду. После запуска приложения или обновления голоса UI знает голос, а общий settings cache может оставаться с пустым `current_voice_id`.

## Что сделать

1. В `telegram_get_current_voice` добавь необходимые `SettingsManager`/`AppHandle` параметры по существующим паттернам команд проекта.
2. После успешной авторизации и получения `Some(CurrentVoice)` синхронизируй `voice.id` с `settings.tts.telegram.current_voice_id` через существующий безопасный механизм сохранения настроек. Не выполняй запись, если ID пустой или уже совпадает.
3. Сохрани текущие ответы и ошибки команды: при отсутствии авторизации возвращается `Ok(None)`, при отсутствии текущего голоса настройки не меняются.
4. Не меняй формат `PhraseEntry`, cache key или логику других TTS-провайдеров.
5. Обнови зарегистрированный вызов команды/места использования только если сигнатура требует этого; Tauri автоматически разрешает дополнительные `State`/`AppHandle` параметры.

## Проверки

- `npx vue-tsc --noEmit`
- `git diff --check`

Не изменяй несвязанные пользовательские правки и не создавай коммит.
