## КРИТИЧЕСКОЕ архитектурное уточнение (важнее плана!)

**«Двойной источник» — это НЕ два файла.** Разведка подтвердила:
- **Единственный persistent файл:** `soundpanel_bindings.json` (голый `Vec<SoundBinding>` сейчас).
- **In-memory source of truth:** `SoundPanelState.bindings` (HashMap).
- **DTO-проброс:** `get_all_app_settings` (commands/mod.rs:1203) зовёт
  `soundpanel_state.get_all_bindings()` → кладёт в `AllSourcesParams.soundpanel_bindings` →
  `AppSettingsDto.soundpanel_bindings`. Это RUNTIME-проброс из state, НЕ отдельное хранилище.
- `SoundPanelTab.vue:204` читает `appSettings.value.soundpanel_bindings` (тот же проброс).
- `SoundPanelApp.vue:47` читает `sp_get_bindings` (тот же state).

⇒ **Миграция проще:** один файл + state-структура + DTO-проброс. НЕ надо «убирать дубль с
appSettings» (дубля нет — это view на state). Вместо этого: DTO `soundpanel_bindings` должен
продолжать отдавать bindings **активного набора** (обратная совместимость).

## Цель
Несколько наборов (Sets) звуков, каждый со своими A-Z. Переключение активного набора — в основном
окне и в окне вызова. Бэк-совместимость: старый `soundpanel_bindings.json` (Vec) → один Set «Основной».

## Часть A — Бэкенд (Rust)

### A1. `src-tauri/src/soundpanel/state.rs` — новые структуры + state
1. Новые структуры (вместе с существующим `SoundBinding`, НЕ удаляя его):
   ```rust
   #[derive(Debug, Clone, Serialize, Deserialize, Default)]
   pub struct SoundSet {
       #[serde(default)] pub id: String,
       #[serde(default)] pub name: String,
       #[serde(default)] pub bindings: Vec<SoundBinding>,
   }

   #[derive(Debug, Clone, Serialize, Deserialize, Default)]
   pub struct SoundSets {
       #[serde(default)] pub active_set_id: String,
       #[serde(default)] pub sets: Vec<SoundSet>,
   }
   ```
2. `SoundPanelState` — заменить поле `bindings: Arc<Mutex<HashMap<char, SoundBinding>>>` на
   `sets: Arc<Mutex<SoundSets>>`. **Внимание:** проверь ВСЕ места, использующие `self.bindings`
   (grep `self.bindings` / `.bindings` в state.rs) — перепиши под `sets`.
3. Методы state (перепиши под Sets):
   - `get_all_bindings()` → bindings **активного набора** (отсортированный Vec<SoundBinding>),
     как сейчас (обратная совместимость — `sp_get_bindings`/`sp_play_binding` не меняют интерфейс).
     Если `active_set_id` невалиден или sets пуст → вернуть пустой Vec (или bindings первого).
   - `get_sets() -> SoundSets` — clone всех наборов.
   - `get_active_set() -> SoundSet` — активный набор (или первый/пустой).
   - `set_active_set(id: &str)` — сменить active_set_id (в памяти; persistence через save_sets).
   - `add_set(name: &str) -> SoundSet` — создать набор (id = gen), добавить, сделать активным.
   - `rename_set(id: &str, name: &str)`.
   - `remove_set(id: &str)` — удалить; если удалили активный → выбрать соседний/первый; НЕ удалять
     аудиофайлы (могут использоваться в других наборах).
   - `get_binding(key) -> Option<SoundBinding>` → из **активного** набора.
   - `add_binding_to_set(set_id: &str, binding: SoundBinding)` (или `add_binding` для активного).
   - `remove_binding_from_set(set_id: &str, key: char)`.
   - gen id: helper `fn gen_set_id() -> String` (как `genId` во фронте — `crypto` недоступен в Rust,
     используй счётчик AtomicU64 ИЛИ `uuid` crate ЕСЛИ уже в deps. Проверь Cargo.toml — если uuid
     есть, используй; иначе AtomicU64 counter с префиксом `"set-"`).
4. `play_sound` (state.rs:136) — без изменений (принимает binding, играет файл).

### A3. `src-tauri/src/soundpanel/bindings.rs` — команды
1. Существующие:
   - `sp_get_bindings` — БЕЗ ИЗМЕНЕНИЙ (возвращает bindings активного набора — `get_all_bindings()`).
   - `sp_add_binding(key, description, file_path)` — добавить параметр `set_id: Option<String>`

2. Новые команды:
   ```rust

   #[tauri::command] pub fn sp_get_active_set(state) -> SoundSet { state.get_active_set() }
   #[tauri::command] pub fn sp_set_active_set(app_handle, state, id: String) -> Result<(), String>
   #[tauri::command] pub fn sp_add_set(app_handle, state, name: String) -> Result<SoundSet, String>
   #[tauri::command] pub fn sp_rename_set(app_handle, state, id: String, name: String) -> Result<(), String>
   ```
