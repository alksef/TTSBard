# Task 77-round1-01: Editor Tabs (план 77, stage 06)

Реализуй фичу «вкладки редактора» по плану `docs/plans/77-2026-06-30-editor-tabs.md`.
Прочитай этот план ПОЛНОСТЬЮ перед началом — там точные точки интеграции, архитектура,
риски и критерии готовности. Также прочитай `docs/stage/06-editor-tabs-multiple-texts.md`.

## Суть
Фронтенд-only: строка табов над редактором, каждый таб — независимый текст. Переключение
мгновенное, без потери содержимого. Persist — **только в памяти сессии** (бэкенд НЕ трогать).
Это **отдельная сущность от drafts** (план 75, `PhraseHistoryList`) — НЕ merge, НЕ дублировать.

## Что создать / изменить (точно по плану 77)

### 1. НОВЫЙ `src/composables/useEditorTabs.ts`
Composable с CRUD табов. Интерфейс `EditorTab { id, title, text }`. Функции: `create`,
`close`, `select`, `rename`. Computed `active` (get/set). **Всегда ≥1 таб** (close последнего
→ создать пустой и выбрать). `genId()` через `crypto.randomUUID()` с fallback на счётчик.
Полный код-скелет есть в плане 77 — следуй ему. НЕ module-level singleton (локальный инстанс).

### 2. НОВЫЙ `src/components/editor/EditorTabs.vue`
Строка табов. Props: `tabs`, `activeId`. Emits: `create`, `close`, `select`, `rename`.
- Каждый таб: `title` (дабл-клик → inline `<input>` переименование) + кнопка `×` (закрыть).
- Активный таб подсвечен `--color-accent`.
- Кнопка `+` — создать таб (emit `create`).
- **Только CSS-vars** для темы (light/dark автоматически): `--color-bg-elevated`,
  `--color-border`, `--color-accent`, `--color-text-primary`, `--color-text-muted`.
- **Minimal-mode:** строка табов ВИДНА в minimal (в отличие от `correct-button`).
  `overflow-x: auto` на контейнере табов — узкие табы скроллятся по горизонтали.

### 3. ИЗМЕНИТЬ `src/components/InputPanel.vue`
- Импортировать `useEditorTabs`, `EditorTabs`.
- `const text = ref('')` (строка 14) → **computed-прокси** к активному табу:
  ```ts
  const text = computed<string>({
    get: () => active.value.text,
    set: (v) => { active.value.text = v },
  })
  ```
  `<TtsEditor v-model="text">` **остаётся без изменений** — CodeMirror через существующий
  `watch(modelValue)` подхватит смену документа при переключении таба.
- Добавить `ref` на `TtsEditor` (`editorRef`) для фокус-менеджмента.
- Обработчики `onCreate`/`onSelect`: после переключения `nextTick(() => editorRef.value?.focus())`.
- В шаблон, **над** `.textarea-wrapper` (внутри `.input-group`, строка ~200), добавить:
  ```vue
  <EditorTabs
    :tabs="tabs"
    :active-id="activeId"
    @create="onCreate"
    @close="closeTab"
    @select="onSelect"
    @rename="renameTab"
  />
  ```
- `handleEnter` (quick-editor, ~144-165) очищает `text.value = ''` после speak — с
  computed-прокси это очистит **активный** таб. Это КОРРЕКТНО. Добавь короткий комментарий.

## Важные ограничения (НЕ нарушать)
- **Бэкенд НЕ трогать.** Никаких Rust-команд, `tauri.conf.json`, `lib.rs`. Только фронт.
- **Не дублировать** с `PhraseHistoryList` (план 75) — табы отдельная сущность.
- **Не пересоздавать** CodeMirror — используется существующий `watch(modelValue)` в `TtsEditor`.
- **Без подтверждения** при закрытии таба с текстом (in-memory, текст не критичен).
- Тема — только CSS-vars, никаких хардкод-цветов.

## Критерии готовности (проверь сам перед завершением)
1. Создание/переключение/закрытие табов не теряет текст активного таба.
2. Всегда ≥1 таб (нельзя закрыть последний до пустоты).
3. CodeMirror не пересоздаётся при переключении.
4. Дабл-клик по title → переименование (inline input).
5. Minimal-mode: табы видны, `overflow-x: auto` работает.
6. Light/dark тема через CSS-vars.
7. `npx vue-tsc --noEmit` — **0 ошибок**. Запусти сам и убедись.

## Не делай
- НЕ пиши юнит-тесты (в проекте их нет для компонентов).
- НЕ трогай `TtsEditor.vue` (он уже умеет менять документ через watch).
- НЕ делай сохранение курсора/скролла при переключении — это отдельный будущий этап
  (зафиксировано в плане 77 как отложенное).

После правок — запусти `npx vue-tsc --noEmit` и приложи результат. Если есть ошибки —
