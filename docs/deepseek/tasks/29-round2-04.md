# Task 29 Round 2 Iteration 4 — Dedicated scroll area below preview

## Goal

Replace the fragile sticky-preview approach with a structural layout: the `Проверка эффектов` panel stays outside the settings scroll container, and only the panels below it scroll.

## Bounded file

- `src/components/AudioPanel.vue` only

## Target DOM structure

Inside the `Эффекты и DSP` tab:

```text
.unified-tab
├── .preview-panel-fixed
└── .effects-scroll
    ├── voice effects section
    ├── DeepFilterNet section
    ├── voice-effects save section
    ├── DSP section
    └── DSP save section
```

The names may differ, but the preview must be a sibling before the scroll container, not a child of it.

## Required behavior

1. `Проверка эффектов` remains visible while the settings below scroll.
2. Do not use `position: sticky` for the preview.
3. There must be exactly one settings scrollbar in the unified tab: `.effects-scroll` with `overflow-y: auto`.
4. Avoid hardcoded viewport heights such as `calc(100vh - 280px)`. Use the available panel height through flex layout (`height: 100%`, `min-height: 0`, `flex: 1`) so the layout adapts to the existing app shell and window resize.
5. Keep tab navigation outside the settings scroll area. It may remain visually normal; do not add a new opaque background.
6. Preserve current draft preview behavior, collapsible sections, save/cancel actions, themes, and narrow-width behavior.
7. The scroll container must prevent horizontal overflow without creating another vertical scroll context on `.unified-tab`.

## CSS guidance

- `.audio-panel`: fill available height with `height: 100%`, `min-height: 0`, and column flex layout where compatible with the existing device tab.
- `.unified-tab`: column flex, `flex: 1`, `min-height: 0`, `overflow: hidden`.
- preview wrapper: `flex: 0 0 auto`.
- `.effects-scroll`: `flex: 1 1 auto`, `min-height: 0`, `overflow-y: auto`, `overflow-x: hidden`.

Do not make the scroll container cover the preview.

## Acceptance criteria

- Preview does not move when scrolling EQ/compressor/limiter settings.
- Settings scroll starts below preview and never slides underneath it.
- No white strip caused by nested or mismatched scroll containers.
- No opaque white tab background regression.
- `npx vue-tsc --noEmit` passes.
