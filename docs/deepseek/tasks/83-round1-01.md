# Task 83-round1-01: Исправить очистку не того таба после speak (quick-editor race)

По плану `docs/plans/83-2026-06-30-quick-editor-clear-wrong-tab.md`. Прочитай план — там
воспроизведение, причина, решение с кодом.

## Суть бага
`text` — computed-прокси к **активному** табу (план 77). В quick-editor mode `speak()` идёт
fire-and-forget, `recordHistory()` и очистка читают `text.value` **после** await-паузы. Если
пользователь переключил таб за это время — читается/очищается **другой** таб, не тот, с которого
отправили текст.

## Файл
`src/components/InputPanel.vue`

## Что сделать

### 1. `speak(textToSend)` — принимать текст параметром
```ts
async function speak(textToSend: string) {
  if (!textToSend.trim()) return
  try {
    debugLog('[InputPanel] Speaking:', textToSend)
    await invoke('speak_text', { text: textToSend })
    recordHistory(textToSend)
  } catch (e) {
    debugError('[InputPanel] Failed to speak:', e)
    showError(e as string)
  }
}
```
Не читать `text.value` внутри speak — только параметр.

### 2. `recordHistory(textToRecord)` — принимать текст параметром
```ts
async function recordHistory(textToRecord: string) {
  try {
    await invoke('record_history', { text: textToRecord })
  } catch {
    // silently fail
  }
}
```

### 3. `handleEnter()` — захватить текст и id таба СИНХРОННО до await
```ts
async function handleEnter() {
  // Захват СИНХРОННО: текст и id активного таба до любых await.
  // Иначе при переключении таба во время fire-and-forget speak очистится не тот таб.
  const currentText = text.value
  const senderTabId = activeId.value

  if (!currentText.trim()) return

  const quickEditorEnabledValue = editorSettings.value?.quick ?? false
  if (quickEditorEnabledValue && !currentText.trim()) return

  if (quickEditorEnabledValue) {
    speak(currentText)              // fire-and-forget, но текст уже зафиксирован
    // Очистить ИМЕННО тот таб, с которого отправили (по id, не по active):
    const tab = tabs.value.find(t => t.id === senderTabId)
    if (tab) tab.text = ''
    await hideMainWindow()
  } else {
    await speak(currentText)
    const tab = tabs.value.find(t => t.id === senderTabId)
    if (tab) tab.text = ''
  }
}
```

### 4. Проверить остальные вызовы speak/recordHistory
- `grep` по `speak(` и `recordHistory(` в `InputPanel.vue` — если есть другие вызовы со старой
  сигнатурой (без аргумента), обновить их: передавать `text.value` (текущий активный текст).
- `correctText`/`completeText` уже читают `text.value` синхронно в начале и пишут в `text.value`
  (active) — их НЕ трогать, они работают с активным табом в момент клика (не fire-and-forget).

## Важные ограничения
- НЕ менять поведение normal mode (не-quick) — просто текст передаётся явно.
- НЕ вводить race: очистка по `senderTabId` (find по tabs), а не по `active` computed —
  чтобы переключение таба не повлияло.
- НЕ ломать `recordHistory` (он вызывается только из speak — пробросить параметр).

## Критерии готовности
1. `speak(text)` и `recordHistory(text)` — параметр, не читают `text.value` после await.
2. `handleEnter` захватывает `currentText` + `senderTabId` синхронно до await.
3. Очистка по `senderTabId` (find в tabs), не по active.
4. Все вызовы speak/recordHistory обновлены под новую сигнатуру.
5. **Тест-кейс:** таб 1 → Enter → переключиться на таб 2 → после TTS таб 1 очищен, таб 2 цел.
