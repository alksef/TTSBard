# Task 29 Round 1 Iteration 1 — DSP backend core and settings

## Goal

Implement the backend/data-model part of the DSP post-processing feature described in `docs/stage/29-dsp-audio-postprocessing.md`. This task is deliberately limited to the DSP core and persisted settings; do not redesign the Vue UI yet.

## Bounded files

- `src-tauri/src/audio/effects.rs`
- `src-tauri/src/audio/mod.rs` if exports are required
- `src-tauri/src/config/settings.rs`
- `src-tauri/src/config/dto.rs`
- `src-tauri/src/commands/playback.rs` and `src-tauri/src/lib.rs` only for the minimum settings command wiring
- `src/types/settings.ts` only if the persisted DTO type must stay aligned
- `src-tauri/Cargo.toml` only if strictly necessary; prefer no new dependency

Do not modify Vue templates/styles in this iteration.

## Required behavior

1. Add a serializable DSP settings model with serde defaults and backward-compatible loading. It must contain independent `enabled` flags and parameters for EQ, compressor, and limiter.
2. Keep DeepFilterNet settings separate from DSP settings.
3. Implement native Rust DSP processing over interleaved `f32` PCM, preserving sample rate, channels, frame count (except no intentional time processing), and finite samples.
4. Implement at least:
   - biquad EQ with low-cut, up to three parametric bands, and high-shelf;
   - soft-knee compressor with threshold, ratio, attack, release, knee, and makeup gain;
   - safety limiter with ceiling and release; lookahead may be implemented as a bounded/simple variant if the existing architecture makes a true delay impractical, but it must never create NaN/Inf or unbounded gain.
5. Apply DSP after the existing DeepFilterNet/Signalsmith steps in `apply_effects`, with each block independently bypassable. Do not change the existing playback volume semantics.
6. Add focused unit tests for mono/stereo layout, multiple sample rates, silence, finite output, compressor/limiter gain reduction, and limiter ceiling tolerance.
7. Add the minimum settings API/persistence plumbing needed for future UI use, with validation/clamping of user-controlled numeric fields.

## Existing source of truth to inspect first

- `src-tauri/src/audio/effects.rs` already owns `AudioPcm` and the current effect order.
- `src-tauri/src/config/settings.rs` owns persisted settings and defaults.
- `src-tauri/src/config/dto.rs` owns frontend DTO conversion.
- `src-tauri/src/commands/playback.rs` contains the existing audio-effects settings commands.
- Preserve the project's current naming/serde conventions and fallback behavior.

## Acceptance criteria

- `cargo check --manifest-path src-tauri/Cargo.toml` succeeds.
- Rust unit tests for the new DSP code pass.
- Existing DeepFilterNet and Signalsmith tests remain intact.
- Existing configs without DSP fields deserialize successfully.
- No Vue template/style changes are included in this iteration.
- Report files changed and any limitations, especially around limiter lookahead.
