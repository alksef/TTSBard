## Архитектурное решение (рекомендация, закрепить в коде)
**Не плодить второй hook/state.** Intercept-состояние живёт **внутри**
`SoundPanelState` (persisted `intercept.json`, загружается при `SoundPanelState::new`),
доступно из hook-proc через существующий `SP_HOOK_STATE`. Один hook обслуживает и
(будущее) soundpanel-режим, и Intercept — переключается по persisted-toggle.

```rust
pub const INTERCEPT_FILE: &str = "intercept.json";

#[derive(Serialize, Deserialize, Clone, Default)]
pub struct InterceptSettings {
    pub enabled: bool,
    pub bindings: Vec<InterceptBinding>,

#[derive(Serialize, Deserialize, Clone)]
pub struct InterceptBinding {
    pub key: String,    // canonical VK name: "NUMPAD0".."NUMPAD9","NUMPAD_ADD",

                        // "NUMPAD_DECIMAL","F1".."F24"

                        // | "show_playback_control_window"
                      //   | "playback_pause" | "playback_stop" | "playback_repeat"
}
// load(path) / save(path, &settings) — по образцу storage.rs (serde_json, %APPDATA%/ttsbard/)
```

### B3. Actions-слой: вынести handle_* в публичные reusable-функции
В `hotkeys.rs` (или новом `src-tauri/src/actions.rs`, импортируемом из hotkeys — на выбор,
но оставить hotkey-регистрации рабочими):
- Сделать функции публичными и вызываемыми по строке action:
```rust
pub fn run_action(app_handle: &AppHandle, action: &str) {
    match action {
        "show_main_window"         => handle_main_window(app_handle.clone()),
        "show_soundpanel_window"   => handle_sound_panel(/* app_state */, app_handle.clone()),
        "show_playback_control_window" => handle_playback_control_window(app_handle.clone()),
        "playback_pause"           => handle_playback_pause(app_handle.clone()),
        "playback_stop"            => handle_playback_stop(app_handle.clone()),
        "playback_repeat"          => handle_playback_repeat(app_handle.clone()),
        other => warn!(action = other, "Unknown intercept action"),
    }
}
```
- `handle_sound_panel` нужен `AppState` — получить через `app_handle.state::<AppState>()`
  внутри `run_action` (НЕ тащить параметром). Проверить, что `AppState` — Tauri state (manage).
- Hotkey-регистрации продолжают вызывать `handle_*` напрямую (без изменений) — `run_action`
  лишь обёртка для Intercept.

### B4. Hook: Intercept-логика в proc
В `hook.rs:soundpanel_keyboard_proc`:
- Когда срабатывает `WM_KEYDOWN|WM_SYSKEYDOWN`:
  1. Взять `InterceptSettings` из `SP_HOOK_STATE` (`state.get_intercept()`).
  2. Если **не** `enabled` → `CallNextHookEx` (проходит насквозь). Это заменяет
     старую проверку `is_interception_enabled()` (soundpanel runtime-флаг теперь
     не управляет hook — ЧАСТЬ A).
  3. Если `enabled`: перевести `vk_code` в canonical name (см. B5 карту VK) и
     найти `InterceptBinding` по `key`.
     - Есть биндинг → вызвать `run_action(app_handle, &binding.action)` + **вернуть
       `LRESULT(1)`** (фильтруем клавишу, не отдаём в систему).
     - Нет биндинга → `CallNextHookEx` (проходит насквозь — незабинженные клавиши
       НЕ блокируем).
- **НЕ гасить** toggle после срабатывания (Intercept работает, пока пользователь не выключит).
- Нужен `AppHandle` внутри proc. Завести `static APP_HANDLE: OnceLock<AppHandle>`
  (или хранить в SoundPanelState), установить при `initialize_soundpanel_hook`.
  `run_action` вызывать через этот handle.

### B5. Карта VK ↔ canonical name (для B4 и UI)
Реализовать helper `fn vk_to_name(vk_code: u32) -> Option<String>` (в `intercept.rs` или `hook.rs`):
- `0x60..=0x69` → `NUMPAD0..NUMPAD9`
- `0x6A` → `NUMPAD_MULTIPLY`, `0x6B` → `NUMPAD_ADD`, `0x6D` → `NUMPAD_SUBTRACT`,
  `0x6E` → `NUMPAD_DECIMAL`, `0x6F` → `NUMPAD_DIVIDE`
- `0x70..=0x87` → `F1..F24`
- прочее → `None` (вне диапазона — игнорировать)

## Критерии приёмки (итерации)
1. `intercept.json` создаётся в `%APPDATA%/ttsbard/`, persist между запусками.
2. Toggle «Перехват» вкл/выкл сохраняется; состояние восстанавливается при старте.
3. Забинженная NumPad/F-клавиша, когда toggle вкл, **не доходит до системы**
   (LRESULT(1)) и вызывает выбранный action (открытие окна / playback-команда).
4. Незабинженные клавиши проходят насквозь (CallNextHookEx).
5. Actions = ровно 6, переиспользуют `handle_*` (через `run_action`).
6. Перехват работает, даже когда главное окно в фокусе.
7. Панель «Перехват» в сайдбаре: toggle + запись (только NumPad/F) + выбор action + очистка.
8. Toggle НЕ гаснет после срабатывания клавиши.
9. `npx vue-tsc --noEmit` (корень) → 0 errors; `cargo check` (`src-tauri/`) → green (0 errors, минимум warnings).
