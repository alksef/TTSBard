# Task 29 Round 2 Iteration 3 — Fix sticky preview and tab background

## Problem

The current screenshot shows two regressions:

1. `Проверка эффектов` is not staying pinned at the top while the app scrolls.
2. The tab bar background became an opaque white rectangle in the light theme.

## Bounded file

- `src/components/AudioPanel.vue` only

## Required fix

1. Remove `overflow-x: hidden` from `.unified-tab`. Any non-visible overflow can establish a containing scroll context and prevent the sticky preview from sticking to the app's `.main-content` scroll owner.
2. Keep `.audio-tabs` sticky at `top: 0`, but use a transparent/glass-compatible background consistent with the app shell instead of opaque `var(--color-bg-panel)`. The tab bar must not become a white block in light theme or an unintended solid block in dark theme.
3. Keep `.preview-panel-sticky` below the sticky tab bar using the existing offset, without introducing another scroll container.
4. Do not alter preview logic, settings, DSP, or backend code.

## Acceptance criteria

- Sticky preview remains visible at the top of the app content while scrolling.
- Preview does not cover the tab buttons.
- Tab bar background shows the existing app gradient/glass surface, not an opaque white rectangle.
- No horizontal overflow regression at narrow width.
- `npx vue-tsc --noEmit` passes.
