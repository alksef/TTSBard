# Task 106-round1-02: Extract DSP UI as presentational subpanel

Предыдущая попытка полностью перенести DSP state зацепила preview, потому что `AudioEffectsTab` использует `draftDsp` для `preview_audio_file`. Сделай безопасный вариант.

Создай `src/components/audio/DspSettings.vue` как presentational-компонент: перенеси туда только DSP template и DSP-specific styles (presets, EQ, Compressor, Limiter, DSP save section). Не переносить туда composables, draft state или invoke.

`DspSettings.vue` получает через props:

- `draftDsp`, `dspDirty`, `dspSaveStatus`, `dspSaveError`;
- `dspMainCollapsed`, `dspPreset`, `dspCollapsed`.

Для действий объяви emits с типизированными payload:

- `mark-dirty`;
- `set-preset` (`natural | clear`);
- `toggle-main`;
- `toggle-section` (`eq | compressor | limiter`);
- `save`, `cancel`.

В `AudioEffectsTab.vue` замени старый DSP template на `<DspSettings ... />`, сохранив в нём весь текущий DSP state, watcher, preset/save/cancel функции и `draftDsp`, чтобы preview продолжил работать без изменений. Не менять поведение и backend-команды.

Используй небольшие Edit/apply-patch операции, не shell-скрипты для записи больших файлов. Проверки: `npx vue-tsc --noEmit`, `git diff --check`. Не трогай task-файлы с занятыми именами и не форматируй Rust.
