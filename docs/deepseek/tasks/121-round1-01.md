# Task 121-round1-01: Интеграция нативного DeepFilterNet в аудиопайплайн (Rust + Vue)

Вы должны интегрировать нативный Rust-крейт `deep_filter` для очистки аудио в реальном времени.

## Шаг 1: Изменение Cargo.toml
Добавьте git-зависимость в `src-tauri/Cargo.toml`:
```toml
deep_filter = { git = "https://github.com/Rikorose/DeepFilterNet", subdirectory = "libDF", features = ["tract", "default-model"] }
```
Запустите `cargo check` для проверки успешности загрузки и компиляции.

## Шаг 2: Добавление настроек в конфигурацию
1. В `src-tauri/src/config/settings.rs` обновите `AudioEffects`:
   - Добавьте `pub enhance_enabled: bool` (дефолт: `false`).
   - Добавьте `pub enhance_atten_db: f32` (дефолт: `12.0`).
   - Обновите все конструкторы, мьютаторы и дефолтные реализации.
2. В `src-tauri/src/config/dto.rs` обновите `AudioEffectsDto` соответствующим образом.
3. В `src/types/settings.ts` обновите интерфейс `AudioEffectsDto`.

## Шаг 3: Реализация инференса в `effects.rs`
1. В `src-tauri/src/audio/effects.rs` импортируйте `deep_filter::tract::{DfTract, DfParams, RuntimeParams}`.
2. Создайте статический или лениво инициализируемый инстанс `DfTract` (например, через `parking_lot::Mutex` или `std::sync::OnceLock`). Мы должны загрузить модель ровно один раз!
3. Внутри функции `apply_effects`:
   - Если `effects.enhance_enabled` включен, выполните обработку f32 семплов.
   - Нам нужно разбить деинтерливнутые PCM-данные на фреймы размером `hop_size` и прогнать через `DfTract::process` (или аналогичный метод обработки фреймов в `deep_filter`). Обратите внимание на многоканальность (модель DeepFilterNet ожидает моно или стерео, настроенный через `RuntimeParams::default_with_ch(channels)`).
   - Пример инициализации модели:
     ```rust
     let rp = RuntimeParams::default_with_ch(channels).with_atten_lim(effects.enhance_atten_db);
     let df_params = DfParams::default();
     let mut df = DfTract::new(df_params, &rp)?;
     ```
   - Запустите обработку и замените `samples` обработанными значениями перед ре-энкодингом в WAV.

## Шаг 4: Обновление команд Tauri и Vue
1. В `src-tauri/src/commands/audio.rs` (или `effects.rs`) обновите Tauri команды:
   - `set_audio_effects_enhance_enabled(state: State<AppState>, enabled: bool)`
   - `set_audio_effects_enhance_atten_db(state: State<AppState>, atten_db: f32)`
   - Не забудьте зарегистрировать их в `src-tauri/src/lib.rs`.
2. В `src/components/tts/AudioEffectsPanel.vue` добавьте:
   - Переключатель (Toggle) для очистки.
   - Слайдер для регулирования глубины очистки (`atten_lim_db` от 5 до 30 dB). Чем меньше значение db, тем мягче очистка.

Выполните задачу аккуратно. Код должен успешно собираться (`cargo check` + `npx vue-tsc --noEmit`).
