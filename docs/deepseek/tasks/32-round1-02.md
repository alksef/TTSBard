# Task 32-round1-02: настройка и per-phrase boundary cleanup

## Цель

Подключить подготовленный `audio::boundary::process_boundaries` к backend pipeline одной TTS-фразы и добавить сохраняемую настройку, которой позже будет управлять UI. Настоящую склейку соседних фраз и UI не делать в этом task.

## Предыдущий результат

В `src-tauri/src/audio/boundary.rs` уже есть `process_boundaries`, `process_boundaries_with_config` и `crossfade`. Первый task проверен независимо: 15 boundary-тестов проходят, `cargo check` проходит.

## Разрешённые файлы

- `src-tauri/src/config/settings.rs`;
- `src-tauri/src/config/dto.rs`;
- `src-tauri/src/commands/tts_pipeline.rs`;
- `src-tauri/src/audio/effects.rs` — только если нужен аккуратный вызов boundary processor в общем pipeline;
- `src-tauri/src/commands/playback.rs` — только если требуется сохранить/прочитать настройку существующими командами;
- связанные backend-тесты.

Не изменять `src/components`, `src/types`, `src-tauri/src/playback.rs` и `src-tauri/src/audio/device.rs`.

## Требуемое поведение

1. Добавить в `AudioEffectsSettings` поле `boundary_cleanup_enabled: bool`.
2. Значение по умолчанию — `true` для новых установок и при чтении старого конфига без этого поля.
3. Провести поле через DTO/serde так, чтобы старые настройки загружались без ошибки.
4. В `apply_audio_effects_pipeline` выполнять boundary cleanup после decode/основных per-phrase эффектов и до передачи PCM в playback. Порядок должен быть документирован в коде.
5. Boundary cleanup не должен включать DeepFilterNet, DSP или изменять их параметры.
6. При выключенной настройке вернуть PCM без boundary cleanup; DeepFilterNet и DSP должны продолжить работать как раньше.
7. При ошибке/невалидном результате необязательной boundary-обработки сохранить исходный PCM как fallback, не ломая воспроизведение.
8. Не подключать `crossfade` между разными queued phrases в этом task.
9. Проверить, что export/synthesis-only путь не получает playback boundary cleanup автоматически, если он явно не проходит через playback pipeline.

## Тесты

Добавить или обновить backend-тесты для:

- default старой конфигурации без поля;
- сериализации/десериализации `boundary_cleanup_enabled`;
- включённой настройки в pipeline;
- выключенной настройки;
- сохранения DeepFilterNet/DSP поведения при любом значении boundary-флага;
- сохранения sample rate, channels и frame count.

## Ограничения

- Не запускать crossfade для одиночной фразы.
- Не менять API DeepFilterNet.
- Не добавлять UI-поля или frontend fallback в этом task.
- Не переформатировать посторонние файлы.
