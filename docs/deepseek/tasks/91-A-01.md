# Task 91-A-01: ЧАСТЬ A — SoundPanel как обычное окно (план 91)

План: `docs/plans/91-2026-07-08-keyboard-input-redesign-soundpanel-window-and-intercept.md`
Stage: `docs/stage/11-keyboard-input-mechanism-redesign.md`

Это **только ЧАСТЬ A** (SoundPanel как окно). ЧАСТЬ B (Intercept) — отдельная
итерация, **не делать** в этой задаче.

## Контекст

Сегодня SoundPanel открывается глобальным хоткеем → `set_interception_enabled(true)`
→ низкоуровневый `WH_KEYBOARD_LL` hook (`soundpanel/hook.rs`) ловит **A-Z глобально**
→ играет звук → **сам гасит перехват и скрывает окно**. Окно показывается **без
фокуса** (`show_soundpanel_window` без `set_focus`), поэтому клавиши ловит hook.

Цель: SoundPanel ведёт себя как обычное окно — открыли → **окно в фокусе** →
нажали A-Z (если забинжено) → воспроизвелось → окно закрылось. Клавиши ловит
**само окно** через DOM `keydown`, не hook. Hook из основного пути панели убирается
(A-Z-ветка hook удаляется или становится pass-through).

## Точные изменения

### A1. `src-tauri/src/soundpanel_window.rs` — показывать окно с фокусом

В `show_soundpanel_window()` после `window.show()?` добавить `window.set_focus()?`.
(Между `show()` и applying clickthrough — см. существующий порядок в файле.)

### A2. `src-tauri/src/hotkeys.rs` — убрать включение перехвата для панели

В `handle_sound_panel()` **удалить** строку `sp_state.set_interception_enabled(true)`
(и связанный с этим блок, если он только ради перехвата). Оставить:
`app_state.set_active_window(ActiveWindow::SoundPanel)` и
`emit_event(AppEvent::ShowSoundPanelWindow)`.

> Проверить: `interception_enabled` после этого больше нигде не включается для
> панели (поиск `set_interception_enabled(true)` в коде). Если включается ещё где-то
> в контексте панели — убрать.

### A3. `src-soundpanel/SoundPanelApp.vue` — DOM keydown → звук → закрытие

В `<script setup>`:
1. Добавить функцию `onKeydown(e: KeyboardEvent)`:
   - Если `e.key === 'Escape'` → `closeWindow()`; `return`.
   - Игнорировать, если зажат модификатор (`e.ctrlKey || e.shiftKey || e.altKey || e.metaKey`).
   - `const key = e.key.toUpperCase()`. Проверить `/^[A-Z]$/`.
   - Найти биндинг: `const b = bindings.value.find(x => x.key === key)`.
   - Если есть → `e.preventDefault()`; `await invoke('sp_play_binding', { key })`;
     затем `closeWindow()`.
   - Если нет → `showNoBinding(key)` (окно остаётся в фокусе, как сегодня).
   - Прочие клавиши — игнорировать.
2. В `onMounted`: `window.addEventListener('keydown', onKeydown)`.
3. В существующий `onUnmounted` (он уже есть в файле): `window.removeEventListener('keydown', onKeydown)`.

> НЕ вызывать `set_ignore_cursor_events` из Vue. НЕ трогать `showNoBinding`/`defineExpose`.

### A4. Новая Tauri-команда `sp_play_binding` (Rust)

В `src-tauri/src/soundpanel/bindings.rs` (там уже лежат `sp_*`-команды) добавить:
```rust
#[tauri::command]
pub fn sp_play_binding(key: String, app_handle: AppHandle) -> Result<(), String> {
    // проверить key — одна буква A-Z
    // sp_state.get_binding(char) -> если есть -> play_sound + emit(HideSoundPanelWindow)
    // если нет -> Err("No binding for key ...")
}
```
- Использовать `app_handle.state::<SoundPanelState>()`.
- `HideSoundPanelWindow` уже существует в `AppEvent` и уже скрывает окно
  (см. `setup.rs` event-loop). НЕ дублировать логику скрытия.
- **Зарегистрировать** команду в `invoke_handler` (там, где зарегистрированы
  другие `sp_*` — поиск `sp_get_bindings` покажет место регистрации).

### A5. `src-tauri/src/soundpanel/hook.rs` — убрать A-Z-перехват из панели

В `soundpanel_keyboard_proc`:
- **Удалить** ветку, которая на A-Z играла звук и гасила перехват
  (соответствующий `match vk_code { _ => { if (VK_A..=VK_Z).contains(...) ... } }`).
- `VK_ESCAPE`-ветку (закрытие панели из hook) — тоже удалить (теперь Escape ловит
  само окно, A3).
- SoundPanelNoBinding из hook — не эмитить (окно делает локально через showNoBinding).
- После удаления proc должен стать **pass-through** для этих клавиш
  (`CallNextHookEx`), т.е. когда `interception_enabled` — клавиши просто проходят
  в систему (ничего не блокируем, ничего не играем).
- `is_interception_enabled()` / `set_interception_enabled()` в `state.rs` — **НЕ
  удалять** (понадобятся для ЧАСТИ B/Intercept в следующей итерации). Просто из
  proc убрать зависящее от них поведение для A-Z.
- **Логирование:** убрать `debug!`/`info!` про A-Z/KeyDown, которые стали
  нерелевантны. Оставить логи инициализации hook.

> Hook целиком НЕ удалять (нужен для ЧАСТИ B). Только убрать A-Z/Escape-логику.

### A6. Auto-hide по blur + снятие clickthrough на активной фазе

**Blur → hide** (надёжнее на Rust, в `setup.rs` или `soundpanel_window.rs`):
- Подписаться на window-event окна `soundpanel` → `WindowEvent::Focused(false)`.
- На blur → вызвать скрытие (переиспользовать `hide_soundpanel_window`, если есть;
  иначе `window.hide()` + `set_active_window(ActiveWindow::None)`).
- Уточнить: `Focused(false)` триггерится только при уходе фокуса за пределы
  окна — внутренние DOM-элементы НЕ должны его вызывать. Проверить, что
  transparency-control внутри окна не вызывает blur.

**Clickthrough ↔ фокус:**
- В `show_soundpanel_window`: если `floating_clickthrough` включён — временно
  вызвать `window.set_ignore_cursor_events(false)` (снять clickthrough, чтобы окно
  получило фокус и ловило keydown).
- В скрытии (blur/escape/звук → HideSoundPanelWindow): восстановить
  `set_ignore_cursor_events(clickthrough_enabled)` из состояния.

> Если снять/вернуть clickthrough в show/hide слишком сложно — как минимум
> гарантировать: при открытии панели `set_ignore_cursor_events(false)`, чтобы
> keydown работал. Восстановление при hide — желательно, но не блокирующее.

## Критерии приёмки (итерации)
1. `show_soundpanel_window` вызывает `set_focus`.
2. `handle_sound_panel` **не** включает `interception_enabled`.
3. В окне: A-Z (без модификаторов) → звук (если забинжено) → окно закрылось.
   Если не забинжено → `showNoBinding`, окно осталось.
4. Escape → окно закрылось.
5. Клик мимо окна (blur) → окно скрылось.
6. Clickthrough-режим: на активной фазе clickthrough снят (окно получает фокус
   и ловит keydown).
7. Глобальный hook больше **не блокирует** A-Z (клавиши проходят в систему, когда
   перехват не активен).
8. Кнопки title-bar (transparency, clickthrough-toggle, close), drag — без регрессий.
9. `npx vue-tsc --noEmit` (в корне) → 0 errors; `cargo check` (в `src-tauri/`) → green.
