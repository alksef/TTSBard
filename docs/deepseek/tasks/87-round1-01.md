# Task 87-round1-01: Replay из «Недавних» через кеш аудио (без повторного TTS)

По плану `docs/plans/87-2026-06-30-playback-replay-from-cache.md`. Прочитай план — там причина
(id-несоответствие) и решение.

## Суть бага
В окне управления секция «Недавние» (последние 5 реплик). Клик → `replay_phrase(id)` →
`pb.replay_from_cache(id)`. Но id из recent (= PhraseEntry.id из HistoryManager, свой UUID) **не
совпадает** с id в `audio_cache` (= phrase_id из enqueue). Replay не находит аудио.

## Файлы
- `src-tauri/src/playback.rs` — audio_cache, enqueue, replay_from_cache, get_state, PlaybackStateDto.
- `src/components/...` — фронт DTO (если правится тип).
- `src-playback/PlaybackControlApp.vue` — DTO recent (проверить совместимость).

## Что сделать

### 1. `playback.rs` — тип audio_cache + RecentPhrase
Сейчас `audio_cache: VecDeque<(String, Arc<[u8]>)>` (строка 60). Заменить на структуру с text+timestamp:
```rust
#[derive(Clone)]
struct CachedPhrase {
    id: String,           // phrase_id
    text: String,
    audio: Arc<[u8]>,
    timestamp: i64,       // Utc::now().timestamp() при enqueue
}
```
`audio_cache: VecDeque<CachedPhrase>`.

### 2. `enqueue` (строка 308) — сохранять text + timestamp
```rust
pub fn enqueue(&self, id: String, text: String, audio: Vec<u8>) -> bool {
    let arc_audio: Arc<[u8]> = audio.into();
    let ts = chrono::Utc::now().timestamp();
    let mut s = self.state.write();
    s.audio_cache.push_back(CachedPhrase { id: id.clone(), text: text.clone(), audio: Arc::clone(&arc_audio), timestamp: ts });
    if s.audio_cache.len() > AUDIO_CACHE_SIZE { s.audio_cache.pop_front(); }
    // ... остальная логика очереди (QueuedPhrase { id, text, audio }) без изменений
}
```

### 3. `replay_from_cache(id)` (строка 379) — использовать CachedPhrase
```rust
pub fn replay_from_cache(&self, id: &str) {
    let replay: Option<(String, String, Arc<[u8]>)> = {
        let s = self.state.read();
        s.audio_cache.iter().find(|c| c.id == id)
            .map(|c| (c.id.clone(), c.text.clone(), Arc::clone(&c.audio)))
    };
    if let Some((id, text, audio)) = replay {
        self.enqueue(id, text, audio.to_vec());   // повторно enqueue → воспроизведение из кеша
    }
}
```
(Старый вариант искал text в current/queue — больше не нужно, text теперь в кеше.)

### 4. `get_state` (строка 416) — recent из audio_cache
```rust
recent: s.audio_cache.iter().rev().take(5)
    .map(|c| RecentPhrase { id: c.id.clone(), text: c.text.clone(), timestamp: c.timestamp })
    .collect(),
```
`PlaybackStateDto.recent: Vec<RecentPhrase>` (был `Vec<PhraseEntry>`).
```rust
#[derive(Serialize, Clone)]
pub struct RecentPhrase {
    pub id: String,
    pub text: String,
    pub timestamp: i64,
}
```

### 5. AUDIO_CACHE_SIZE
Проверь константу `AUDIO_CACHE_SIZE` — должна быть ≥5 (иначе recent < 5). Если < 5 — увеличить
до 5 (или 10 для запаса).

### 6. Фронт DTO
`src-playback/PlaybackControlApp.vue:11` — `recent: { id: string; text: string; timestamp: number }[]`.
После бэкенд-правки типы совпадают (RecentPhrase = {id,text,timestamp}). Проверить, что клик
→ `invoke('replay_phrase', { id })` работает (id теперь = phrase_id из кеша = совпадает с
replay_from_cache). Фронт править НЕ нужно, если DTO уже совпадает — только проверить.

## Важные ограничения
- `chrono::Utc::now().timestamp()` — chrono уже в deps (используется в history.rs).
- Не сломать M9 fallback (`playback.rs:253` использует replay_from_cache) — после правки он
  работает так же (найдёт в кеше).
- `Arc<[u8]>` clone дешёвый, ок.

## Критерии готовности
1. `audio_cache` хранит CachedPhrase (id+text+audio+timestamp).
2. `get_state().recent` — из audio_cache (до 5, id = phrase_id).
3. `replay_from_cache(id)` находит аудио по id из recent → воспроизведение БЕЗ повторного TTS.
4. `cargo check` + `cargo clippy --lib` + `npx vue-tsc --noEmit` — 0 ошибок, 0 warnings.
5. Runtime: 2 фразы → клик по первой в «Недавних» → мгновенное воспроизведение из кеша.
