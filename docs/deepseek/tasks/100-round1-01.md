# Task 100-round1-01 — Backend persistent cache for phrase history

Implement only the backend part of Plan 100. Do not edit Vue files.

Read `docs/stage/33-phrase-history-audio-cache.md` and `docs/deepseek/plan/100-phrase-history-audio-cache.md` first.

## Scope

- `src-tauri/src/history.rs`
- `src-tauri/src/commands/history.rs`
- `src-tauri/src/commands/tts_pipeline.rs`
- `src-tauri/src/commands/mod.rs` / `src-tauri/src/lib.rs` only as needed to register commands
- relevant backend tests

## Requirements

1. Extend `PhraseEntry` backward-compatibly with provider, voice, and cache key metadata. Language is out of scope.
2. Preserve loading of old `phrase_history.json` files.
3. Create/use a persistent app cache directory under the existing app config directory. Store ready-to-play audio with safe deterministic cache keys and atomic writes. Do not store absolute cache paths in history.
4. Ensure the cache identity includes the actual processed text plus provider, voice, and audio/effects-relevant settings; avoid collisions between different audio versions.
5. Record provider/voice/cache metadata only after synthesis/effects have produced audio suitable for playback. Keep existing history behavior compatible where possible, but allow the same text to have multiple provider/voice/audio-version records.
6. Add a backend replay operation that resolves one history entry/cache key and attempts to read the specific file at click time. Do not scan the cache directory. A missing cache must return a distinguishable error such as `CacheMiss`; it must not trigger TTS regeneration.
7. Reuse existing playback configuration and PCM/audio decode path where practical. Do not regress current `replay_phrase` memory-cache behavior.
8. Add focused tests for old JSON deserialization, cache-key separation, atomic/cache write-read roundtrip, and missing-cache error.

## Constraints

- No UI changes in this task.
- No arbitrary filesystem paths accepted from the frontend.
- Preserve unrelated worktree changes. Do not commit, reset, or clean.
- Run focused tests and `cargo check --manifest-path src-tauri/Cargo.toml`.
