# Task 103-round1-01: Клик в центр редактора снимает фокус

План: `docs/deepseek/plan/103-2026-07-08-editor-focus-loss-on-click.md` (читать).

## Симптом
Если кликнуть в центр редактора — фокус снимается (курсор не появляется / печать не идёт в
CodeMirror).

## Контекст (читать обязательно перед фиксом)
- Редактор: `src/components/editor/TtsEditor.vue` — CodeMirror 6 монтируется в `editorRef` div.
- Обёрнут в `src/components/InputPanel.vue` (`<TtsEditor v-model="text" ... />`).
- CodeMirror 6 сам управляет фокусом при клике внутри `.cm-content`. Если клик в «центре» снимает
  фокус — над/вокруг contenteditable есть слой, перехватывающий клик, ИЛИ `.cm-content` не занимает
  всю область клика.

## Задача (DeepSeek)
1. **Найти причину** (читая код, не runtime):
   - В `TtsEditor.vue`: какие слои/decorations над `.cm-content`? Есть ли оверлей с
     `pointer-events` ≠ `none`? (linter-tooltip, autocomplete, ghost-decoration, placeholder).
   - В `InputPanel.vue`: чем обёрнут `<TtsEditor>`? Есть ли на родителе `@click`/`@mousedown`/
     `data-tauri-drag-region`/`preventDefault`?
   - Размер: `EditorView.theme({ '&': { height: 'auto' } })` → если контейнер выше contenteditable,
     клик в пустой части ниже текста — мимо `.cm-content`.
2. **Применить фиксы** (комбинация, безопасная и универсальная):
   - **A (универсальный):** на корневом div редактора добавить `@click`, который фокусирует
     CodeMirror (`view.value?.focus()`). Гарантирует фокус при любом клике в области.
   - **B (CSS):** растянуть contenteditable на всю область клика, чтобы CodeMirror сам ставил
     курсор по позиции клика:
     ```css
     /* в theme или global */
     .cm-editor { height: 100%; }
     .cm-scroller { min-height: 100%; }
     .cm-content { min-height: 100%; }
     ```
     (через `EditorView.theme` — НЕ ломать существующий `height: auto` если он нужен; согласовать).
   - **C:** если найден оверлей с `pointer-events` — дать `pointer-events: none` (если он не
     интерактивный). Если `data-tauri-drag-region` на родителе редактора — убрать (редактор не
     должен быть drag-region).
3. **Не ломать:** cursor-position по клику (CodeMirror сам ставит курсор), selection, drag-region
   у title-bar (если есть — не трогать, только у редактора).

## Приоритет
- Сначала **A** (focus-on-click) — минимальный, точно чинит «фокус снимается».
- **B** — если проблема в «клик мимо contenteditable» (cursor не ставится в пустой области).
- **C** — если найден конкретный виновник.

## Верификация (DeepSeek прогоняет)
- `npx vue-tsc --noEmit` → 0 ошибок.
- Чтение кода: убедиться, что `@click="..."` на корневом div не конфликтует с CodeMirror
  (mousedown → CodeMirror ставит курсор; click → focus. Порядок: mousedown_first. Если
  `view.focus()` в @click мешает cursor-позиционированию — переместить на `@mousedown` с
  осторожностью, или использовать `view.focus()` только если клик НЕ попал в contenteditable).

## Не трогать
- keymap, extensions (только focus/click/CSS pointer-events).
- Backend.
