# Task 120-phase-1-01: Frontend UI Decomposition into Composables

План: `docs/plans/120-2026-07-11-frontend-composables-decomposition.md` (читать обязательно).

## Описание задачи
Необходимо вынести бизнес-логику работы с Tauri-командами, состоянием и подписками на события из трех Vue-компонентов во вновь создаваемые Vue-композиции (composables):
1. `src/components/WebViewPanel.vue` -> `src/composables/useWebView.ts`
2. `src/components/TwitchPanel.vue` -> `src/composables/useTwitch.ts`
3. `src/components/SoundPanelTab.vue` -> `src/composables/useSoundPanel.ts`

Все Tauri-слушатели событий (`listen`) должны отписываться автоматически при размонтировании в composable.

---

## Шаг 1. Создание `src/composables/useWebView.ts`
Выдели логику из `WebViewPanel.vue`:
- Загрузка настроек `get_webview_settings`.
- Сохранение настроек `save_webview_settings`.
- Генерация, получение и копирование токенов доступа.
- Перезапуск сервера `webview_restart_server`.
- Экспортируй реактивные переменные: `settings`, `isLoading`, `errorMessage`, `accessToken`, `upnpEnabled`.
- Слежение за жизненным циклом и подписка на события Tauri.

Примерная сигнатура:
```typescript
import { ref, onUnmounted } from 'vue'
import { invoke } from '@tauri-apps/api/core'
import { listen } from '@tauri-apps/api/event'
// ...
export function useWebView() {
    // ...
    return {
        settings,
        isLoading,
        errorMessage,
        accessToken,
        // ...методы
    }
}
```

Обнови `WebViewPanel.vue`, чтобы он вызывал `useWebView` и отображал данные.

## Шаг 2. Создание `src/composables/useTwitch.ts`
Выдели логику из `TwitchPanel.vue`:
- Локальное состояние настроек и статуса подключения (`Disconnected`, `Connecting`, `Connected`, `Error`).
- Валидация и конвертация статуса из Rust enum.
- Подписка на событие `"twitch-status-changed"` с автоматической очисткой слушателя в `onUnmounted`.
- Функции: `restartTwitch()`, `stopTwitch()`, `saveSettings()`.

Обнови `TwitchPanel.vue` на использование `useTwitch`.

---

## Шаг 3. Создание `src/composables/useSoundPanel.ts`
Выдели логику из `SoundPanelTab.vue`:
- Состояние: `bindings` (активные привязки), `sets` (список наборов), `activeSetId`, `isLoading`, `errorMessage`.
- Управление наборами (sets): `loadSets()`, `switchSet()`, `addSet()`, `removeSet()`, `renameSet()`.
- Управление привязками (bindings): `loadBindings()`, `addBinding()`, `removeBinding()`, `playBinding()`.
- Настройки внешнего вида: цвет фона, прозрачность, clickthrough, stayVisible.
- Подписка на события Tauri с обязательным cleanup (хранение функций отписки и вызов в `onUnmounted`).

Обнови `SoundPanelTab.vue` на использование `useSoundPanel`.

---

## Требования по качеству и проверке
1. Все импорты во Vue и composables должны быть типизированы. Никаких `any`!
2. В файлах components разметка и стили должны остаться полностью неизменными (мы меняем только блок `<script setup>`).
3. Запусти `node_modules/.bin/vue-tsc --noEmit` в конце, чтобы убедиться, что типы сходятся на 100% и ошибок нет.
