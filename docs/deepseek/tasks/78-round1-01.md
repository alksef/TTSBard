# Task 78-round1-01: Общий spellcheck-linter-слой (план 78)

Реализуй общий spell-linter-слой по плану `docs/plans/78-2026-06-30-spellcheck-linter-layer.md`.
Прочитай план ПОЛНОСТЬЮ — там точные точки интеграции, архитектура, код-скелеты, риски.

## Суть
**Каркас** linter-слоя для онлайн/офлайн орфографии (чтобы планы 07/08 не дублировали
linter-логику). Источник переключаемый: `online` / `offline` / `off`. **Сам движок проверки
(онлайн/офлайн) НЕ входит** — только интеграционный каркас + composable + linter-расширение
+ токенизация + тема + настройка `#[serde(default)]`.

## Что создать / изменить (точно по плану 78)

### 1. Установить зависимость
`npm i @codemirror/lint` (версия 6.x, совместимая с существующими @codemirror/* 6.x в
`package.json:13-18`). Подтверди, что версия 6.x.

### 2. НОВЫЙ `src/types/spell.ts`
```ts
export interface SpellResult {
  word: string
  correct: boolean
  suggestions: string[]
}
```

### 3. НОВЫЙ `src/composables/useSpellcheck.ts`
Composable с типом `SpellSource = 'online' | 'offline' | 'off'`. Computed `source` (из
настроек editor.spellcheck_enabled + editor.spellcheck_source), computed `enabled`.
`checkWords(words: string[])` → `invoke<SpellResult[]>` с выбором команды по источнику:
`'online' → 'check_spelling_online'`, `'offline' → 'spellcheck'`. Полный код-скелет в плане.
> **Важно:** этих бэкенд-команд пока НЕТ — `checkWords` будет падать, но linter-источник
> ловит ошибку в catch и молчит (без крашей). Это позволяет внедрить каркас сейчас.

### 4. НОВЫЙ `src/components/editor/spellLinter.ts`
Чистая функция `createSpellLinter(checkWords, enabled)`, возвращает `linter(...)` с
`{ delay: 400 }` (debounce). Токенизация рус+лат: `WORD_RE = /[a-zа-яё][a-zа-яё-]*/giu`
(флаг `u` обязателен — `\w` без него не ловит кириллицу). Маппинг `SpellResult[]` →
`Diagnostic[]` с `severity: 'warning'`, message `«word» — нет в словаре`, `actions[]` =
`suggestions.slice(0,5)` (apply через `view.dispatch({ changes })`). catch → `return []`.
Полный код-скелет в плане 78 — следуй ему.

### 5. ИЗМЕНИТЬ `src/components/editor/TtsEditor.vue`
- Импортировать `createSpellLinter` и `useSpellcheck`.
- Получить `const { checkWords, enabled } = useSpellcheck()`.
- `const spellLinter = createSpellLinter(checkWords, () => enabled.value)`.
- В `createState()` (строки ~275-300), в массив `extensions`, добавить `spellLinter`.

### 6. ИЗМЕНИТЬ Rust (с `#[serde(default)]`)
`src-tauri/src/config/settings.rs`:
- В `EditorSettings` (строки ~434-444) добавить поля:
  ```rust
  #[serde(default)]
  pub spellcheck_enabled: bool,
  #[serde(default)]
  pub spellcheck_source: SpellSource,
  ```
- Новый enum (рядом, default = Offline):
  ```rust
  #[derive(Serialize, Deserialize, Clone, Debug, PartialEq, Default)]
  #[serde(rename_all = "lowercase")]
  pub enum SpellSource {
      Online,
      #[default]
      Offline,
  }
  ```
- Геттер/сеттер по образцу `set_editor_ai_completion`/`get_editor_ai_completion`
  (строки ~1086-1115): `set_editor_spellcheck_enabled`, `get_editor_spellcheck_enabled`,
  `set_editor_spellcheck_source`, `get_editor_spellcheck_source`. Используют
  `update_field`/`cache.read()`.
- Зарегистрировать новые Tauri-команды в `src-tauri/src/lib.rs` `invoke_handler`
  (по образцу существующих set/get editor-настроек).

## Важные ограничения
- **`#[serde(default)]`** на структуре `EditorSettings` уже есть — миграция старых конфигов
  без новых полей пройдёт как default (false/Offline). Урок `playback_pause` исключён.
- Каркас должен работать **без бэкенд-движка** — при отсутствии команд `checkWords` падает
  в catch, linter молчит, без крашей.
- Тема linter-декораций (`cm-lintRange`, `cm-diagnosticText`) — только CSS-vars
  (`--color-danger`, `--color-bg-elevated`).
