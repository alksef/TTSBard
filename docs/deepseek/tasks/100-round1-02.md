# Task 100-round1-02 — Phrase history quick replay UI

Implement the frontend part of Plan 100 after reading the stage and plan files. Backend command `replay_phrase_from_cache` is registered and accepts `{ phraseId }`; it returns `CacheMiss` when the specific cached audio is unavailable. Do not change backend files in this task.

## Scope

- `src/types/phrase.ts`
- `src/composables/usePhraseHistory.ts`
- `src/components/PhraseHistoryList.vue`
- parent component wiring only if required to surface an existing toast/error mechanism

## Requirements

1. Extend the frontend `PhraseEntry` type with optional/backward-compatible `provider`, `voice`, and `cache_key` fields.
2. Add a quick-play action to each phrase row using the existing icon/button conventions. It must have both `title` and `aria-label`, stop row selection, and not add/replace text.
3. Invoke `replay_phrase_from_cache` by phrase history id. Do not preflight the cache and do not list/scan the cache directory.
4. On `CacheMiss`, show a small local error notice/pill in the history panel. Do not regenerate TTS automatically. Clear the notice on a successful replay or a later successful operation; avoid one global error making every row look broken.
5. Display provider and voice as small secondary metadata at the bottom of the phrase card. Omit empty values and do not display language or cache availability state.
6. Preserve current append/replace/delete/clear behavior, compact layout, dark/light theme variables, and narrow-width behavior. Follow the existing `.phrase-meta`, `.phrase-action-btn`, and error styles in `PhraseHistoryList.vue` as source of truth.
7. Add loading/duplicate-click protection for replay if needed, with stale async result handling.

## Acceptance

- `npx vue-tsc --noEmit` passes.
- A row can be played without selecting its text.
- Missing cache produces only a concise local notice and no TTS request.
- Provider/voice metadata is visible when present and absent when empty.
