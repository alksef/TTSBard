# Task 113 / Round 1 / Iteration 02 — корректно исключить controls из drag region

## Review finding

В предыдущей итерации на `.set-selector` и `.buttons` добавлен атрибут `data-tauri-drag-region="false"`. Для Tauri небезопасно полагаться на строковое значение `false`: наличие атрибута может всё равно рассматриваться как drag marker. В компоненте уже есть CSS `-webkit-app-region: no-drag` для этих областей.

## Что сделать

1. Удали `data-tauri-drag-region="false"` с `.set-selector` и `.buttons` в `src-soundpanel/SoundPanelApp.vue`.
2. Оставь `data-tauri-drag-region` только на `.title-bar`; существующие CSS-правила `.set-selector` и `.buttons` с `-webkit-app-region: no-drag` должны продолжить исключать интерактивные элементы из перетаскивания.
3. Не меняй backend-логику focus/click-through из предыдущей итерации.
