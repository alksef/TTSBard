# Task 29 Round 1 Iteration 2 — DSP UI block and preview integration

## Goal

Complete the user-facing DSP feature: one parent DSP block in the existing audio settings UI, with independent EQ, compressor, and limiter controls, and make preview use the same DSP settings as normal TTS playback.

## Bounded files

- `src/components/AudioPanel.vue` and/or the existing TTS audio-effects component if that is the actual mounted settings surface
- `src/composables/useAppSettings.ts` if needed to expose the DTO
- `src/types/settings.ts` if types need adjustment
- `src-tauri/src/commands/playback.rs` for preview DSP wiring
- `src-tauri/src/audio/device.rs` only to revert unrelated formatting-only changes from the previous iteration

Do not change DSP algorithms or persisted field names in this iteration.

## UI requirements

1. Render one parent block titled `DSP-постобработка`.
2. Inside it render three independently collapsible sections:
   - `EQ`: enable toggle, low-cut, three parametric bands, high-shelf;
   - `Compressor`: enable toggle, threshold, ratio, attack, release, knee, makeup;
   - `Limiter`: enable toggle, ceiling and release. Clearly show that limiter is a safety ceiling.
3. Use the existing `AudioPanel.vue` settings rows, toggles, sliders, buttons, spacing, radii, colors, hover/active/disabled states, and theme behavior as source of truth. Do not invent a second visual language.
4. Use one atomic `save_dsp_settings` call. Load through `get_dsp_settings` or the existing settings composable. Preserve unsaved draft/cancel behavior used by the surrounding panel.
5. Provide `title` and `aria-label` for icon-only collapse/reset controls.
6. Verify narrow width, light/dark themes, and no horizontal overflow.

## Preview requirements

1. Preview's `has_effects` must include whether any DSP block is enabled.
2. Preview must construct `DspConfig` from persisted settings and pass it to `apply_effects`, just like the TTS pipeline.
3. Keep playback volume semantics unchanged.

## Cleanup

Revert only the unrelated formatting-only changes in `src-tauri/src/audio/device.rs`; do not remove user-authored functional changes.

## Acceptance criteria

- `cargo check --manifest-path src-tauri/Cargo.toml` passes.
- `npx vue-tsc --noEmit` passes.
- Preview and normal TTS use identical DSP settings and order.
- Existing audio-effects settings continue to work.
- One parent DSP UI block contains the three independent subsections.
- No horizontal overflow at narrow width and no new theme regressions.
