# Task 106-round1-01: Extract DSP settings container

Прочитай stage `docs/stage/35-audio-panel-subpanels.md` и review `docs/deepseek/reviews/105-round1-review.md`.

Вынеси из `src/components/audio/AudioEffectsTab.vue` DSP-часть в новый `src/components/audio/DspSettings.vue`.

Перенеси в `DspSettings.vue`:

- `createDefaultDsp`, `createNaturalDsp`, `createClearDsp`, preset detection/setter;
- `draftDsp`, `savedDsp`, `dspDirty`, save/cancel status и collapse state;
- watcher `useDspSettings`, `saveDsp`, `cancelDsp`, `markDspDirty`, `toggleDspCollapse`;
- DSP template: presets, EQ, Compressor, Limiter и DSP save section;
- DSP-specific styles.

Не меняй поведение `preview_audio_file`: preview должен продолжить использовать актуальные DSP settings; передай их через props/events либо вынеси preview в composable так, чтобы draft DSP не потерялся.

Не выноси пока EQ/Compressor/Limiter в отдельные файлы — это следующий task. Не меняй backend. Используй небольшие Edit/apply-patch операции, не shell-скрипты для записи больших файлов.

Проверки: `npx vue-tsc --noEmit`, `git diff --check`. Не форматируй Rust и не изменяй чужие task-файлы.
