# Task 106-round1-01: Окно управления не показывает воспроизведение — resizeToFit ломает onMounted

План: `docs/deepseek/plan/106-2026-07-09-playback-window-resizetofit-breaks-onmounted.md` (читать обязательно).

## Симптом (от пользователя)
«при отправке на ттс в окне управления не отображается управление. раньше работало»
В консоли DevTools окна `playback-control`:
```
window.set_size not allowed. Permissions: core:window:allow-set-size
playback-B3XZzOcn.js:1 Uncaught (in promise) window.set_size not allowed.
```

## Корневая причина (найдена Claude, подтверждена пользователем)
`PlaybackControlApp.vue` → `onMounted` (async):
- строка 101: `await fetchState()` — отрабатывает ✅
- строка 103: `await resizeToFit()` → `win.setSize(LogicalSize(...))` → **бросает rejection**
  (нет capability `core:window:allow-set-size`).
- onMounted прерывается → строки 105-123 (регистрация listen-слушателей `playback-started`,
  `playback-finished`, `queue-changed`, `refresh-state`, и т.д.) **НЕ выполняются**.
- Окно не получает события воспроизведения → не зовёт `fetchState()` по событию → всегда Idle /
  «Нет активной фразы». Звук при этом играет (backend корректен).

## Задача DeepSeek — ДВА слоя фикса

### Слой 1 — capabilities (ГЛАВНЫЙ)
Файл `src-tauri/capabilities/default.json`, массив `permissions`. Добавить:
- `"core:window:allow-set-size"` — **обязательно** (не хватает именно его).
- Также проверить и при отсутствии добавить (нужны для `resizeToFit`, иначе упадёт следом):
  `"core:window:allow-outer-size"`, `"core:window:allow-scale-factor"`.

Сверить точные имена permission по `src-tauri/gen/schemas/desktop-schema.json`
(поиск `allow-set-size` / `set-size`). Если schema ещё не сгенерирована — `cargo build`
сгенерирует. НЕ выдумывать имена.

### Слой 2 — defensive (чтобы regression такого рода больше не молча ломал окно)
Файл `src-playback/PlaybackControlApp.vue`:
1. Функцию `resizeToFit` (строки ~30-42) обернуть целиком в `try { ... } catch (e)
   { console.warn('resizeToFit failed', e) }`. Sizing — best-effort, не должен рвать onMounted.
2. **Порядок в onMounted** (строки ~92-124): переставить так, чтобы **сначала** регистрировались
   слушатели (блок `unlisteners = [...]`), а `resizeToFit()` звался **после** (и в try/catch).
   Тогда даже если resize упадёт в будущем — слушатели уже повешаны, окно обновляется.

   Текущий порядок:
   ```
   onMounted: fetchState() → resizeToFit() → unlisteners=[...]
   ```
   Должно стать:
   ```
   onMounted: fetchState() → unlisteners=[...] → resizeToFit() (в try/catch)
   ```
   (fetchState оставить первым — он нужен для начального состояния; но слушатели — раньше
   resize, это главное.)
