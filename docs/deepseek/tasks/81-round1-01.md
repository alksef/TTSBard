# Task 81-round1-01: Офлайн-орфография — spellbook backend (план 81)

Реализуй бэкенд офлайн-орфографии по плану `docs/plans/81-2026-06-30-offline-spellcheck.md`.
Прочитай план ПОЛНОСТЬЮ — там API spellbook (подтверждён spike 80), архитектура, риски.

## Контекст
Spike 80 ПРОЙДЕН: `spellbook v0.4.2` работает на русском. Фронт-слой уже есть (план 78):
`useSpellcheck.ts` вызывает `invoke('spellcheck', { words })` при `source='offline'`. Этот
план делает так, чтобы команда `spellcheck` **существовала и работала** на бэкенде.

## Подтверждённый API spellbook (из spike 80, версия 0.4.2)
```rust
let aff = std::fs::read_to_string("...ru.aff")?;
let dic = std::fs::read_to_string("...ru.dic")?;
let dict = spellbook::Dictionary::new(&aff, &dic)?;  // Result<_, ParseDictionaryError>
dict.check("привет");    // bool
let mut sugg = Vec::new();
dict.suggest("првиет", &mut sugg);   // заполняет Vec<String>
```
Словари: `ru.aff` (~70KB) + `ru.dic` (~3.4MB), UTF-8, источник wooorm/dictionaries.

## Что сделать (точно по плану 81)

### 1. Зависимость
`cargo add spellbook` в `src-tauri/Cargo.toml`. Зафиксируй версию 0.4.x.

### 2. Словари в бандле
- Создай `src-tauri/resources/dict/ru.aff` и `src-tauri/resources/dict/ru.dic`.
  **Если файлов нет в репо** — СКАЧАЙ их: ru.aff + ru.dic из публичного источника
  (wooorm/dictionaries на GitHub, UTF-8 Hunspell). Если скачивание невозможно — СОЗДАЙ
  минимальные тестовые ru.aff/ru.dic (5-10 слов) и ОТМЕТЬ это в отчёте как временное.
- `src-tauri/tauri.conf.json` → в секцию `bundle` добавить:
  ```json
  "resources": ["resources/dict/*"]
  ```

### 3. Менеджер `src-tauri/src/spellcheck.rs` (НОВЫЙ, по образцу history.rs)
- `pub struct SpellcheckManager { dict: RwLock<Option<spellbook::Dictionary>>, cache: RwLock<HashMap<String, SpellResult>> }`
- `pub struct SpellResult { word: String, correct: bool, suggestions: Vec<String> }` (serde::Serialize).
- `pub fn new(aff_path: PathBuf, dic_path: PathBuf) -> Self` — грузит словарь ОДИН раз.
  **Ошибка загрузки НЕ должна крашить приложение:** dict остаётся `None`, в stderr лог,
  spellcheck возвращает всем `correct=true` (фича молча выключена).
- `pub fn check_words(&self, words: &[String]) -> Vec<SpellResult>` — с кэшем: если слово в
  кэше — вернуть из кэша; иначе check+suggest, сохранить в кэш.
- **Borrow-checker внимание:** `dict` (read-lock) и `cache` (write-lock) — разные RwLock.
  Не держи read-lock словаря во время write-lock кэша. Реализуй: проверить слово под
  read-lock словаря, собрать результат, отпустить read-lock, потом писать в кэш под
  write-lock. (parking_lot::RwLock, как в history.rs.)
- Словарь **read-only**, не персистим — `spawn_save` НЕ нужен (в отличие от HistoryManager).

### 4. Команда `src-tauri/src/commands/spellcheck.rs` (НОВЫЙ)
```rust
use crate::spellcheck::{SpellcheckManager, SpellResult};
use std::sync::Arc;
use tauri::State;

pub struct SpellcheckState(pub Arc<SpellcheckManager>);

#[tauri::command]
pub fn spellcheck(words: Vec<String>, state: State<'_, SpellcheckState>) -> Result<Vec<SpellResult>, String> {
    Ok(state.0.check_words(&words))
}
```

### 5. Регистрация
- `src-tauri/src/commands/mod.rs` — `pub mod spellcheck;`
- `src-tauri/src/lib.rs` — импорт `commands::spellcheck::spellcheck`, добавить в `invoke_handler`.
- `src-tauri/src/state.rs` — поле `spellcheck_manager: Arc<Mutex<Option<Arc<SpellcheckManager>>>>`
  (по образцу `history_manager`, строки ~136-138).
- `src-tauri/src/setup.rs` — инициализация: получить resource_dir через Tauri API
  (`app.path().resource_dir()` или аналог), собрать пути `<resource>/resources/dict/ru.{aff,dic}`,
  создать `SpellcheckManager::new(...)`, положить в AppState. **Проверь, что пути существуют**
  (лог warning если нет, но не краш).

## Важные ограничения
- **Не крашить приложение**, если словарь не загрузился / файлы отсутствуют — фича молча
  выключена (dict=None → все слова correct).
- `spellbook` версия 0.4.x (spike подтвердил 0.4.2). Зафиксируй в Cargo.toml.
- Кэш в памяти — не персистить.
- Фронт НЕ трогать (план 78 уже вызывает `spellcheck`).
