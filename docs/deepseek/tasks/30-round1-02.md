# Stage 30, round 1, task 02

## Цель

Исправить тестирование fallback-контракта DeepFilterNet из замечаний `docs/deepseek/reviews/30-round1-01.md`.

## Файлы

- `src-tauri/src/audio/effects.rs`

## Требования

1. Не менять уже реализованный вариант A: успешный enhancement возвращает samples и фактический sample rate модели; обратного ресемплинга нет.
2. Убрать или переписать тесты, которые называются fallback, но фактически не вызывают ошибку.
3. Добавить честный unit-тест ошибочной ветки, подтверждающий, что при `fail_on_enhance_error = false` сохраняются исходные samples и исходный sample rate, а при `true` возвращается ошибка. Допустим небольшой приватный helper или тестовая инъекция функции enhancement; не менять публичный production API и не маскировать ошибку успешным early return.
4. Тесты должны быть детерминированными и не зависеть от случайного падения модели. Если helper принимает функцию/замыкание, production path должен использовать реальный `apply_enhance`, а тест — детерминированный `Err`.
5. Сохранить существующие проверки пяти sample rates, длительности, mono/stereo, короткого сигнала и disabled enhancement.

## Проверка

- `cargo check --manifest-path src-tauri/Cargo.toml`
- `cargo test --manifest-path src-tauri/Cargo.toml audio::effects::tests`

Не изменять другие файлы и не делать commit.
