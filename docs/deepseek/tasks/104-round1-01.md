# Task 104-round1-01: Ctrl+Z не работает после выбора из истории (round-trip фикс)

План: `docs/deepseek/plan/104-2026-07-08-history-select-append-and-undo.md`, **Часть 2 (bug)**.
Часть 1 (UX append/replace) — отдельный раунд, НЕ делать здесь.

## Корень
`src/components/editor/TtsEditor.vue`:
- `watch(() => props.modelValue, ...)` (~строка 330) dispatch-ит замену всего документа с
  флагом `isExternalUpdate.value = true`.
- `EditorView.updateListener` (~строка 303) emit-ит `update:modelValue` только если
  `!isExternalUpdate.value`.

Проблема: `isExternalUpdate` — **времянка** (true только в момент dispatch). При undo CodeMirror
откатывает замену → `docChanged` → но `isExternalUpdate` уже false → listener emit-ит откаченное
значение → `text.value` откатывается → watch снова срабатывает → повторно dispatch-ит → **undo
немедленно перебивается round-trip-ом**.

## Фикс — annotation-based (правильный)
Заменить флаг-времянку на **`Annotation`** из `@codemirror/state`. External-update помечается
annotation; listener игнорирует транзакции с этой annotation.

Изменить `src/components/editor/TtsEditor.vue`:

1. Импорт: `import { EditorState, Annotation } from '@codemirror/state'` (проверить, что
   `Annotation` есть в текущем импорте `@codemirror/state` — если нет, добавить).

2. Определить annotation (на уровне модуля/setup, ОДИН раз — не внутри watch):
   ```ts
   const ExternalUpdate = Annotation.define<boolean>()
   ```

3. Убрать `const isExternalUpdate = ref(false)` (и все его использования).

4. В `watch(() => props.modelValue, ...)`:
   ```ts
   watch(() => props.modelValue, (newVal) => {
     const v = view.value
     if (!v) return
     const currentDoc = v.state.doc.toString()
     if (newVal !== currentDoc) {
       v.dispatch({
         changes: { from: 0, to: currentDoc.length, insert: newVal },
         annotations: ExternalUpdate.of(true),
       })
     }
   })
   ```

5. В `EditorView.updateListener.of(...)`:
   ```ts
   EditorView.updateListener.of((update) => {
     if (!update.docChanged) return
     // игнорировать транзакции, инициированные внешним обновлением modelValue (annotation)
     const isExternal = update.transactions.some(tr => tr.annotation(ExternalUpdate) !== undefined)
     if (isExternal) return
     emit('update:modelValue', update.state.doc.toString())
   })
   ```

## Почему это чинит undo
- External update (история/вкладка/quick-editor) → dispatch с annotation → listener видит
  annotation → **не emit-ит** (нет round-trip).
- При undo — это user-инициированный undo, БЕЗ annotation → listener emit-ит откаченное значение
  → `text.value` откатывается → но watch НЕ сработает повторно (значение уже = тому, что откатил
  undo) → undo остаётся. ✅

## `addToHistory`?
НЕ менять — оставить undoable (по умолчанию `true`). Чтобы undo возвращал к «до выбора из истории».
Annotation уже чинит round-trip, `addToHistory` трогать не нужно.
