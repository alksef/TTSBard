# Task 122-round1-01: Устранение лишнего обратного ресэмплинга после DeepFilterNet

Вы должны оптимизировать функцию `apply_enhance` и ее вызов в `apply_effects` в файле `src-tauri/src/audio/effects.rs`, чтобы не делать обратный ресэмпл из 48 kHz в 24 kHz (частота модели на выходе должна остаться 48 kHz).

## Шаг 1: Сделайте `sample_rate` изменяемым в `apply_effects`
В `src-tauri/src/audio/effects.rs` найдите функцию `apply_effects`:
1. Измените `let sample_rate = pcm_data.sample_rate;` на `let mut sample_rate = pcm_data.sample_rate;`.
2. В блоке успешного применения очистки обновляйте `sample_rate = 48000;` (так как DeepFilterNet на выходе всегда дает 48000 Hz):
   ```diff
        match apply_enhance(&samples, sample_rate, channels, effects.enhance_atten_db) {
            Ok(enhanced) => {
                samples = enhanced;
+               sample_rate = 48000; // DeepFilterNet model output is always 48 kHz
                tracing::debug!(
                    atten_db = effects.enhance_atten_db,
                    channels,
                    "Applied DeepFilterNet enhancement"
                );
            }
   ```

## Шаг 2: Уберите обратный ресэмплинг в `apply_enhance`
В функции `apply_enhance`:
1. Удалите блок обратного ресэмплинга (ориентировочно строки 415-421):
   ```rust

            .map_err(|e| format!("Resample to {} Hz failed: {:?}", sample_rate, e))?

        enhanced
    };
   ```
2. Обновите секцию Re-interleave (строки 423-431), чтобы она считывала данные напрямую из `enhanced` (вместо удаленной переменной `output`):
   ```rust

        for ch in 0..channels {

        }
    }
   ```
