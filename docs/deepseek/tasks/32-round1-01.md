# Task 32-round1-01: чистый boundary processor для PCM-фраз

## Цель

Добавить небольшой самостоятельный backend-модуль для безопасной обработки границ одной `AudioPcm`-фразы. На этом шаге не менять Vue UI, настройки, Tauri-команды и логику очереди playback.

## Контекст

Текущий PCM-контракт находится в `src-tauri/src/audio/effects.rs` (`AudioPcm`). Основная обработка выполняется пофразно. Нужно подготовить чистые функции, которые следующая задача сможет подключить к pipeline и playback.

## Разрешённые файлы

- `src-tauri/src/audio/boundary.rs` — новый модуль;
- `src-tauri/src/audio/mod.rs` — регистрация модуля и публичные re-export при необходимости;
- `src-tauri/src/audio/effects.rs` — только если требуется аккуратно вызвать общую boundary-функцию для локального теста, без изменения публичного поведения;
- `src-tauri/Cargo.toml` — только если новая зависимость действительно необходима; предпочтительно без зависимостей.

Не изменять `src/components`, `src/types`, `src-tauri/src/config`, `src-tauri/src/commands` и `src-tauri/src/playback.rs` в этом task.

## Требуемое поведение

1. Работать с interleaved `f32` PCM и метаданными `sample_rate`/`channels`.
2. Проверять входные данные через существующие `AudioPcm` invariants или принимать `&AudioPcm` и возвращать новый валидный `AudioPcm`.
3. Обрабатывать DC offset осторожно и не вычитать его безусловно из нормальной речи.
4. Применять короткий fade-in только если начало фразы содержит заметный скачок.
5. Применять короткий fade-out только если конец содержит заметный скачок.
6. Не менять sample rate, channels и обычную длительность.
7. Для пустых, однофреймовых и очень коротких фраз не паниковать.
8. Добавить отдельную pure-функцию для equal-power crossfade двух PCM-буферов с явными проверками совместимости. Crossfade должен быть ограничен доступной длиной и не должен создавать NaN/Inf.
9. Не выполнять crossfade автоматически между фразами на этом шаге: только подготовить и протестировать функцию.

## Тесты

Добавить unit-тесты минимум для:

- normal phrase без discontinuity практически не меняется;
- ненулевой первый sample получает мягкий старт;
- резкий пик на конце получает мягкий fade-out;
- фраза короче выбранной длины не вызывает panic;
- пустая фраза обрабатывается безопасно;
- crossfade сохраняет валидность, каналы и ожидаемую длину;
- несовместимые sample rate/channels отклоняются;
- результат не содержит non-finite samples.

## Ограничения реализации

- Не использовать subprocess, Python или нейросетевые модели.
- Не добавлять необязательный автоматический fade на каждую фразу.
- Не трогать DeepFilterNet, DSP-пресеты и UI.
- Не считать прохождение тестов заменой проверки diff.
