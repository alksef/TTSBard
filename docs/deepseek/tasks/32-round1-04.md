# Task 32-round1-04: boundary UI и DSP-пресеты

## Цель

Завершить пользовательскую часть Stage 32: отдельная плашка boundary cleanup с независимой галкой и DSP-пресеты `Natural`, `Clear`, `Custom`. Overlap-crossfade и playback queue не изменять.

## Предыдущий результат

Backend уже содержит `AudioEffectsSettings.boundary_cleanup_enabled` (default `true`), DTO и применение boundary cleanup после per-phrase эффектов. `save_audio_effects` ещё не принимает новый флаг.

## Разрешённые файлы

- `src-tauri/src/commands/playback.rs`;
- `src-tauri/src/config/settings.rs` — удалить неиспользуемый setter из предыдущего task, если вместо него используется атомарная save-команда;
- `src/types/settings.ts`;
- `src/components/AudioPanel.vue`;
- `src-tauri/src/lib.rs` — только если меняется список invoke-команд;
- при необходимости `src-tauri/src/config/dto.rs`.

Не изменять `src-tauri/src/playback.rs`, `src-tauri/src/audio/boundary.rs`, DeepFilterNet pipeline и `src-tauri/src/audio/device.rs`.

## Boundary UI

В начале вкладки эффектов, до «Преобразование голоса»/DeepFilterNet, добавить отдельную плашку **«Обработка границ фраз»**.

Внутри — независимый toggle/checkbox **«Исправлять границы и склеивать фразы»**:

- default `true`;
- значение хранится в `draftEffects.boundary_cleanup_enabled`;
- изменение помечает обычные audio effects как dirty;
- не меняет состояние DeepFilterNet и DSP;
- сохраняется через существующую атомарную `save_audio_effects`;
- при cancel возвращается сохранённое значение.

Не обещать overlap-crossfade в UI. Допустимое описание: исправление резких начал/концов фраз.

## DSP presets

В начале DSP-плашки, сразу под её заголовком, добавить компактный segmented control:

```text
Режим: [Natural] [Clear] [Custom]
```

Поведение:

- `Natural` применяет безопасный минимальный профиль и оставляет boundary toggle без изменений;
- `Clear` применяет мягкий профиль для разборчивости: low-cut, небольшой presence boost, лёгкий compressor, limiter;
- любое ручное изменение DSP-параметра переводит режим в `Custom`;
- переключение на Natural/Clear заменяет только DSP-настройки;
- DeepFilterNet и boundary cleanup не входят в пресеты и не изменяются при их переключении;
- добавить действия сброса к Natural/Clear только если они вписываются в текущий компактный UI без дублирования управления.

Использовать существующие `createDefaultDsp`, `markDspDirty`, `saveDsp`, `cancelDsp` и стили кнопок/setting-row как source of truth. Не создавать новую визуальную систему.

Профиль Natural должен включать только safety limiter или существующий безопасный минимум. Профиль Clear должен быть умеренным и не использовать DeepFilterNet.

## Backend save command

Расширить `save_audio_effects` новым аргументом `boundary_cleanup_enabled: bool`, записать его в `settings.audio_effects` и сохранить атомарно. Обновить только существующую команду и её frontend invoke-вызов.

## Состояния и доступность

- начальная загрузка настроек не должна затирать dirty draft;
- cancel должен восстанавливать и boundary flag, и DSP draft;
- preset buttons должны иметь active/hover/disabled состояния;
- controls должны работать в узкой ширине панели без горизонтального overflow;
- тексты и aria-label/title должны быть понятными.
