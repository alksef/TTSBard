# Task 113 / Round 1 / Iteration 01 — drag-зона и focus-поведение SoundPanel

## Контекст

Floating окно `soundpanel` скрывается в `src-tauri/src/lib.rs` на каждом `WindowEvent::Focused(false)`. При включённом click-through окно намеренно пропускает клики, теряет фокус и сразу исчезает. В `src-soundpanel/SoundPanelApp.vue` у `.title-bar` уже есть CSS `-webkit-app-region: drag`, но drag-зону нужно явно обозначить для Tauri, сохранив интерактивность кнопок.

## Что сделать

1. В обработчике `Focused(false)` для `soundpanel` проверь состояние `SoundPanelState::is_floating_clickthrough_enabled()`. Не вызывай `hide_soundpanel_window`, если click-through включён; при выключенном режиме оставь существующее автоскрытие.
2. В `SoundPanelApp.vue` добавь `data-tauri-drag-region` на верхнюю drag-область (`.title-bar` или отдельный свободный header-элемент). Для `.buttons`, `.set-selector` и кнопок явно задай `data-tauri-drag-region="false"` либо проектный эквивалент no-drag, чтобы клики по ним не начинали перемещение. Не меняй обработчики кнопок.
3. Проверь, что drag-region не применяется к content и не мешает click-through классу.
4. Не меняй сохранение позиции и настройки click-through.
