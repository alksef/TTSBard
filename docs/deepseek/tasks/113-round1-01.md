# Task 113-round1-01: WebView — blocking `recv_timeout` → tokio async channel

План: `docs/deepseek/plan/113-2026-07-11-webview-tokio-channel.md` (читать обязательно).

## Контекст

`run_webview_server` — async-функция, выполняется в Tokio runtime.
Принимает `webview_rx: std::sync::mpsc::Receiver<AppEvent>` и вызывает
`webview_rx.recv_timeout(Duration::from_secs(1/2))` — синхронно блокирующий вызов.
Это блокирует Tokio worker на 1–2 секунды при каждой итерации цикла.

## Объём изменений

Нужно изменить **тип канала** во всей цепочке. Прочитай все затронутые файлы
перед правкой:

### 1. `src-tauri/src/state.rs`
Найди поле `webview_event_sender` (тип `Mutex<Option<std::sync::mpsc::Sender<AppEvent>>>`).
Измени на:
```rust
pub webview_event_sender: Mutex<Option<tokio::sync::mpsc::UnboundedSender<AppEvent>>>,
```

### 2. `src-tauri/src/setup.rs`
Найди создание канала для webview (что-то вроде `std::sync::mpsc::channel()`).
Измени на:
```rust
let (webview_tx, webview_rx) = tokio::sync::mpsc::unbounded_channel::<AppEvent>();
```

### 3. `src-tauri/src/servers/webview.rs`
Измени сигнатуру функции:
```rust
// БЫЛО:
pub async fn run_webview_server(
    ...
    webview_rx: std::sync::mpsc::Receiver<AppEvent>,
)

// СТАЛО:
pub async fn run_webview_server(
    ...
    mut webview_rx: tokio::sync::mpsc::UnboundedReceiver<AppEvent>,
)
```

Замени ОБА вызова `recv_timeout`:

```rust
// БЫЛО (строки 102 и 170):
match webview_rx.recv_timeout(std::time::Duration::from_secs(1)) {
    Ok(event) => { ... }
    Err(std::sync::mpsc::RecvTimeoutError::Timeout) => { continue }
    Err(std::sync::mpsc::RecvTimeoutError::Disconnected) => { return }
}

// СТАЛО:
match tokio::time::timeout(
    tokio::time::Duration::from_secs(1),
    webview_rx.recv(),
).await {
    Ok(Some(event)) => { /* та же обработка event */ }
    Ok(None) => { /* канал закрыт = Disconnected */ return; }
    Err(_) => { /* таймаут = Timeout */ continue; }
}
```

### 4. Другие файлы с `.send()` на webview sender

В `commands/mod.rs` и других местах, где делается `tx.send(AppEvent::...)` через
`webview_event_sender` — `UnboundedSender::send()` **не async**, возвращает `Result`
вместо `()`. Обнови обработку ошибок: `let _ = tx.send(event)` → ок (игнорируем, как было).

## Проверь cascade

```
grep -rn "webview_event_sender" src-tauri/src/
```
И для каждого убедись, что `.send()` корректно обновлён.

## Fallback (только если основное не компилируется)
Если cascade не сходится — реализуй spawn_blocking:
```rust
// В webview.rs, вместо recv_timeout:
let event = tokio::task::spawn_blocking({
    let rx = &webview_rx;  // нужно Arc<Mutex<Receiver>> если spawn_blocking
    move || rx.recv_timeout(Duration::from_secs(1))
}).await??;
```
Но основной вариант (UnboundedSender) — предпочтительный.
