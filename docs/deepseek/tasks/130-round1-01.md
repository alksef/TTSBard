# Task 130 round1-01: restore editor focus after tray reveal

Implement the focused fix described below. Do not modify unrelated behavior or
files.

## Bug

The main Tauri window is hidden with `window.hide()` and later shown from the
tray or the global main-window hotkey. The native window receives focus, but the
CodeMirror editor does not. `TtsEditor.vue` focuses itself only from `onMounted`,
and hiding the native window does not unmount the Vue component. The user must
click the editor before typing.

## Required implementation

1. In `src/components/InputPanel.vue`, expose a small method that calls the
   existing `editorRef.value?.focus()`.
2. In `src/App.vue`, keep a ref to `InputPanel` and subscribe to the main
   window's `getCurrentWindow().onFocusChanged` event. Follow the installed Tauri
   API signature and destructure the event payload (`({ payload: focused })`).
3. When the window becomes focused and `currentPanel.value === 'input'`, wait
   for the next Vue tick and call the exposed InputPanel focus method. This must
   work for both tray-click and global-hotkey reveal paths.
4. Store and invoke the returned unlisten function during unmount. Handle the
   async subscription setup in the style already used in the repository.
5. Do not change the Rust tray/hotkey implementation, CodeMirror behavior,
   hotkey mappings, or focus behavior of other panels.

## Acceptance criteria

- After hide-to-tray followed by tray click or main-window global hotkey, text
  can be typed immediately without a mouse click.
- A blur event must not call the editor focus method.
- If another panel is active, the hidden editor is not explicitly focused.
- No listener leak is introduced.
- `npx vue-tsc --noEmit` passes.
