# Task 29 Round 2 Iteration 2 — Fix unified Effects/DSP scroll layout

## Problem

After merging the tabs, `src/components/AudioPanel.vue` creates a nested scroll layout: the app's `.main-content` already scrolls, while `.unified-tab` adds `max-height: calc(100vh - 280px)` and `overflow-y: auto`. This causes a white strip at the bottom and confusing scrolling behavior around the sticky `Проверка эффектов` panel.

## Goal

Make the unified Effects/DSP tab use one predictable scroll container and remove the bottom white strip, while keeping the preview panel sticky above the settings.

## Bounded file

- `src/components/AudioPanel.vue` only

## Requirements

1. Remove the hardcoded viewport `max-height` and avoid nested scrolling between `.main-content` and `.unified-tab`.
2. Choose a layout compatible with the existing App shell (`.main-content` is already `overflow-y: auto`, `.app-container` is `height: 100vh`, `.app-content-wrapper` is flex/overflow hidden).
3. The tab must occupy available content height without leaving a white bottom strip. Do not use a hardcoded height that assumes a particular titlebar/padding size.
4. Keep `Проверка эффектов` sticky while the settings below it scroll. It must not cover the tab navigation unexpectedly; account for the existing tab navigation/padding if the outer `.main-content` remains the scroll owner.
5. Preserve narrow-width behavior, dark/light theme colors, and `overflow-x: hidden` only where it is needed to prevent horizontal overflow.
6. Do not change preview logic, DSP settings, or backend code.

## Acceptance criteria

- No white strip at the bottom of the Effects/DSP tab.
- Exactly one scrollbar is responsible for scrolling the settings in the normal app layout.
- Sticky preview remains visible while EQ/compressor/limiter settings scroll.
- `npx vue-tsc --noEmit` passes.
