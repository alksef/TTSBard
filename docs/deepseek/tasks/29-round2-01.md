# Task 29 Round 2 Iteration 1 — Unified Effects and DSP preview UI

## Goal

Refactor the audio settings UI so voice effects and DSP are one unified tab named `Эффекты и DSP`, with one shared preview panel fixed at the top of the scrollable content and collapsible settings sections below it.

## Bounded files

- `src/components/AudioPanel.vue`
- `src/composables/useAppSettings.ts` only if needed
- `src-tauri/src/commands/playback.rs` only if preview payload needs a minimal adjustment

Do not change DSP algorithms, settings field names, persisted format, or unrelated audio files.

## Required UI behavior

1. Replace separate `Эффекты` and `DSP` tabs with one tab titled `Эффекты и DSP`.
2. Put one shared preview panel at the top of that tab. It must use `position: sticky` within the tab's scroll container, not fixed to the entire window.
3. Preview controls:
   - `Оригинал`: no voice effects and no DSP;
   - `Все эффекты`: current draft voice-effects and current draft DSP settings, including unsaved changes;
   - `Стоп` while playback is active;
   - clear status/error text and unsaved-changes indication.
4. The preview panel must not require saving settings first. Existing save/cancel actions remain separate.
5. Below the preview, keep two collapsible parent sections:
   - existing voice effects settings;
   - existing `DSP-постобработка` settings with EQ, Compressor, Limiter subsections.
6. Preserve existing draft/save/cancel behavior for both settings groups. A preview must not mutate saved settings or mark them saved.
7. Existing visual source of truth is `AudioPanel.vue`: reuse its current tabs, setting rows, toggles, buttons, CSS variables, light/dark theme, and responsive patterns. Do not introduce a second visual language.
8. Preview buttons and any icon-only controls need `title` and `aria-label`. Verify narrow panel width and no horizontal overflow.
9. Avoid duplicate preview implementations. There should be one `playPreview` path and one shared preview panel.

## Preview data correctness

- The processed preview must pass draft DSP, not `savedDsp`.
- It must pass draft voice effects as it already does.
- Original preview must explicitly pass disabled/null effects and DSP.
- Preserve playback volume semantics and stop/race handling.

## Acceptance criteria

- One visible tab is labelled `Эффекты и DSP`; separate `DSP` tab is gone.
- Preview panel remains visible while settings below scroll.
- Original/processed/stop controls work through the existing preview command.
- Unsaved changes are audible in processed preview and remain unsaved until Save.
- Both settings groups can collapse independently.
- `npx vue-tsc --noEmit` passes.
- `cargo check --manifest-path src-tauri/Cargo.toml` passes.
