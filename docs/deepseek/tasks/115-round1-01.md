# Task 115 / Round 1 / Iteration 01 — настройка hide-on-blur для SoundPanel

## Контекст

Сейчас автоскрытие SoundPanel по `Focused(false)` временно удалено, потому что оно мешало перетаскиванию. Нужно вернуть это поведение как настраиваемое: по умолчанию включено, но пользователь может отключить его для настройки положения панели.

## Что сделать

### Backend

1. В `src-tauri/src/config/windows.rs` добавь `hide_on_blur: bool` в `SoundPanelWindowSettings` с `#[serde(default = "default_soundpanel_hide_on_blur")]`, default-функцией `true` и значением `true` в `Default`.
2. Добавь в `WindowsManager` методы `set_soundpanel_hide_on_blur(bool)` и `get_soundpanel_hide_on_blur()`, по существующему паттерну `stay_visible`.
3. Добавь Tauri-команду `sp_set_hide_on_blur` в `src-tauri/src/soundpanel/bindings.rs`, экспортируй её через `soundpanel/mod.rs` и зарегистрируй в `src-tauri/src/lib.rs`. Команда должна сохранять значение через `WindowsManager`.
4. В обработчике `Focused(false)` в `src-tauri/src/lib.rs` получай `WindowsManager` и вызывай `hide_soundpanel_window` только если `get_soundpanel_hide_on_blur()` возвращает true. Не используй для этого `stay_visible`.

### Frontend

5. В `src/types/settings.ts` добавь `hide_on_blur: boolean` в `SoundPanelWindowSettingsDto`.
6. В `src/components/settings/SettingsInterface.vue` добавь состояние `spHideOnBlur`, синхронизацию из `settings.soundpanel.hide_on_blur` и checkbox в секции «Звуковая панель»:
   - текст: `Скрывать при потере фокуса`;
   - hint: `Выключите, чтобы настроить положение панели перетаскиванием`;
   - изменение вызывает `sp_set_hide_on_blur`.
7. Не возвращай настройки окна в `SoundPanelTab.vue`; не меняй click-through/stay-visible логику воспроизведения.
