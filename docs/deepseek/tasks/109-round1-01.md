# Task 109-round1-01: История фраз не обновляется после отправки TTS

План: `docs/deepseek/plan/109-2026-07-09-phrase-history-not-refreshing.md` (читать обязательно).

## Симптом
Отправленная фраза (напр. «яблоко») не появляется в открытом списке «История фраз», пока не
свернуть/развернуть панель. Воспринимается как «не попало в историю».

## Корневая причина (найдена Claude, запись РАБОТАЕТ)
`src/components/PhraseHistoryList.vue` обновляет `phrases` только при: смене фильтра
(`watch(filterDebounced)` строка 45), раскрытии (`watch(isExpanded)` строка 49), удалении
(строка 72). **Нет реакции на новую отправку TTS.** Запись в `phrase_history.json` происходит
корректно (`record_phrase` в commands/mod.rs:269 — подтверждено, «яблоко» есть в файле).

Доступное событие: backend эмитит `"text-sent-to-tts"` в `speak_text_internal`
(commands/mod.rs:186) — доходит до webview (имя: `text-sent-to-tts`, events.rs:111).

## Задача DeepSeek — Вариант A (минимальный, фронтенд-only)

В `src/components/PhraseHistoryList.vue`:
1. Импортировать: `listen` из `@tauri-apps/api/event`, и `onMounted` из `vue` (onUnmounted уже есть).
2. Завести переменную для unlisten: `let unlistenTextSent: UnlistenFn | null = null`
   (тип `UnlistenFn` из `@tauri-apps/api/event`).
3. В `onMounted`:
   ```ts
   onMounted(async () => {
     unlistenTextSent = await listen('text-sent-to-tts', () => {
       if (isExpanded.value) loadPhrases()
     })
   })
   ```
   перезагружать только если панель раскрыта (`isExpanded.value`), чтобы не дёргать backend
   вхолостую когда список скрыт.
4. В `onUnmounted` (уже есть, строки 91-95) — добавить вызов `unlistenTextSent?.()` для cleanup.
5. (Опц., если просто) лёгкий debounce reload, чтобы flurry отправок не вызвал N запросов подряд.

### НЕ ломать
- Существующие триггеры reload (filter/expand/remove) — оставить.
- Логику записи на бэкенде (`record_phrase`/`record_text`) — не трогать (она корректна).
- Дедупликацию/поиск фраз (контракт case-insensitive, history.rs:255-257).

### Альтернатива (если A кажется недостаточным)
Вариант B: бэкенд эмитит `"phrase-history-changed"` после persist в `record_phrase`/
`record_text`/`delete_phrase`/`clear_phrases`, фронт подписывается на него. Это надёжнее
(покрывает все пути записи), но требует проброса AppHandle в HistoryManager или эмит из
обёрток commands/history.rs. **Дефолт: реализовать вариант A** (закрывает симптом). Если
решишь делать B — сделай A+B, но A обязательно.

## Главное правило (из WORKFLOW)
НЕ доверять чек-боксам. Реально: список должен обновляться на событие без ручного
сворачивания. Проверить, что `loadPhrases` зовётся по событию и только когда `isExpanded`.

## Runtime (пользователь проверит)
- Открыть «Историю фраз». Ввести НОВОЕ слово (которого нет, напр. «мандарин») → Enter (TTS).
- Слово появляется в списке **автоматически**, без сворачивания панели.
- Повтор той же фразы → count растёт, список снова обновляется.
- Фильтр/раскрытие/удаление — работают как прежде.
