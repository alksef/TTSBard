# Task 107-round1-01: Split DSP sections into subpanels

Прочитай stage `docs/stage/35-audio-panel-subpanels.md` и review `docs/deepseek/reviews/106-round1-review.md`.

Раздели UI в `src/components/audio/DspSettings.vue` на три presentational-компонента:

- `src/components/audio/EqSettings.vue` — EQ и его bands;
- `src/components/audio/CompressorSettings.vue` — компрессор;
- `src/components/audio/LimiterSettings.vue` — limiter и hint.

Каждый дочерний компонент получает свой section object и состояние collapse через props, а изменения/переключение collapse сообщает через emits `mark-dirty` и `toggle`. `DspSettings.vue` остаётся владельцем `draftDsp` и передаёт `draftDsp.eq`, `draftDsp.compressor`, `draftDsp.limiter`; presets/save/cancel не менять.

Не переносить backend, composables или draft state. Не дублировать общие DSP-стили: оставь их в `DspSettings.vue` без scoped-изоляции либо вынеси в общий CSS так, чтобы дочерние элементы сохраняли текущий вид. Сохрани все диапазоны, labels, v-model и disabled conditions.

Проверки: `npx vue-tsc --noEmit`, `git diff --check`. Используй небольшие Edit/apply-patch операции, без shell-скриптов для записи больших файлов.
