# Task 101-round1-01: Саундпанель «постоянно висящая» + оставить click-through

План: `docs/deepseek/plan/101-2026-07-08-soundpanel-clickthrough-evaluation.md` (читать).

## Решение пользователя
«если сделать саундпанель постоянно висящей, то смысл [click-through] есть. тогда оставляем».

⇒ **Оставить click-through** (не убирать). Сделать саундпанель **«постоянно висящей»** — не
скрывать автоматически после воспроизведения звука, чтобы она висела как оверлей (и click-through
получает смысл: окно поверх, не перехватывает клики).

## Контекст (читать обязательно)
- `src-tauri/src/soundpanel_window.rs`:
  - `show_soundpanel_window` — `window.show()` + `set_focus()` + **`set_ignore_cursor_events(false)`**
    (строка 43, «Disabling clickthrough for active panel» — клики ловит).
  - `hide_soundpanel_window` — `window.hide()` + если clickthrough enabled →
    `set_ignore_cursor_events(true)`.
- **Авто-hide после play:** `sp_play_binding` (`soundpanel/bindings.rs:186-200`) после
  воспроизведения зовёт `state.emit_event(AppEvent::HideSoundPanelWindow)` → окно скрывается.
  Это НЕ «постоянно висящая» — каждый звук = окно исчезает.
- Click-through тумблер: плавающее окно (кнопка) + основное окно (чекбокс SoundPanelTab).
- A-Z играют через DOM keydown (нужен фокус) — план 97.

## Задача (DeepSeek)

### Этап 1 — диагностический факт-чек (зафиксировать, без правок)
Подтвердить текущее поведение:
1. После выбора звука (A-Z) окно скрывается? (Да — `HideSoundPanelWindow` в sp_play_binding.)
2. Click-through ON + окно показано → `set_ignore_cursor_events(false)` (show) vs `(true)`
   (hide+restore). Т.е. пока окно ВИДИМО, click-through фактически **выключен** (show форсирует
   false). ⇒ Чтобы click-through работал на видимом окне, show НЕ должен форсировать
   `set_ignore_cursor_events(false)` когда click-through включён.
3. A-Z через DOM keydown: при click-through (ignore cursor events) окно может терять фокус →
   A-Z не работают. Зафиксировать (это ограничение, не блокер — Intercept/NumPad работают через hook).

### Этап 2 — фиксы (по решению «постоянно висящая + оставить click-through»)

#### Фикс A: опция «не скрывать после воспроизведения» (persist)
Добавить настройку `soundpanel_stay_visible` (bool, default: false — обратная совместимость; ИЛИ
true — решить, см. ниже) в `WindowsManager` / windows.json (по образцу других soundpanel-настроек
opacity/bg_color/clickthrough).

- При **ON**: `sp_play_binding` НЕ зовёт `HideSoundPanelWindow` (окно остаётся висеть). Звук играет,
  окно на месте.
- При **OFF**: текущее поведение (скрыть после play).
- UI: чекбокс в `SoundPanelTab.vue` (раздел внешнего вида) + опционально кнопка в плавающем окне.
  Label: «Оставлять окно видимым (не скрывать после звука)».

#### Фикс B: показывать click-through-окно корректно
Сейчас `show_soundpanel_window` ВСЕГДА `set_ignore_cursor_events(false)`. Если click-through
включён И «stay visible», окно должно **оставаться click-through** после показа (не форсировать
false). Изменить show-логику:
```rust
let sp_state = app_handle.state::<SoundPanelState>();
let clickthrough = sp_state.is_floating_clickthrough_enabled();
// НЕ форсировать false, если click-through включён — уважать настройку
let _ = window.set_ignore_cursor_events(clickthrough);
```
(Вместо нынешнего безусловного `false`.)
При этом `set_focus()` для click-through окна всё равно звать (фокус на момент показа), но после —
ignore cursor events = true (мышь проходит сквозь).

⚠️ Решить с пользователем (отметить): при click-through + stay-visible A-Z (DOM keydown) могут не
работать без фокуса. Это известное ограничение — Intercept/NumPad (hook) работают. Документировать
в hint.
