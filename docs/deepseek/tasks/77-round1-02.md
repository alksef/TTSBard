## C1 (CRITICAL): `active` computed — синхронизировать `activeId` при невалидном
**Файл:** `src/composables/useEditorTabs.ts`, computed `active` (~20-26).
**Проблема:** при невалидном `activeId` fallback `?? tabs.value[0]` тихо возвращает первый
таб, но `activeId` остаётся невалидным → `active.set()` пишет не туда / теряет данные.
**Решение:** в `get()` при невалидном `activeId` обновлять его:
```ts
const active = computed<EditorTab>({
  get: () => {
    const tab = tabs.value.find(t => t.id === activeId.value)
    if (tab) return tab
    // activeId невалиден — синхронизируем на первый таб.
    if (tabs.value.length > 0) activeId.value = tabs.value[0].id
    return tabs.value[0]
  },
  set: (v) => { /* см. C2 */ },
})
```

## C2 (CRITICAL): `active.set` — прямое присваивание вместо Object.assign
**Файл:** `src/composables/useEditorTabs.ts`, computed `active.set` (~22-25).
**Решение:**
```ts
set: (v) => {
  const t = tabs.value.find(t => t.id === activeId.value)
  if (t) {
    t.id = v.id
    t.title = v.title
    t.text = v.text
  }
},
```

## C3 (CRITICAL): `close()` — вычислять следующий активный ДО удаления
**Файл:** `src/composables/useEditorTabs.ts`, `close()` (~36-50).
**Проблема:** текущая логика `Math.min(idx, length-1)` после splice хрупкая.
**Решение:** явно вычислить nextActive до splice:
```ts
function close(id: string) {
  const idx = tabs.value.findIndex(t => t.id === id)
  if (idx === -1) return

  const wasActive = id === activeId.value
  // Сосед для выбора, если закрываем активный (предпочтительно предыдущий).
  let nextActiveId: string | null = null
  if (wasActive && tabs.value.length > 1) {
    const nextIdx = idx > 0 ? idx - 1 : idx + 1
    nextActiveId = tabs.value[nextIdx].id
  }

  tabs.value.splice(idx, 1)

  if (tabs.value.length === 0) {
    // Не оставлять пустое состояние — всегда ≥1 таб.
    const tab: EditorTab = { id: genId(), title: 'Текст 1', text: '' }
    tabs.value.push(tab)
    activeId.value = tab.id
    return
  }

  if (nextActiveId) activeId.value = nextActiveId
}
```

## M1 (MINOR): template ref вместо querySelector в startRename
**Файл:** `src/components/editor/EditorTabs.vue` (~20-28, шаблон ~53-61).
**Проблема:** `document.querySelector('.tab-rename-input')` — баг фокуса при нескольких
инпутах (быстрый дабл-клик на разные табы).
**Решение:** использовать template ref на `<input>`. Поскольку инпут рендерится только для
редактируемого таба (`v-if="editingId === tab.id"`), одного ref достаточно:
```ts
const renameInputRef = ref<HTMLInputElement | null>(null)

function startRename(tab: EditorTab) {
  editingId.value = tab.id
  editValue.value = tab.title
  nextTick(() => {
    renameInputRef.value?.focus()
    renameInputRef.value?.select()
  })
}
```
В шаблоне: `<input ref="renameInputRef" v-model="editValue" ... />`
