# Task 111-round1-01: SoundPanelTab — утечка Tauri-listener'ов

План: `docs/deepseek/plan/111-2026-07-11-soundpanel-lifecycle.md` (читать обязательно).

## Суть проблемы

Файл: `src/components/SoundPanelTab.vue`, строки ~323–360.

`onUnmounted` вызывается ВНУТРИ `async onMounted`, после множества `await`.
Vue 3 не привязывает lifecycle-хуки после первого `await` — колбек никогда
не вызовется. Три Tauri-listener'а утекают навсегда.

## Задача

Прочитай весь `<script setup>` в `SoundPanelTab.vue`. Найди текущую структуру:
- `onMounted(async () => { ... onUnmounted(...) ... })`
- Вызовы `await listen(...)` и регистрацию `onUnmounted` после них.

Переделай по паттерну:

```typescript
// 1. НА ВЕРХНЕМ УРОВНЕ <script setup> (синхронно, до onMounted):
const _cleanups: Array<() => void> = []

// 2. СИНХРОННЫЙ onUnmounted НА ВЕРХНЕМ УРОВНЕ:
onUnmounted(() => {
  _cleanups.forEach(fn => fn())
  _cleanups.length = 0
})

// 3. В onMounted — собирать cleanup через push:
onMounted(async () => {
  await loadSets()
  await loadBindings()
  // ...
  
  const unlistenAppearance = await listen('...', handler)
  _cleanups.push(() => unlistenAppearance())
  
  const unlistenBindings = await listen('...', handler)
  _cleanups.push(() => unlistenBindings())
  
  const unlistenActiveSet = await listen('...', handler)
  _cleanups.push(() => unlistenActiveSet())
})
```

## Правила

- Прочитай весь файл перед правкой. Не убирай существующую логику загрузки.
- Если в файле уже есть `onUnmounted` на верхнем уровне — объедини логику,
  не создавай дублей.
- Все имена событий (строки `listen(...)`) оставь без изменений.
- `_cleanups` — приватная переменная, назови как угодно (но не конфликтуй с
  существующими переменными файла).
- Убедись что не сломал reactive-переменные, которые используются внутри listener'ов.

## Верификация
1. `npx vue-tsc --noEmit` — 0 ошибок.
2. В отчёте: показать новый `onUnmounted` (строки), показать что три `_cleanups.push()`
   присутствуют, показать что `onUnmounted` уже НЕ внутри `onMounted`.

## Не делать
- Не переписывать логику loadSets / loadBindings.
- Не трогать другие компоненты.
- Не добавлять debounce или throttle к listener'ам — это отдельная задача.
