# Task: backend safe logging audit

Implement only the backend part of `docs/stage/28-secret-safe-logging.md`.

## Scope

Modify only backend files under `src-tauri/src/`, focused on logging, AI clients, TTS clients, and proxy utilities. Do not change frontend, DTO contracts, startup settings loading, old log files, or add an export command.

## Requirements

1. Add a small reusable backend helper for limited masking of sensitive values. For sufficiently long values use the format: first 4 characters + `[MASK]` + last 4 characters. Empty or short values must not be revealed in full; return `[MASK]` or an equivalent safe value. Add unit tests for normal, short, empty, and Unicode-safe inputs.
2. Audit and fix backend logs that currently output API-key prefix/suffix, proxy URLs with possible credentials, URLs with credential query parameters, or other sensitive values. Preserve useful metadata such as provider, model, has_value, scheme/host/port without user/password/query/fragment, status, and lengths.
3. In particular check `ai/openai.rs`, `ai/deepseek.rs`, `ai/zai.rs`, `ai/custom.rs`, `tts/openai.rs`, `tts/proxy_utils.rs`, local TTS, and analogous locations found by search. Do not log request bodies, authorization headers, API keys, or access tokens.
4. Normalize URLs before logging so username/password, query, and fragment are excluded. If safe normalization is not supported by current dependencies, log only scheme/host/port or whether a URL is present.
5. Do not add a global text filter and do not modify existing old log files.

## Checks

- Run `cargo fmt --check` (or format only changed Rust files if needed).
- Run targeted helper tests and `cargo check`.
- Review the final diff and search for remaining open secrets in backend logs.

## Constraints

Do not modify frontend in this iteration. Do not claim completion without actual diff and test verification.
