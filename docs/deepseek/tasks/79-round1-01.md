# Task 79-round1-01: Меню редактора — AI + история (план 79, stage 07 этап 1)

Реализуй dropdown-меню редактора по плану `docs/plans/79-2026-06-30-editor-menu.md`.
Прочитай план ПОЛНОСТЬЮ — там точки интеграции, архитектура, код-скелет, риски.

## Суть
Dropdown-меню из кнопки в `InputPanel`: пункты «AI: корректировать», «AI: дописать»,
«История фраз». **Этап 1 — без орфографии** (онлайн-орфография = отдельный план, этап 2).
Ключевое: **переиспользуем существующие команды и UI**, не дублируем логику.

## Что создать / изменить (точно по плану 79)

### 1. НОВЫЙ `src/components/editor/EditorMenu.vue`
Dropdown-меню. Props: `isAiEnabled: boolean`, `hasText: boolean`. Emits: `correct`,
`complete`, `toggle-history`.
- Кнопка-триггер (иконка из lucide-vue-next, напр. `MoreHorizontal`).
- Dropdown с пунктами (см. шаблон в плане 79): «AI: корректировать», «AI: дописать»
  (обе `:disabled="!hasText || !isAiEnabled"`), разделитель, «История фраз».
- **Click-outside-to-close:** `document` listener + маркер `data-editor-menu` (надёжнее
  `@click.self` overlay — popover не перекрывает контент). **Без overlay.**
- **Escape:** keydown listener → close.
- `run(fn)`: close + вызвать emit.
- **Тема — только CSS-vars** (`--color-bg-elevated`, `--color-border-strong`, `--shadow-soft`,
  `--color-text-primary`, `--color-accent`).
- **a11y:** `aria-expanded`, `aria-haspopup` на триггере; фокус на первый пункт при открытии.
- **onMounted/onUnmounted:** добавить/убрать document listeners (без утечек).
Полный код-скелет в плане 79 — следуй ему.

### 2. ИЗМЕНИТЬ `src/components/InputPanel.vue`
- Импортировать `EditorMenu`, иконку меню.
- **Оставить `correct-button`** (привычный быстрый доступ к AI) **И добавить `EditorMenu`
  рядом.** Меню — для остальных действий. Не ломать существующий UX.
- `EditorMenu` позиционируется рядом с `correct-button` (CSS-vars тема).
- Обработчики:
  - `@correct` → существующий `correctText()` (строки 131-142). **Переиспользовать, НЕ дублировать.**
  - `@complete` → новая тонкая обёртка `completeText()`:
    ```ts
    async function completeText() {
      if (!text.value.trim()) return
      isCorrecting.value = true
      try {
        const addition = await invoke<string>('get_ai_completion', { context: text.value })
        if (addition) text.value = `${text.value} ${addition}`.trim()
      } catch (e) {
        debugError('[InputPanel] AI completion failed:', e)
        showError('Не удалось дописать текст')
      } finally {
        isCorrecting.value = false
      }
    }
    ```
    Это **явный ручной вызов** той же команды `get_ai_completion`, что autocomplete в
    `TtsEditor.vue:176-200`. Логику НЕ дублировать.
  - `@toggle-history` → переключить `showHistory` ref; `v-if="showHistory"` на
    `<PhraseHistoryList>` (строка 210). По умолчанию `showHistory = true` (текущее поведение).
- **Minimal-mode:** меню-кнопка скрыта в minimal (`v-if="!isMinimalMode"`), как
  `correct-button`. Зафиксировать.
