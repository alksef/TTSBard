# Task 108-round1-01: AI-автодополнение — добавить debounce + in-flight guard

План: `docs/deepseek/plan/108-2026-07-09-ai-completion-no-debounce.md` (читать обязательно).

## Симптом
При наборе текста в редакторе запросы автодополнения в DeepSeek уходят **на каждое нажатие**, а
должны — только во время паузы в наборе. Лишняя трата токенов + тормоза + устаревшие ответы.

## Корневая причина (найдена Claude)
`src/components/editor/TtsEditor.vue`, `hybridSource` (строки 131-223) — это CodeMirror
`CompletionSource`. В AI-ветке (layer 2, строки 188-214) стоит прямой `await
invoke<string>('get_ai_completion', { context: beforeCursor })` **без дебаунса и без кэша**.
CodeMirror вызывает source при каждом вводе → сетевой запрос на каждое нажатие.

Доказательство: `grep -rn "debounce\|setTimeout" src/components/editor/` — пусто.

## Задача DeepSeek

### 1. Debounce-хелпер (если в проекте нет готового)
Проверить наличие `debounce` в `src/utils/` / `src/composables/`. Если есть — использовать.
Если нет — добавить простой в `src/utils/` (или локально в TtsEditor.vue, если нигде больше не
нужно). Требования к хелперу для async:
- Принимает async-fn и delayMs.
- Возвращает Promise результата ИЛИ `null`, если вызов superseded более поздним (чтобы caller
  понял, что ответ устарел и проигнорировал его).
- Внутри: таймер (setTimeout/clearTimeout), сбрасываемый при новом вызове; token/флаг «последний».

### 2. Применить к AI-слою в `hybridSource` (TtsEditor.vue:188-214)
- Константа `AI_COMPLETION_DEBOUNCE_MS` = **700** (между 600–800).
- AI-запрос вызывать через debounced-хелпер. Если вернулся `null` (superseded) — **не
  добавлять** AI-option на этом тике (локальные слои 0/1 уже отрисованы; AI подтянется на паузе).
- Минимальный порог контекста: не звать AI, если `contextWords` слишком короткий
  (напр. `contextWords.split(/\s+/).length < 2` ИЛИ `beforeCursor.trim().length < 8`) — константа.

### 3. НЕ трогать
- `activateOnTyping` (оставить true — локальные слои должны быть мгновенными).
- `presetSource` (quick-insert `\key`/`%key`, план 99) — локальный, быстрый.
- Слои 0 (`get_history_suggestions`) и 1 (`get_phrase_completion`) — локальные, дебаунсить НЕ надо.
- Бэкенд `get_ai_completion` (`src-tauri/src/commands/ai.rs`).
- Формат AI-option (`✨ words`, detail 'AI', кастомный apply) — сохранить как есть.

### 4. (Опционально, если просто и безопасно)
Лёгкий in-memory кэш `Map<context, string>` с TTL ~10с для повторного показа того же контекста.
Если добавляет сложности — пропустить, debounce достаточно.

## Главное правило (из WORKFLOW)
НЕ доверять чек-боксам. Реально: AI-запрос должен идти через debounce с отбросом устаревших.
Проверить, что локальные слои остались мгновенными (не обёрнуты в debounce).

## Runtime (пользователь проверит)
- AI-автодополнение включено, DeepSeek настроен.
- Быстрый набор без пауз → запросы в DeepSeek уходят ТОЛЬКО после паузы (~700мс тишины),
  не на каждый символ.
- На паузе — `✨ ...` появляется. Локальные история/фразы — мгновенно всегда.
- Продолжение печати во время ожидания — устаревший ответ отбрасывается.
