## Goal

The `playback-control` window has a transparent empty area at the bottom because:
- Window is fixed 350Ã—400 (tauri.conf.json)
- Card `.playback-window` has `min-height: 150px` but doesn't fill the window
- `body` has no `height:100vh`
- `transparent: true` so the gap below the card is see-through

Fix: autosize window height to content + CSS safety background on body.

## Part 1: Autosize window height to content (Vue â†’ Tauri)

In `src-playback/PlaybackControlApp.vue`:

1. Add import: `watch` and `nextTick` from `vue` (already have `ref, onMounted, onUnmounted, computed`).

2. Add import: `LogicalSize` from `@tauri-apps/api/window` or `@tauri-apps/api/dpi`.

3. Add constants at the top of `<script setup>`:
   ```
   const MIN_H = 150
   const MAX_H = 600
   ```

4. Add a template ref on the `.playback-window` div (the one at line 115). Name it e.g. `playbackCard`.

5. Write a `resizeToFit()` async function that:
   - Calls `await nextTick()` and `requestAnimationFrame` before measuring (use `new Promise(r => requestAnimationFrame(r))`)
   - Gets the card's `offsetHeight` or `scrollHeight` from the template ref
   - Clamps it between `MIN_H` and `MAX_H`
   - Gets current window size via `getCurrentWindow().outerSize()`
   - If new height differs from current height, calls `getCurrentWindow().setSize(new LogicalSize(350, clampedHeight))`
   - Must NOT trigger fetchState (check: does setSize cause any events that lead to fetchState? It shouldn't â€” state events come from Rust backend only).

6. Call `resizeToFit()`:
   - In `onMounted`, after `fetchState()` (both after state load AND after appearance load, at the end of onMounted)
   - Via `watch(() => state.value, ...)` with `{ deep: true }` â€” after each state change (this covers queue/recent appearing/disappearing)
   - In the `playback-appearance-update` event listener, after setting opacity/bgColor

7. Optimize: skip setSize if calculated height equals current height (compare numbers). Use `outerSize()` for current.

8. Width must be 350. Use `LogicalSize` (Tauri v2: `import { LogicalSize } from '@tauri-apps/api/dpi'`). This matches the tauri.conf.json unit and is DPI-correct.

## Part 2: CSS background safety net

In `src-playback/PlaybackControlApp.vue`:

1. Global `<style>` block (lines 186-218):
   - Add `height: 100vh` and `margin: 0` to `html, body` (or `html` and `body` separately)
   - Set body background to `var(--bg)` â€” this fills the entire window area even when card is smaller than window height
   - Keep `overflow: hidden` on body

2. Scoped `<style>` block (lines 220+):
   - Change `.playback-window` from `min-height: 150px` to use `width: 100%; height: 100%` (or `flex: 1` if body becomes flex container)
   - Actually, simpler approach: make body `display: flex; align-items: flex-start; justify-content: center;` and keep card auto-sized. The body background (`var(--bg)`) already handles the fill.
   - The key: `.playback-window` keeps `min-height: 150px` and grows with content. Body background fills gaps.

   VERIFY: `.playback-window` has `border-radius: 16px` for rounded corners. Setting body background to `var(--bg)` is safe â€” the card sits on top with its borders and rounded corners. Transparency only shows at the very corners outside the border-radius, which is correct.

3. Ensure the `overlayStyle` (computed bg color with opacity) is also reflected on body. Either:
   - Apply `overlayStyle` to body via a computed style binding, OR
   - Make body use `var(--bg)` as a CSS variable that the overlayStyle mechanism doesn't override (body stays static CSS), OR  
   - Simply set body inline style to match: use `overlayStyle.backgroundColor` for body background.

   Simplest correct approach: set `document.body.style.background = overlayStyle.value.backgroundColor` in a function called after appearance load and on appearance-update events. This way the body background matches the card's dynamic color/opacity at all times.
