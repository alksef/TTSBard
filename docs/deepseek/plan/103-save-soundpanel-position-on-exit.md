# Plan 103: Сохранение позиции звуковой панели при завершении

## Контекст

Позиция `soundpanel` обычно сохраняется при скрытии окна, но прямой выход через `app.exit(0)` может завершить приложение, пока панель остаётся видимой. Для `playback-control` уже добавлен общий lifecycle-хук `RunEvent::ExitRequested`; нужно расширить его на звуковую панель.

UI-команда `quit_app` и пункт выхода в трее используют `app.exit(0)`, поэтому оба должны покрываться одним обработчиком завершения.

## Цель

Перед завершением приложения сохранить актуальную физическую позицию видимой или существующей `soundpanel`, не нарушив текущую логику скрытия панели.

## Ограниченный набор файлов

- `src-tauri/src/soundpanel_window.rs` — вынести общий helper сохранения позиции и использовать его при hide.
- `src-tauri/src/lib.rs` — вызвать сохранение soundpanel в существующем `RunEvent::ExitRequested` рядом с playback-control.

## Требования

1. Helper должен безопасно завершаться, если окно, `WindowsManager` или `outer_position()` недоступны.
2. `hide_soundpanel_window` должен сохранить прежнее поведение и использовать helper без дублирования.
3. Lifecycle-хук не должен показывать/скрывать окна и не должен блокировать завершение.
4. Не менять `quit_app` и tray-логику: оба уже вызывают `app.exit(0)` и должны автоматически попасть в `ExitRequested`.
5. Не трогать пользовательские несвязанные изменения.

## Приёмка

- При выходе из UI-кнопки «Выход» координаты `soundpanel` сохраняются.
- При выходе через трей координаты `soundpanel` сохраняются.
- Скрытие панели по-прежнему сохраняет позицию.
- `cargo check --manifest-path src-tauri/Cargo.toml` проходит.
- Релевантные Rust-тесты проходят.
