# Plan 125 — устойчивый Windows debug build для libclang

## Цель

Сделать `scripts/build-debug.bat` устойчивым к зависимости `espeak-rs-sys` от
`libclang.dll`: автоматически находить LLVM в стандартных Windows-путях и
передавать найденный каталог через `LIBCLANG_PATH`, а при отсутствии библиотеки
завершаться понятным сообщением с инструкцией установки.

## Контекст

Ветка `dev_tts` добавила `espeak-rs = "0.2.0"` в коммите `331f951`. Его
`espeak-rs-sys` запускает bindgen во время Cargo-сборки. Сейчас
`scripts/build-debug.bat` доходит до Rust-компиляции и падает с
`Unable to find libclang`; стандартная проверка toolchain эту зависимость не
проверяет.

## Область изменений

- `scripts/build.ps1` — проверка/автоопределение `libclang.dll` до запуска Tauri.
- При необходимости — краткая документация в существующем build-документе.

Не менять зависимости Piper, runtime или пользовательские локальные правки.
Не скрывать ошибку и не отключать Piper на Windows.

## Требования

1. Учитывать уже заданный пользователем `LIBCLANG_PATH`, если в нём есть
   `libclang.dll`.
2. Иначе проверить типовые каталоги LLVM, включая `C:\Program Files\LLVM\bin`
   и пользовательский каталог LLVM.
3. Передать найденный путь дочернему Cargo-процессу.
4. При отсутствии `libclang.dll` сообщить точную причину и команду/ссылку для
   установки LLVM, завершив скрипт до долгой сборки.
5. Сохранить UTF-8 BOM файла `scripts/build.ps1`.

## Проверка

- `cmd /c scripts\\build-debug.bat` не должен падать с неясной ошибкой bindgen.
- При доступном LLVM `LIBCLANG_PATH` должен быть виден Cargo.
- Проверить PowerShell 5.1-синтаксис и отсутствие изменений вне указанной
  области.
