# План 108: Bug — AI-автодополнение шлёт запросы в DeepSeek на каждое нажатие (нет дебаунса)

- **Дата:** 2026-07-09
- **Тип:** bug (editor autocomplete, performance + cost)
- **Симптом (от пользователя):** «надо изучить как работает автодополнение — дебаунс правильно работает?
  я набираю текст и сразу отправляются запросы в дипсик — а должно быть во время пауз»
- **Контекст цикла:** см. `docs/deepseek/WORKFLOW.md`

---

## Корневая причина (найдена Claude)

CodeMirror `autocompletion` подключён так (`src/components/editor/TtsEditor.vue:327-333`):
```ts
autocompletion({
  override: [hybridSource, presetSource],
  closeOnBlur: true,
  selectOnOpen: false,
  icons: true,
  defaultKeymap: true,
})
```
`hybridSource` (`TtsEditor.vue:131-223`) — это `CompletionSource`, который CodeMirror вызывает при
каждой активации автодополнения (по умолчанию `activateOnTyping: true`). Внутри `hybridSource`
**три invoke подряд**, включая **тяжёлый сетевой** AI-запрос:

```ts
// TtsEditor.vue:188-214 — layer 2, AI
const aiEnabled = editorSettings.value?.ai_completion ?? false
if (aiEnabled && contextWords) {
  try {
    const aiResult = await invoke<string>('get_ai_completion', {   // ← сетевой round-trip в DeepSeek
      context: beforeCursor,
    })
    ...
```

**Проблема:** этот `invoke('get_ai_completion')` выполняется **без дебаунса и без кэша**. CodeMirror
переоткрывает popup / перевызывает source при наборе → на каждое нажатие (после короткой внутренней
задержки CodeMirror ~170мс, которой недостаточно для сетевого AI-запроса) уходит полный
round-trip в DeepSeek. Это:
1. **Лишние траты токенов** (каждый набранный символ → запрос, пока не наступила пауза).
2. **Тормоза UI** — popup ждёт сетевого ответа.
3. **Поток устаревших ответов** — поздний ответ от раннего запроса может прийти позже и попасть в popup.

Лёгкие слои (layer 0 `get_history_suggestions`, layer 1 `get_phrase_completion`) — локальные,
быстрые, им дебаунс менее критичен, но AI-слой (layer 2) — критичен.

### Доказательство отсутствия дебаунса
```bash
grep -rn "debounce\|setTimeout" src/components/editor/
# (пусто — никакого debounce в редакторе)
```
`useTextCompletion.ts` — тоже без debounce (простая обёртка invoke).

## Фикс

Цель: AI-запрос должен уходить **только после паузы в наборе** (trailing debounce), а не на каждый
символ. Плюс — отбрасывать устаревшие ответы.

### Подход (выбрать в реализации — рекомендован A)

**A. Debounce + in-flight guard для AI-слоя (рекомендуется).**
В `TtsEditor.vue`:
1. Добавить модульный хелпер debounce (или использовать уже имеющийся в проекте — проверить
   `src/utils/` / `src/composables/` на наличие `debounce`; если нет — написать простой):
   - Подпись вида `debounce<T>(fn: (arg) => Promise<T>, delayMs): (arg) => Promise<T | null>`,
     который возвращает `null`, если вызов superseded более поздним (отбрасывает устаревший).
   - Задержка для AI: **600–800 мс** (константа `AI_COMPLETION_DEBOUNCE_MS`).
2. В `hybridSource`, в AI-ветке (layer 2, ~строки 188-214): вызывать AI через debounced-функцию.
   Если текущий вызов superseded (пользователь продолжил печатать) — не добавлять AI-option
   (вернуть без AI на этом тике; CodeMirror покажет локальные варианты, а AI подтянется на
   следующей паузе).
3. Минимальная длина контекста для AI: не звать AI, если `contextWords` короче N (напр. 2 слова
   или < 8 символов) — чтобы не спамить на одно слово. Завести константу.

**B. (Дополнительно, опционально) Кэш по контексту.**
In-memory кэш `Map<context, string>` (с TTL ~10–15с или ограничением по размеру), чтобы повторный
показ popup на том же контексте не дёргал сеть. Если делать — аккуратно инвалидировать при
изменении текста.

### Что НЕ ломать
- Не отключать `activateOnTyping` (это полезно для локальных слоёв 0/1 — пользователь видит
  историю/фразы мгновенно). Дебаунсить **только AI-слой**.
- Не трогать `presetSource` (`\key` / `%key` quick-insert, план 99) — он локальный и быстрый.
- Не менять бэкенд-команду `get_ai_completion` (`src-tauri/src/commands/ai.rs`) — проблема на фронте.
- Сохранить существующий UX: AI-вариант `✨ words` появляется как completion-option.

## Верификация
1. `npx vue-tsc --noEmit` — 0/0.
2. `cargo check` — не нужен (фронт-only), но запустить для уверености 0/0.
3. **Runtime (главное):**
   - Включить AI-автодополнение в настройках (`ai_completion`), провайдер DeepSeek настроен.
   - Набирать текст **быстро** (несколько символов без пауз). В логах / network — запросы в DeepSeek
     уходят **только после паузы** (~600–800мс тишины), а не на каждый символ. (Можно сверить по
     логу приложения или счётчику.)
   - После паузы — AI-вариант `✨ ...` появляется в списке автодополнения.
   - Локальные слои (история/фразы) по-прежнему появляются мгновенно, без задержки.
   - Если продолжить печатать во время ожидания AI — устаревший ответ отбрасывается (не мелькает).

## Не делать
- Не переписывать архитектуру completion (только добавить debounce/guard к AI-слою).
- Не убирать слои 0/1/2 — только дебаунсить layer 2.
- Не трогать TTS, playback, capabilities (это планы 106/107).
