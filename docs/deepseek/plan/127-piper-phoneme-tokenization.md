# Piper: исправить тишину после синтеза

## Причина

`phoneme_id_map` Piper индексирует IPA-токены, включая одиночные символы и
некоторые многосимвольные комбинации. Текущий runtime делает `split_whitespace`
и ищет целое слово в карте, отбрасывая почти все фонемы. В результате ONNX
получает только BOS/EOS и генерирует почти нулевой/очень короткий звук.

## Решение

В `src-tauri/src/tts/piper/runtime.rs` заменить преобразование фонем на greedy
longest-match по UTF-8 строке: в каждой позиции сначала выбирать самый длинный
ключ `phoneme_id_map`, который совпадает с префиксом; при отсутствии совпадения
пропускать текущий Unicode scalar. Сохранять pad между найденными токенами,
BOS/EOS и текущую безопасную обработку неизвестных символов.

Добавить тесты для одиночных и составных IPA-токенов, включая пробелы и
двухсимвольный ключ, проверяющие, что токены не теряются.

Проверить `cargo fmt` только для изменённого файла и целевые тесты Piper.
