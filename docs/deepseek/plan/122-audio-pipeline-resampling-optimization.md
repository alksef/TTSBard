# Plan 122: Оптимизация аудиопайплайна (Устранение лишнего ресэмплинга)

**Дата:** 2026-07-11  
**Сложность:** Низкая — изменение логики ресэмплинга в `effects.rs`.

---

## 1. Проблема

В текущей схеме `apply_enhance` делает ресэмплинг входного сигнала до 48 kHz (потому что DeepFilterNet3 требует 48 kHz), обрабатывает его и делает ресэмплинг назад до исходной частоты (например, 24 kHz). Затем, если изменена скорость, `apply_speed` делает еще один ресэмплинг. Это неэффективно и снижает качество звука.

---

## 2. Решение

Оставить сигнал в 48 kHz после обработки DeepFilterNet:
1. Убрать обратный ресэмплинг из `apply_enhance`.
2. Обновить переменную `sample_rate` на `48000` в `apply_effects` при успешном применении очистки.
3. Все последующие эффекты обработают сигнал на частоте 48 kHz, и финальный WAV запишется как 48 kHz.

---

## 3. Шаги реализации

### 3.1 `src-tauri/src/audio/effects.rs`
1. Изменить сигнатуру `apply_enhance`:
   - Либо возвращать `Result<(Vec<f32>, u32), String>` (где `u32` — новый sample rate, т.е. 48000).
   - Либо просто задокументировать, что успешный вызов возвращает данные с частотой `model_sr` (48000 Hz).
2. Внутри `apply_enhance`:
   - Удалить блок:
     ```rust
     let output = if needs_resample { ... } else { enhanced };
     ```
   - Заменить его на прямое переинтерливирование матрицы `enhanced`.
3. Внутри `apply_effects`:
   - При успешном вызове `apply_enhance` обновлять мутабельную переменную `sample_rate`:
     ```rust
     sample_rate = 48000;
     ```

---

## 4. Критерии приемки
1. Проект собирается (`cargo check`).
2. При включенном DeepFilterNet на выходе получается WAV с частотой дискретизации 48000 Hz.
