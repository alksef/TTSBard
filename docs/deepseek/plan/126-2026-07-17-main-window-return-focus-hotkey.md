# План 126: Локальная клавиша возврата фокуса главного окна

## Цель

Добавить в главное окно локальную настраиваемую клавишу «Вернуть фокус в предыдущее
окно». Значение по умолчанию — `Ctrl+F`. Сценарий: пользователь вызывает TTSBard
глобальной клавишей поверх игры, вводит текст в окне на втором мониторе, затем нажимает
локальную клавишу; игра получает foreground и клавиатурный ввод, а TTSBard остаётся
видимым.

## Архитектура

- Добавить отдельное поле `return_previous_window` в `HotkeySettings` с `#[serde(default)]`.
- Дефолт: `Ctrl+F`.
- Поле присутствует в Rust DTO и TypeScript `HotkeySettingsDto`.
- Поле не регистрировать как global shortcut.
- Запись/сброс использовать существующие `set_hotkey` и `reset_hotkey_to_default`,
  расширив валидацию допустимых имён и проверку конфликтов со всеми сохранёнными
  global hotkeys.
- При `handle_main_window`, до `show`, `set_always_on_top` и `set_focus`, сохранить
  внешний foreground HWND в runtime state. Не сохранять окна TTSBard.
- Native-команда возврата должна снять `always_on_top` у main, вызвать
  `SetForegroundWindow(saved_hwnd)` и мягко обработать отсутствие/невалидность HWND
  или отказ Windows. `hide()` не использовать.
- Реализация Win32 должна быть под `cfg(windows)` с безопасным no-op/error на других
  платформах.
- Локальный keydown listener должен работать только в главном окне, использовать
  физический `KeyboardEvent.code`, учитывать Ctrl/Shift/Alt/Meta, предотвращать default
  только для совпавшего binding и вызывать native-команду возврата.
- В HotkeysPanel добавить строку в отдельном блоке «Главное окно» с записью, сбросом и
  отображением `Ctrl+F`; существующие global строки не менять по поведению.

## Acceptance criteria

- Старые settings.json без нового поля загружаются.
- После запуска дефолт отображается как `Ctrl+F`.
- Пользователь может записать другое сочетание и сбросить его к `Ctrl+F`.
- Глобальное открытие TTSBard запоминает игру/предыдущее внешнее окно до активации.
- Локальная клавиша возвращает foreground сохранённому HWND без скрытия TTSBard.
- Локальная клавиша не срабатывает в soundpanel/playback-control и вне главного окна.
- Конфликт с существующим global shortcut отклоняется.
- `cargo check`, `npx vue-tsc --noEmit`, `git diff --check` проходят.

## Ограничения

- Windows foreground lock может отказать в активации другого процесса; это не должно
  приводить к скрытию TTSBard или панике.
- Не использовать `AttachThreadInput` как основной механизм.
- Не менять экспериментальный NumPad Control и не смешивать его с этим binding.
