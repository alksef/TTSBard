# План 130: восстановление фокуса редактора после возврата из трея

## Проблема

После скрытия главного окна в трей вызов окна глобальной горячей клавишей показывает
его, но ввод в CodeMirror не начинается до клика мышью. Native-фокус окна
восстанавливается, а DOM-фокус редактора — нет.

## Диагностика

- `src-tauri/src/hotkeys.rs` и `src-tauri/src/setup.rs` восстанавливают только
  видимость и фокус native-окна.
- `src/components/editor/TtsEditor.vue` вызывает `view.focus()` только при
  монтировании; скрытие окна через `window.hide()` компонент не размонтирует.
- Frontend не подписан на восстановление фокуса главного окна.

## Решение

Добавить frontend-обработку `getCurrentWindow().onFocusChanged` в `App.vue`.
Если главное окно снова получило фокус и активна панель ввода, вызвать у
`InputPanel` явно экспортированный метод фокусировки редактора после `nextTick`.
Подписку обязательно снять при размонтировании. Backend и поведение остальных
панелей не менять.

## Приёмка

1. Скрыть окно в трей, вызвать его глобальной горячей клавишей и сразу напечатать
   текст без клика мышью.
2. Повторить сценарий через клик по иконке трея.
3. Убедиться, что при активной другой панели скрытый редактор не перехватывает
   фокус.
4. `npx vue-tsc --noEmit` и `cargo check --manifest-path src-tauri/Cargo.toml`
   проходят.
