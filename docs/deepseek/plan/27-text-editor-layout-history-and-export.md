# План реализации Stage 27

## Область

Реализовать `docs/stage/27-text-editor-layout-history-and-export.md` целиком: стабильную панель действий редактора, отдельную историю, вертикальный resize поля с сохранением, компактный режим с настраиваемым размером 300–500 px и сохранение исходных байтов TTS-ответа.

## Решение

1. Перестроить `InputPanel.vue` так, чтобы действия находились в отдельной горизонтальной панели под редактором и не зависели от раскрытия истории. Сохранить единый обработчик Enter/«Озвучить». В compact показывать только кнопку истории; AI и меню скрывать.
2. Перенести toggle истории в панель действий, оставив `PhraseHistoryList` только списком/панелью. При изменении состояния истории через событие временно менять высоту окна; обновления элементов списка не должны вызывать resize.
3. Добавить в настройки редактора сохранённую высоту обычного текстового поля, ограниченную безопасным диапазоном, и применять её к CodeMirror. Использовать resize CSS/обработчик изменения размера и сохранить значение с debounce или на `pointerup`.
4. Расширить настройки главного окна двумя размерами компактного режима (`compact_width`, `compact_height`) с default 450×400 и валидацией 300–500. Добавить backend-команду/методы сохранения и использовать их при переключении минимального режима. Не менять сохранённый размер обычного окна.
5. Сделать compact layout подстраиваемым под доступную область окна; при недостатке места использовать внутренний scroll. Для раскрытия истории применять временный размер с ограничением 500 px и возвращать исходный compact size после закрытия.
6. Добавить пункт «Сохранить аудио…» в меню. Frontend открывает save dialog, backend получает уже подготовленный текст, синтезирует через текущего провайдера и записывает исходные bytes без эффектов, playback и decode/re-encode. Расширение и фильтр должны соответствовать формату провайдера. Историю записывать только после успешного сохранения.
