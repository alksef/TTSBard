# План: обработка границ фраз и DSP-пресеты

## Решение

Реализовать Stage 32 поэтапно. Обработка границ фраз — самостоятельный этап аудиопотока, не часть DSP. DeepFilterNet сохраняется отдельным независимым эффектом. DSP получает три пользовательских режима: `Natural`, `Clear`, `Custom`.

## Разбиение работ

### Task 32-round1-01 — boundary processor

Чистые PCM-функции для проверки и исправления начала/конца фразы, с unit-тестами. Не менять UI и не менять публичный playback pipeline до отдельной интеграционной задачи.

### Task 32-round1-02 — settings and per-phrase pipeline integration

Добавить сохраняемую настройку `boundary_cleanup_enabled` и подключить обработку одной фразы в backend pipeline. Настоящую склейку соседних фраз оставить для отдельной задачи.

### Task 32-round1-03 — queue stitching integration (deferred)

Отложено по решению пользователя. Настоящий overlap-crossfade и переделка playback queue под lookahead не входят в текущую реализацию.

### Task 32-round1-04 — DSP preset model and UI

Добавить режимы `Natural`, `Clear`, `Custom`, применение пресетов, автоматический переход в `Custom` после ручного изменения DSP и UI-плашку обработки границ в начале секции эффектов. DeepFilterNet оставить независимым.

### Task 32-round1-05 — verification and polish

Расширить A/B-проверку, тесты коротких фраз и выполнить итоговые TypeScript/Rust checks. Исправления формировать отдельными итерационными task-файлами по результатам проверки.

## Общие требования

- Не менять поведение DeepFilterNet при переключении DSP-пресетов.
- Обработка границ включена по умолчанию и может быть выключена отдельной галкой.
- `Natural` не должен отключать boundary cleanup.
- Любая обработка должна сохранять число каналов, sample rate и длительность настолько, насколько это возможно.
- При ошибке необязательной обработки должен сохраняться playback fallback.
- После каждого task: независимо посмотреть diff и выполнить релевантные проверки до запуска следующего task.
