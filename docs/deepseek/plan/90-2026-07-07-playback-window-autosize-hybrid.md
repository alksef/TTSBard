# План 90: Убрать пустую прозрачную область внизу окна «Управление воспроизведением» (гибрид: автосайз + фон-страховка)

- **Дата:** 2026-07-07
- **Тип:** bugfix (UI/Tauri)
- **Связанный баг:** `docs/bugs/01-playback-window-empty-transparent-bottom.md`
- **Подход:** C (гибрид) — автосайз высоты окна под контент + фон-страховка на всю площадь
- **Контекст цикла:** см. `docs/deepseek/WORKFLOW.md`

---

## Постановка бага (кратко)

Окно `playback-control` (src-playback) имеет **фиксированный размер 350×400**
(`src-tauri/tauri.conf.json:41-42`, `resizable: false`), прозрачно
(`transparent: true`) и без рамки (`decorations: false`). Контент
`.playback-window` (`src-playback/PlaybackControlApp.vue:220-230`) имеет лишь
`min-height: 150px` и **не растягивается на всю площадь окна**, а `body`
(строки 211-217) не задаёт `height:100vh`. В итоге между низом карточки и низом
окна (400px) остаётся **прозрачная пустая зона** — «мёртвая» дыра внизу окна,
особенно заметная в пустом состоянии (Idle, без очереди/недавних).

## Цель

1. Окно больше не имеет прозрачной пустоты снизу: видимая карточка-фон должна
   совпадать с реальной высотой окна.
2. Высота окна **адаптируется к контенту** (динамически: при появлении/исчезновении
   очереди и недавних фраз), а не зафиксирована на 400px.
3. Сохранить: ширину 350px, `decorations:false`, `transparent:true`,
   `alwaysOnTop`, `skipTaskbar`, `hiddenTitle`, скругление 16px и рамку карточки.
4. Сохранить drag-region (заголовок `.window-header`) и всю текущую логику
   (пауза/стоп/повтор/replay/закрытие).
5. (опционально, рекомендовано) Сохранять высоту окна в настройках при изменении,
   по аналогии с позицией — см. раздел «Дополнительно (опц.)».

## Выбранный подход (C — гибрид)

**Две независимые части, обе обязательны:**

### Часть 1. Автосайз высоты окна под контент (Vue → Tauri)

Измерять реальную высоту контента и вызывать `getCurrentWindow().setSize(...)`
с фиксированной шириной 350 и вычисленной высотой, при изменении состояния.

- Источник высоты — внешняя карточка `.playback-window` (та, что с фоном и
  рамкой). Замерять её `scrollHeight`/`offsetHeight` через `ref` (template ref)
  или `getBoundingClientRect()`.
- Пересчёт и применение размера:
  - в `onMounted` (после первого рендера и `fetchState`);
  - в `watch(...)` на `state` (глубокий, т.к. меняются вложенные поля
    `queue`/`recent`/`status`);
  - после применения appearance-настроек (`pc_get_appearance`);
  - после каждого `fetchState()` (т.к. `fetchState` мутирует `state` — но
    следить, чтобы не зациклиться: `setSize` не должен триггерить `fetchState`).
- Использовать `requestAnimationFrame` / `nextTick` перед замером, чтобы DOM
  успел отрендериться после изменения `state`.
- Высоту ограничить: `MIN_H = 150`, `MAX_H = 600` (значения вынести в константы,
  подобрать так, чтобы при большой очереди/недавних окно не уходило за экран).
  Замеряемую высоту `clamp(MIN_H, MAX_H)`; если контент выше MAX — внутри
  карточки уже есть `max-height:120px` + `overflow-y:auto` на списках
  (`.queue-list`, `.recent-list`, строки 348-355), так что скролл берёт на себя
  переполнение.
- Ширина всегда `350` (LogicalSize; т.к. окно `transparent`, лучше использовать
  `LogicalSize` для DPI-корректности, либо `PhysicalSize` с учётом
  `window.scaleFactor()` — на усмотрение DeepSeek, но фиксировать явно 350 в
  тех же единицах, что и `tauri.conf.json`).
- Оптимизация: не дёргать `setSize`, если новая высота равна текущей
  (`window.outerSize()` / `innerSize()`), чтобы избежать лишних вызовов.

### Часть 2. Фон-страховка на всю площадь окна (CSS)

Даже если автосайз на мгновение отстанет, фон не должен «проваливаться» в
прозрачность:

- `html, body { height: 100vh; margin:0; }` (или `height:100%` на обоих),
  чтобы `body` занимал всю площадь окна.
- На `body` (или корневой элемент) **продублировать фон карточки**
  (`overlayStyle.backgroundColor` / `var(--bg)`) — либо сделать
  `.playback-window` растягиваемым (`width:100%` / `height:100%`, `flex:1`) так,
  чтобы при любом размере окна карточка всегда закрашивала всю площадь.
- Гарантировать: при любом размере окна между краем карточки и краем окна **нет
  прозрачного зазора**. Рекомендация — фон положить на `body` (через тот же
  `overlayStyle`), а `.playback-window` сделать «контентной обёрткой», занимающей
  100% площади с `border-radius:16px`. Тогда автосайз меняет только высоту окна,
  а визуально фон всегда = площадь окна.

> Важно: `transparent: true` оставляем (нужен для скруглённых углов без рамки
> ОС). Но теперь прозрачными должны быть **только угловые пиксели за пределами
> `border-radius`**, а не целая полоса снизу.

## Дополнительно (опц., рекомендовано) — сохранение высоты в настройках

По аналогии с позицией (`get_playback_position`/`set_playback_position` в
`src-tauri/src/config/windows.rs:247-257` и
`src-tauri/src/playback_window.rs`):

- Добавить в `PlaybackWindowSettings` (`src-tauri/src/config/windows.rs:34`)
  поле высоты (например `saved_height: Option<i32>` или конкретный тип позиции).
- Методы `get_playback_height`/`set_playback_height` в `WindowsManager`.
- Команды-геттеры/сеттеры по образцу `pc_get_appearance`/`pc_set_opacity`
  (`src-tauri/src/commands/playback_window.rs`) + регистрация в
  `src-tauri/src/lib.rs` рядом с другими `pc_*`.
- Vue: применять сохранённую высоту при `onMounted` (до первого автосайза — или
  автосайз её перетрёт, что нормально), а текущую высоту сохранять (debounce)
  после `setSize`.

> Если это удлиняет план и риск багов — можно вынести в отдельный план-сиквел.
> Решение за DeepSeek/ревьюером: минимум — Части 1+2 (обязательны); сохранение
> высоты — по желанию, если не усложняет.

## Критерии приёмки ( Definition of Done )

1. **Нет прозрачной пустоты снизу** при всех состояниях:
   - Idle, пусто (нет очереди/недавних);
   - Idle, есть недавние;
   - Playing/Paused/Stopped, есть очередь и/или недавние;
   - переходы между состояниями (появление/исчезновение секций).
   Проверить визуально: нижний край видимой карточки-фона = нижний край окна.
2. Высота окна меняется при изменении контента (появилась очередь → окно выросло;
   очередь исчезла → окно уменьшилось), без ручного ресайза.
3. Ширина остаётся 350px на всех DPI.
4. Drag работает (за заголовок `.window-header`), кнопки управления
   (пауза/стоп/повтор/replay/✕) кликабельны, реактивность состояния сохранена.
5. Окно не «прыгает» при каждом эвенте: нет лишних `setSize` (равные размеры не
   применяются), нет зацикливания setSize↔fetchState.
6. Тёмная и светлая темы (`[data-theme='light']`) корректны: фон-страховка
   берёт актуальный `--bg`/`overlayStyle`.
7. `cargo check` зелёный, `npx vue-tsc --noEmit` — 0 errors.
8. Позиция окна по-прежнему сохраняется между показами (регрессии по
   `get/set_playback_position` нет).

## Граничные случаи / риски

- **Зацикливание:** `setSize` не должен порождать события, ведущие к `fetchState`
  → `state` change → `setSize`. Следить через проверку «новая высота == текущая».
- **DPI/scaleFactor:** на HiDPI логические vs физические пиксели —
  использовать `LogicalSize` или домножать на `scaleFactor()`; ширина 350 должна
  визуально совпадать с конфигом.
- **Race при onMounted:** DOM может быть не отрисован — замерять после
  `nextTick` / `requestAnimationFrame`; состояние приходит асинхронно из
  `fetchState` — пересчёт после каждого обновления.
- **Списки с max-height:120px:** при переполнении окно не должно расти бесконечно
  (MAX_H=600 + внутренний скролл списков).
- **Прозрачность углов:** `transparent:true` нужен только для скругления;
  убедиться, что фон `body` не закрашивает углы за пределами `border-radius`
  (т.е. фон на `body` — ОК, т.к. `border-radius` у `.playback-window`).

## Не делать

- Не менять команды воспроизведения (`playback_pause`/`_resume`/`_stop`/...
  `replay_phrase`) и DTO `PlaybackStateDto`.
- Не трогать саундпанель и основное окно.
- Не вводить `resizable: true` у окна (управляем размером программно).
- Не переписывать структуру компонента, только точечно: ref + resize + CSS-фон.

## Итог

После фикса окно «Управление» всегда вплотную обнимает контент, прозрачная
дыра снизу исчезает во всех состояниях, высота адаптивна, фон-страховка не даёт
провалов прозрачности.
