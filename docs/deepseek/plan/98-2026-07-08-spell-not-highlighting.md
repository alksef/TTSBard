# План 98: Bug/UX — спелл не подсвечивает ошибки орфографии

- **Дата:** 2026-07-08
- **Тип:** bug/UX (default value + видимость тумблера)
- **Симптом (от пользователя):** «так и не понял как работает спел — не вижу чтобы подсвечивал
  орфографические ошибки»
- **Контекст цикла:** см. `docs/deepseek/WORKFLOW.md`

---

## Что проверено Claude (факты)

1. **Linter установлен корректно.** `src/components/editor/TtsEditor.vue:46` создаёт
   `createSpellLinter(checkWords, () => enabled.value)` и добавляет в `extensions` (line 292).
   `spellLinter.ts` — рабочий: парсит слова (`WORD_RE`), зовёт `checkWords`, строит `Diagnostic`
   с `severity:'warning'` и suggest-действиями._delay 400ms.
2. **`useSpellcheck.ts`:** `source = 'off'` если `!editorSettings.spellcheck_enabled`. `enabled`
   = `source !== 'off'`. `checkWords` зовёт `spellcheck` (офлайн) или `check_spelling_online`.
3. **ДЕФОЛТ = OFF.** `src-tauri/src/config/settings.rs:447-454`:
   ```rust
   #[serde(default)] pub spellcheck_enabled: bool,   // ← default false
   ```
   `bool` + `#[serde(default)]` → `false`. У нового/существующего юзера (если поля нет в
   settings.json) — **выключено**. ⇒ Подсветки нет, пока не включат вручную.
4. **Тумблер ЕСТЬ, но спрятан.** `SettingsAiPanel.vue:390-403` — чекбокс «Проверка орфографии
   (офлайн)» внутри панели «AI». Пользователь логично не нашёл его там (орфография — не AI).
5. **Офлайн-словарь:** `setup.rs:299 init_spellcheck` грузит `resources/dict/ru.{aff,dic}`.
   Если файлов нет → warn + spellcheck disabled. Нужно убедиться, что словарь попадает в
   resource bundle (см. `tauri.conf.json` / bundle.resources) — иначе даже включённый спелл
   молча не работает (команда `spellcheck` возвращает ошибку, linter её глотает → пусто).

⇒ **Два корня:** (а) дефолт off + тумблер не там; (б) проверить что словарь реально в bundle и
команда не падает молча.

## Решение (предлагаемое — подтвердить порядок с пользователем через план)

### 1. Default ON (рекомендуется)
Офлайн-спелл — локальный, бесплатный, полезный. Сделать `spellcheck_enabled = true` по умолчанию
для новых юзеров:
- В `settings.rs` `EditorSettings::Default` (или impl Default) — `spellcheck_enabled: true`,
  `spellcheck_source: SpellSource::Offline`.
- **Осторожно с существующими конфигами:** `#[serde(default)]` на поле даёт `false` только если
  поля нет. У уже сохранённых конфигов поле `spellcheck_enabled: false` явно — serde его
  прочитает как есть (не применит default). Это ОК (не ломаем выбор пользователя), но новые
  юзеры получат ON. Задокументировать.

### 2. Подсветить тумблер / переместить
Орфография — это редактор, не AI. Варианты (выбрать в плане):
- **(a)** Вынести тумблер орфографии из «AI»-панели в отдельную видимую секцию редактора
  (рядом с быстрым редактором / меню редактора). Плюс: пользователь находит.
- **(b)** Добавить пункт в `EditorMenu.vue` (меню редактора — уже есть `correct`/`complete`/
  `toggle-history`): «Орфография: вкл/выкл». Быстрый доступ из редактора.
- **(c)** Оба.

Рекомендация: **(a) + (b)** — вынести в видимое место + пункт в меню редактора.

### 3. Не молчать, если словаря нет (диагностика)
Сейчас: если `spellcheck` команда падает — `spellLinter.ts:21` `catch { return [] }` глотает
ошибку, пользователь видит НИЧЕГО (как сейчас). Добавить:
- Если офлайн-словарь не загрузился (`init_spellcheck` warn), а юзер включил офлайн-спелл —
  показать явную индикацию (toast / значок в редакторе / пункт меню «Словарь не найден»), а не
  молчаливое отсутствие подсветки.
- Проверить `tauri.conf.json` `bundle.resources` включает `resources/dict/*` — иначе в
  production-сборке словаря нет и спелл не работает (в debug может быть, в release — нет).

### 4. Linter: не глотать ошибки молча (минор)
В `spellLinter.ts` catch-return-[] — оставить, но залогировать в debug (уже есть `debugError`?).
Хотя бы в dev-режиме видеть, почему пусто.

## Задача DeepSeek

### Этап 1 — диагностика (сначала)
1. Проверить `tauri.conf.json` (или `*.conf.json`) → `bundle.resources` / `resources`: входит ли
   `resources/dict/ru.aff`, `ru.dic`? Если НЕТ — это причина «не работает в сборке».
2. Проверить: после ручного включения тумблера (Settings → AI → «Проверка орфографии») —
   появляется ли подсветка в редакторе? Если да — корень = default off + видимость. Если нет —
   корень = словарь/команда.
3. В логе (`%APPDATA%\ttsbard\logs` или консоль) — есть ли
   `[spellcheck] ru.aff not found` / `[spellcheck] initialized`?

### Этап 2 — фиксы (по диагностике)
- **Default ON:** `EditorSettings` default → `spellcheck_enabled: true`, source `Offline`.
- **Видимость:** переместить/продублировать тумблер орфографии в видимое место (секция редактора
  или пункт `EditorMenu`). Не ломать существующий путь сохранения (`set_editor_spellcheck_enabled`).
- **Bundle:** если словаря нет в `bundle.resources` — добавить (`resources/dict/*`).
- **Диагностика «нет словаря»:** явная индикация вместо молчания.

### Этап 3 — тесты
- Юнит: `EditorSettings::default().spellcheck_enabled == true`, `source == Offline`.
- Миграция: существующий конфиг с `spellcheck_enabled: false` остаётся `false` (не перетираем).
