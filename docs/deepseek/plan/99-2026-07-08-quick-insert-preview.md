# План 99: Feature — превью быстрой вставки (`\ключ` → замена)

- **Дата:** 2026-07-08
- **Тип:** feature / UX (editor, CodeMirror)
- **Симптом/идея (от пользователя):** «есть быстрая вставка — оценка. возможно после того как
  введу "\\&lt;символ&gt;" показывал на что будет заменено»
- **Контекст цикла:** см. `docs/deepseek/WORKFLOW.md`

---

## Контекст (что есть сейчас)

- **Быстрая вставка** — пресеты `\ключ` (→ replacement) и `%юзернейм` (→ username).
- Данные: `replacements: Map<string,string>` и `usernames` — грузятся в `InputPanel.vue:92`
  (`load_preprocessor_data`), пробрасываются в `TtsEditor` как props `replacements`/`usernames`.
- Подстановка сейчас происходит **по пробелу/Enter** в процессе ввода (логика в TtsEditor /
  `useTextCompletion`).
- README описывает: `\name` + пробел → `Александр`; `%admin` + пробел → `Администратор`.

## Цель
Когда пользователь ввёл токен `\ключ` (или `%key`) и ещё не нажал пробел — показать **рядом с
курсором / в редакторе**, на что он будет заменён. Чтобы не гадать и не «вслепую» жать пробел.

## Варианты реализации (оценить в плане)

### Вариант A — Tooltip/autocompletion рядом с курсором (рекомендуется)
CodeMirror 6 + `@codemirror/autocomplete`. Уже есть autocompletion в `TtsEditor.vue` (для фраз).
Расширить источник: при вводе `\` или `%` показывать попап с匹配ами из replacements/usernames
(текст + preview результата). Это естественный CodeMirror-way, переиспользует существующий UI.

### Вариант B — Inline ghost-text (как hint)
Показывать замену серым текстом (ghost) сразу после токена: `\name |Александр` (серым). Нажатие
пробела/Tab подтверждает. Сложнее в CodeMirror (decorations + phantom text).

### Вариант C — Простой line-tooltip
При наведении курсора на `\ключ` — всплывающая подсказка с заменой. Менее «живой», но проще.

**Рекомендация: Вариант A** — переиспользует autocompletion, единый UX, пользователь видит список
вариантов и preview.

## Задача DeepSeek

### Этап 1 — research (читать обязательно)
- `TtsEditor.vue` — как устроен текущий `autocompletion({ ... })` (line ~296). Какой источник
  (`useTextCompletion.ts`).
- Как сейчас парсится токен для подстановки по пробелу (найти обработчик пробела в
  `TtsEditor.vue` / `useTextCompletion.ts`).
- `replacements`/`usernames` props — формат, доступны ли в контексте autocompletion.

### Этап 2 — реализация (Вариант A)
1. Новый source для `autocompletion`: триггер на `\` и `%`. Показывать matching ключи из
   `replacements`/`usernames` с отображением результата замены (detail: «→ Александр»).
2. `apply` — вставить **результат замены** (а не сам токен) при выборе из попапа. Либо оставить
   подтверждение по пробелу (как сейчас), а попап — только preview/подсказка. Решить в задаче:
   - Если `apply` вставляет результат — это меняет текущее поведение (подстановка по Tab из
     попапа). Документировать.
   - Безопаснее: попап = preview + быстрый выбор, но пробел по-прежнему подставляет (старый путь
     не ломается).
3. Опционально: если введён `\точныйключ` и replacement есть — показать **inline-индикацию**
   (decorations, тонкая подсветка), что при пробеле будет замена.

### Этап 3 — edge cases
- Пустые replacements/usernames — попап не показывать (не мешать).
- `\` без совпадений — не показывать шум.
- `%username` — те же правила.
- Не ломать текущую подстановку по пробелу (regression-риск).

## Верификация
- `vue-tsc --noEmit` 0 ошибок.
- **Runtime:** ввести `\na` → виден preview «→ Александр» (или попап). Пробел — подставляется.
  При пустом списке replacements — попап не мешает вводу.
