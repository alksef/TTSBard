# План: сохранять голос Silero в метаданных аудиокеша

## Проблема

При синтезе Silero голос для ключа аудиокеша и записи истории берётся из `settings.tts.telegram.current_voice_id`. Команда получения текущего голоса из Telegram возвращает голос в UI, но не гарантирует синхронизацию этого значения с настройками. В результате аудиофайл сохраняется, а поле `voice` в истории остаётся пустым.

## Решение

Синхронизировать идентификатор голоса, полученный командой `telegram_get_current_voice`, с `SettingsManager` после успешного получения авторизованного текущего голоса. Сохранять только при отличии значения, не ломая поведение при отсутствии авторизации/голоса и не затрагивая другие провайдеры.

## Область изменений

- `src-tauri/src/commands/telegram.rs`
- тесты/вспомогательные проверки только если они естественно помещаются в существующую структуру проекта

## Приёмка

1. После получения текущего голоса `settings.tts.telegram.current_voice_id` содержит его ID.
2. Следующий вызов TTS для Silero записывает этот ID в `PhraseEntry.voice` и учитывает его в cache key.
3. При `None` или неавторизованном Telegram настройки не изменяются.
4. Существующая логика выбора голоса и остальных провайдеров не меняется.
5. Проходят `cargo check`, `npx vue-tsc --noEmit` и `git diff --check`.
