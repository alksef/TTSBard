# Plan 102: Сохранение позиции окна управления при завершении приложения

## Контекст

`src-tauri/src/playback_window.rs::hide_playback_window` сохраняет `outer_position()` окна `playback-control`, но выход через пункт «Выход» в системном трее вызывает `app.exit(0)` напрямую. В этом сценарии `hide_playback_window` не вызывается, поэтому сохранённые `playback.x/y` не обновляются.

## Цель

Гарантировать сохранение актуальной физической позиции окна `playback-control` перед завершением приложения, включая выход из трея. Существующее поведение скрытия окна и сохранения позиции не сломать.

## Ограниченный набор файлов

- `src-tauri/src/lib.rs` — обработать событие завершения приложения в цепочке запуска Tauri.
- `src-tauri/src/playback_window.rs` — при необходимости вынести/переиспользовать безопасную функцию сохранения позиции, если это уменьшает дублирование.

## Требования к реализации

1. Обработать `tauri::RunEvent::ExitRequested` или эквивалентный lifecycle-hook Tauri в существующей цепочке `.run(...)`.
2. На завершении вызвать логику, которая получает `playback-control` через `AppHandle`, читает `outer_position()` и сохраняет её через `WindowsManager::set_playback_position`.
3. Не показывать и не перемещать окно при выходе; если окна нет или позицию получить нельзя — завершение не блокировать.
4. Не дублировать запись так, чтобы существующее `hide_playback_window` перестало работать. Предпочтительно сделать общий helper сохранения позиции и использовать его из hide и exit-hook.
5. Не менять семантику основного окна: оно по-прежнему скрывается в трей при `CloseRequested`.

## Приёмочные критерии

- Если окно управления перед выходом было перемещено и приложение завершено через трей, в `%APPDATA%\\ttsbard\\windows.json` сохраняются новые `playback.x` и `playback.y`.
- При следующем запуске окно открывается в сохранённой позиции.
- Закрытие окна кнопкой в UI по-прежнему сохраняет позицию.
- `cargo check --manifest-path src-tauri/Cargo.toml` проходит.
- `cargo test --manifest-path src-tauri/Cargo.toml --lib config::windows` или релевантный набор тестов проходит.

## Ограничения

Не трогать историю фраз, аудиокеш, frontend-компонент окна и сторонние незакоммиченные изменения в рабочем дереве.
