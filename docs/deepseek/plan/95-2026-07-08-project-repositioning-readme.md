## Цель
Переработать описание проекта: уйти от абстрактного «TTS для общения» к **двойному позиционированию**:
- **(А) Люди с ограничениями речи** — используют TTS для повседневного общения (assistive).
- **(Б) Стримеры / контент-мейкеры** — TTS + звуковая панель для трансляций (broadcast).

Общий знаменатель: **«быстро превратить мысль в голос с минимумом отвлечения от основного занятия»**.

Толерантный язык (person-first): «люди с ограничениями речи», НЕ «инвалиды»/«больные».

Исторический контекст: проект вырос из **интервью со стримером aleksa**.

   ограничения в отсутствие речи — толерантно». Сохранить кейс, подать уважительно.
2. **Двойное позиционирование ≠ «всем для всех»** — единое ядро + две аудитории как акценты.
3. **Историческая справка** — коротко (1-2 предложения про aleksa), не мемуары.
4. **Актуальность** — править ПОСЛЕ реализации 92-94, описывать как «есть». Если править сейчас —
   фичи 92-94 описывать как «планируется/в разработке».
5. **Локализация** — README на русском; если есть EN-версия — синхронизировать (проверить наличие).
6. **Не задеть техническую точность** — горячие клавиши, API-эндпоинты, форматы — оставить как есть,
   менять только описательную часть.
