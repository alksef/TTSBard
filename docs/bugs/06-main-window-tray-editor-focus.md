# BUG-06: После возврата окна из трея редактор не получает фокус

- **Дата:** 2026-07-17
- **Компонент:** главное окно / трей / CodeMirror editor
- **Статус:** исправлен
- **Приоритет:** высокий — после вызова окна нельзя сразу печатать

## Симптом

После сворачивания главного окна в трей вызов окна глобальной горячей клавишей
показывает интерфейс, но текст с клавиатуры не вводится. Для начала ввода нужно
кликнуть мышью по редактору. Если окно не скрывать в трей и переключать фокус
только горячими клавишами, ввод работает.

## Воспроизведение

1. Открыть главное окно и убедиться, что активна панель ввода.
2. Скрыть окно в трей через закрытие окна либо быстрый режим.
3. Вызвать главное окно глобальной горячей клавишей или кликом по иконке трея.
4. Начать печатать без клика мышью.

## Фактическая причина

Backend восстанавливает видимость и native-фокус окна (`show`, `unminimize`,
`set_focus`), но не DOM-фокус CodeMirror. `TtsEditor` вызывает `view.focus()`
только в `onMounted`, а `window.hide()` не размонтирует Vue-компонент. Поэтому
после возврата окно видно и сфокусировано на уровне ОС, но редактор не является
активным получателем клавиатурного ввода.

## Исправление

При восстановлении фокуса главного окна frontend явно фокусирует редактор, если
активна панель ввода. Подписка на событие фокуса снимается при размонтировании.

## Проверка

- Вызов из трея: печать начинается без клика.
- Вызов глобальной горячей клавишей: печать начинается без клика.
- При другой активной панели редактор не фокусируется.

Автоматические проверки: `npx vue-tsc --noEmit` и `git diff --check` прошли.
`cargo check` в текущем окружении не завершён: сборка `espeak-rs-sys` требует
доступный `libclang.dll`, которого нет в установленном toolchain.
