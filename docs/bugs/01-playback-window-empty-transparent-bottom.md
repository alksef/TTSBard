# BUG-01: Пустая прозрачная область внизу окна «Управление воспроизведением»

- **Дата:** 2026-07-07
- **Окно:** `playback-control` (src-playback/PlaybackControlApp.vue)
- **Статус:** подтверждён, не исправлен

## Симптом

У окна «Управление воспроизведением» (плейер, открывается горячей клавишей
`Ctrl+Shift+F7`) **внизу видна пустая прозрачная область**. Она не закрашена
фоном карточки, сквозь неё просвечивает рабочий стол/другие окна, и она не
реагирует на взаимодействие. Особенно заметно в «пустом» состоянии (Idle, нет
очереди и недавних фраз), но присутствует всегда.

## Корневая причина

Размер окна задан **фиксированно и не зависит от контента**:

`src-tauri/tauri.conf.json:37-50` — окно `playback-control`:
```json
{
  "label": "playback-control",
  "url": "src-playback/index.html",
  "width": 350,
  "height": 400,        // ← фиксированная высота 400px
  "decorations": false,
  "transparent": true,  // ← фон окна прозрачный
  "alwaysOnTop": true,
  "skipTaskbar": true,
  "resizable": false,   // ← нельзя растянуть
  "visible": false,
  "hiddenTitle": true
}
```

Контент же (`src-playback/PlaybackControlApp.vue`) рисует карточку, высота
которой **растёт по содержимому**, а не заполняет всё окно:

`PlaybackControlApp.vue:220-230`:
```css
.playback-window {
  background: var(--bg);
  border: 1px solid var(--border);
  border-radius: 16px;
  padding: 12px;
  display: flex;
  flex-direction: column;
  gap: 12px;
  min-height: 150px;   /* ← минимум, но НЕ height:100vh / flex-fill */
}
```

`body` (`PlaybackControlApp.vue:211-217`) тоже не растягивает карточку:
```css
body {
  background: transparent;
  overflow: hidden;
  /* нет height:100vh, нет display:flex на .playback-window */
}
```

**Следствие:** реальная высота карточки-контента намного меньше 400px (в пустом
состоянии ≈150px). Окно прозрачно (`transparent: true`) и без рамки
(`decorations: false`), поэтому пространство между низом карточки и низом окна
(400px) остаётся **полностью прозрачной пустой зоной**, которая:
- не закрашена фоном (`overlayStyle` / `var(--bg)` применяется только к
  `.playback-window`, не к `body`);
- не является «кликом-сквозь» и не содержит UI — просто «мёртвая» прозрачная
  дыра внизу окна.

В состояниях с очередью/недавними фразами карточка становится выше, и пустота
уменьшается, но полностью не исчезает, пока контент не превысит 400px.

## Почему это баг

1. Визуально окно выглядит «обрезанным снизу» — пользователь видит, что
   рамка/скругление карточки заканчиваются, а окно ещё продолжается прозрачным
   «хвостом».
2. Этот хвост занимает место на экране и перекрывает то, что под ним, не неся
   никакой функции.
3. Высота окна жёстко задана (400px) и не адаптируется к контенту — это
   противоречит динамическому контенту (очередь/недавние появляются/исчезают).

## Возможные подходы к исправлению (для плана)

Нужно решить в плане DeepSeek — выбрать один:

- **A. Динамический размер под контент (рекомендуется).** Сделать высоту окна
  зависимой от высоты контента: измерять реальную высоту `.playback-window` и
  дёргать `getCurrentWindow().setSize(...)` из Vue при изменении состояния
  (watch на `state` / onMounted / после `fetchState`). Это убирает пустоту в
  принципе и адаптирует окно к контенту. Требует осторожности по
  min/max-высоте и сохранения ширины 350.
- **B. Растянуть карточку на всё окно + пересмотреть фикс-высоту.** Задать
  `html,body { height:100vh }` и `.playback-window { flex:1 }` / `height:100%`,
  чтобы фон закрашивал всю область 400×350. Минус: фиксированные 400px могут
  быть избыточны или, наоборот, тесны — не адаптивно, но самый простой фикс.
- **C. Гибрид:** фон/карточка на всю площадь окна (через `body`) **и**
  автоподгон высоты под контент с разумным `min-height`/`max-height`.

Рекомендация для плана — вариант **A (или C)**, т.к. контент динамический
(очередь/недавние), и фикс-высота 400 — первопричина.

## Затронутые файлы

- `src-tauri/tauri.conf.json` (37-50) — размер/свойства окна `playback-control`.
- `src-playback/PlaybackControlApp.vue` (114-218, 220-230) — разметка и стили
  контента, отсутствие растяжения на всё окно.
- (опционально) `src-tauri/src/playback_window.rs` — если решать автосайз через
  Rust-сторону (`set_size`).

## Шаги воспроизведения

1. Запустить приложение.
2. Открыть окно «Управление» (Ctrl+Shift+F7).
3. Наблюдать: карточка плейера занимает верх окна, **ниже неё — прозрачная
   пустая область до нижнего края окна**.
4. Добавить фразы в очередь/недавние — пустота уменьшится, но не исчезнет.
