# BUG-03: Пункты «Голосовой ввод» и «Направление письма» в контекстном меню редактора

- **Дата:** 2026-07-09
- **Окно/компонент:** редактор `src/components/editor/TtsEditor.vue` (CodeMirror 6)
- **Статус:** исследовано, НЕ баг приложения — правка отложена

## Симптом
По правому клику в текстовом редакторе в контекстном меню видны пункты
**«Другие инструменты → Голосовой ввод»** и **«Направление письма»**. Запрос был их
выключить.

## Заключение расследования

Эти пункты **не принадлежат приложению**. Они добавляются **операционной системой Windows**
в контекстное меню любого текстового поля.

### Факты (проверено)
1. **Редактор — CodeMirror 6** (`@codemirror/view` `EditorView`,
   `src/components/editor/TtsEditor.vue:3,42`). CodeMirror **по умолчанию не имеет своего
   контекстного меню**.
2. **В проекте НЕТ ни одного обработчика `contextmenu`** — grep по `src/` (vue/ts) пуст: нет
   `@contextmenu`, нет `addEventListener('contextmenu', ...)`, нет
   `EditorView.domEventHandlers({ contextmenu })`.
3. **Строк «Голосовой ввод» / «Направление письма» нет ни в одном файле** проекта
   (grep по vue/ts/rs/json/html, исключая node_modules) — пусто.
4. Единственное Tauri-меню в проекте — **трей** (`src-tauri/src/setup.rs:389-422`, только «Quit»).
   Своего контекстного меню редактора в коде нет.

### Цепочка появления меню
- Правый клик по редактору → CodeMirror не перехватывает → событие доходит до **WebView2
  (Edge)** → показывается нативное браузерное меню (Вырезать/Копировать/Вставить…).
- **Windows** добавляет к нему свои расширения для текстовых полей:
  - **«Голосовой ввод»** (Voice Typing);
  - **«Направление письма»** (handwriting/pen direction, Windows Ink).
- Это рисует ОС, а не CodeMirror и не WebView2.

### Доказательство
CodeMirror-меню пришлось бы создавать вручную (через `EditorView.domEventHandlers` /
`@contextmenu`), и тогда строки пунктов были бы в коде. Их там нет ⇒ меню не наше.

## Почему не правим сейчас
Убрать именно эти два пункта из кода приложения **нельзя** — их рисует Windows.

Доступные варианты (все с минусами, потому правка отложена):

- **A. Подавить всё нативное меню** (`EditorView.domEventHandlers({ contextmenu(e)
  { e.preventDefault() } })` или `@contextmenu.prevent` на обёртке в `TtsEditor.vue`).
  Минус: пропадёт **всё** меню, включая Вырезать/Копировать/Вставить — серьёзная потеря для
  текстового поля.
- **B. Подавить нативное + сделать своё меню** (Vue-компонент: Вырезать/Копировать/Вставить
  + AI-действия). Системных пунктов не будет, меню под контролем. Средняя задача.
- **C. Отключить системные функции в самой Windows** (Voice Typing / Windows Ink через
  реестр/настройки). Вне приложения — влияет на все программы.

Решение: оставить как есть, покаsuppress всего меню (потеря Копировать/Вставить) или разработка
своего меню (вариант B) не станут оправданными.

## Если решим править позже
- Минимальный фикс, устроит ли потеря браузерного меню → **вариант A**.
- Если нужно сохранить Копировать/Вставить и иметь контроль → **вариант B** (своё меню через
  Vue, эмитить действия в `TtsEditor` / `InputPanel`).
