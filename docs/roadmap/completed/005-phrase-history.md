# ROADMAP-005 — История отправленных фраз

**Дата:** 2026-06-28
**Статус:** `completed` — исследование и реализация завершены
**Решение:** хранение **A** (`phrase_history.json`) + UX **Вариант 2c** (история как черновики:
список фраз в InputPanel, клик → загрузить в редактор для правки перед отправкой).
**Из research:** `docs/research/01-...` → идея **C** (Persistent Phrase History)
**Связано:** [ROADMAP-004](./004-playback-control-floating-window.md)

## Цель (запрос пользователя)
- Главная задача истории — **не вводить фразу заново, а достать её целиком** из журнала и
  при необходимости поправить перед отправкой.
- Поскольку есть редактор (CodeMirror, план 71) — реализовать через него.

> **Уточнение vs ранней формулировки:** «быстрый повтор (озвучить как есть)» уже закрывает
> план 74 (5 недавних фраз в окне `playback-control` + replay). Этот план (75) фокусируется на
> другом кейсе — **извлечь ранее введённый текст в редактор** для правки/переиспользования.
> Поэтому выбран **Вариант 2c**, а не гибрид с мгновенным повтором.

## Контекст кода (что уже есть после 71–73)
- `src-tauri/src/history.rs` — `HistoryManager` (Trie слов + n-граммы), persistence в
  `input_history.json` / `ngrams.json`.
- `record_text` уже вызывается из фронтенда при отправке (план 72).
- Редактор `TtsEditor.vue` (CodeMirror) + autocomplete-источники (слой 0/1/2, план 73).
- События: `TextSentToTts` — момент отправки фразы (хороший триггер для записи в журнал).
- Воспроизведение: `rodio::Sink` (план 04 добавит `PlaybackFinished`).

> **Различие:** план 72 = история **слов** (для автокомплита). Здесь = история **целых фраз**
> (для повтора/переиспользования). Разные модели/файлы, но могут жить в одном `HistoryManager`.

## Варианты хранения
### A. Отдельный файл `phrase_history.json` (рекомендуемый)
- `Vec<PhraseEntry { text, timestamp, count, lastUsed }>`; лимит N (например 100–500),
  кольцевой буфер (старые вытесняются).
- Сортировка: по `lastUsed desc` (недавние сверху) и/или по `count` (частые).
- **Плюсы:** изолировано, просто бэкапить, не мешает Trie/словам.
### B. Часть существующего `input_history.json` (тот же HistoryManager)
- Добавить slice `phrases` рядом со словами и n-граммами.
- **Плюсы:** один менеджер/файл. **Минусы:** смешение ответственностей.
### C. SQLite / embedded DB
- Избыточно для сотен фраз; не соответствует существующей [JSON-стратегии](../../decisions/008-json-persistence.md).

**Рекомендация:** **A** (отдельный `phrase_history.json`), управляется тем же
`HistoryManager` (расширить) или новым `PhraseHistoryManager`.

## Варианты UX (как показывать и повторять)

### Вариант 1 — Список в плавающем окне управления воспроизведением (план 04)
- В окне `playback-control` секция «Последние 5 фраз» → клик = повторно озвучить.
- **Плюсы:** окна управления + история в одном месте; закрывает стримерский кейс.
- **Минусы:** только 5; не для редактирования.

### Вариант 2 — Через редактор CodeMirror (идея пользователя) ⭐
Несколько способов задействовать редактор:
- **2a. Autocomplete-источник «недавние фразы»:** при вводе показать целые недавние фразы
  как варианты продолжения/замены (новый слой в попапе плана 73, `type: "recent"`). Принять
  Tab → вставить фразу в редактор, потом править и отправить.
- **2b. Command Palette / быстрый выбор:** хоткей внутри редактора (например `Ctrl+Space`
  на пустой строке или `/`) открывает список недавних фраз; выбор загружает фразу в редактор.
- **2c. Режим «история как черновики»:** боковая панель/выпадение в InputPanel со списком
  фраз; двойной клик → загрузить в редактор для правки перед отправкой.
- **Плюсы:** повтор + **правка** фразы перед повтором (важно — текст мог устареть);
  переиспользует инфраструктуру автокомплита (план 73).
- **Минусы:** 2a может засорять попап; 2b/2c — дополнительный UI.

### Вариант 3 — Отдельная панель в основном окне (как SoundPanel/Preprocessor)
- Отдельный раздел «История фраз» в sidebar со списком, поиском, повтором, удалением.
- **Плюсы:** полнофункциональное управление (поиск, очистка, избранное). **Минусы:**
  больше UI; не «мгновенный» доступ.

### Вариант 4 — Гибрид (рекомендация)
- **Быстрый повтор (без правки):** 5 недавних фраз в плавающем окне управления (план 04) —
  один клик → озвучить.
- **Правка перед повтором:** через редактор — autocomplete-источник «недавние фразы» (2a)
  ИЛИ command-palette (2b) загружает фразу в редактор.
- **Полное управление:** панель в основном окне (3) — поиск/чистка/избранное.
- **Горячая клавиша** «повторить последнюю фразу» (replay last) — мгновенно.

## Решения (✅ подтверждено пользователем)
1. **Хранение:** **A** — отдельный файл `phrase_history.json` (управляется тем же
   `HistoryManager` или новым `PhraseHistoryManager`); persistent.
2. **UX:** **Вариант 2c** — список фраз внутри InputPanel (боковая панель/выпадение), клик →
   **загрузить фразу в редактор** для правки перед отправкой (не мгновенный повтор).
3. **Объём/вытеснение:** оставить как параметр (по умолчанию ~100, кольцевой буфер по
   `lastUsed`); уточнить при реализации.
4. **Дедупликация:** схлопывать одинаковые фразы (incr count, обновлять `lastUsed`) — не
   хранить каждую отправку отдельно (иначе список засоряется).
5. **Поиск/фильтр:** в списке 2c добавить текстовый фильтр (по подстроке) — обязательно,
   т.к. цель именно «достать нужную фразу», а не листать.
6. **Триггер записи:** `TextSentToTts` (момент отправки) — добавлять/обновлять фразу в журнале.

## Реализация 2c (концепция) + ЕДИНОЕ хранилище фраз
- **Единое хранилище всех фраз** (`phrase_history.json`, persistent): это **единственный**
  источник правды. Все отправленные фразы пишутся сюда (с дедупликацией).
- «5 последних» для окна `playback-control` (план 74) и полный список для `PhraseHistoryList`
  (план 75) — это **разные views/запросы из одного хранилища** (`get_phrases(filter, limit)`),
  а **не два отдельных журнала**.
- Новый Vue-компонент списка фраз (напр. `PhraseHistoryList.vue`) **внутри InputPanel**
  (рядом с `TtsEditor`): свёрнут/развёрнут по кнопке/хоткею.
- Содержимое: отфильтрованный список (поиск по подстроке), каждая строка — превью текста +
  timestamp/счётчик; клик → эмит выбранной фразы → `TtsEditor` заменяет содержимое документа
  (с подтверждением, если в редакторе уже есть несохранённый текст).
- Backend: расширить `HistoryManager` — `record_phrase(text)`, `get_phrases(filter, limit)`
  (этот же метод отдаёт «5 последних» при `limit=5` без фильтра), `delete_phrase(id)`,
  `clear_phrases()`. Persistence в `phrase_history.json`.

## Связь с другими планами
- **План 72** (`input_history.json` / `ngrams.json`) — история **слов** для автокомплита.
  Здесь — история **целых фраз**; отдельный файл, но тот же `HistoryManager`.
- **План 74** — «5 недавних» = **запрос** `get_phrases(limit=5)` из единого хранилища фраз
  (этого плана) для мгновенного повтора. План 75 — полный список с поиском из того же
  хранилища для извлечения в редактор.
