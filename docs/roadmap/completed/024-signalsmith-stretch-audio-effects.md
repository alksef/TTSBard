# ROADMAP-024 — Переход pitch/tempo на Signalsmith Stretch

- **Дата:** 2026-07-12
**Статус:** `completed` — pitch/tempo переведены на Signalsmith Stretch
- **Область:** обработка готового TTS-аудио в `src-tauri/src/audio/effects.rs`

## Цель

Заменить разрозненную связку:

- `rubato::FftFixedInOut` для скорости (меняет одновременно длительность и
  высоту);
- `pitch_shift` для высоты;
- пороговый `trim_silence`, который может обрывать речь,

на единый движок **Signalsmith Stretch**. Он должен независимо менять tempo и
pitch, корректно завершать поток и не терять окончание фразы.

Это улучшение не является исправлением BUG-04. До миграции отдельным маленьким
изменением нужно устранить потерю хвоста в текущем `rubato`-пути, чтобы
пользователь не зависел от сроков новой интеграции.

## Почему Signalsmith Stretch

Signalsmith Stretch — open-source C++11 header-only библиотека с лицензией MIT:

- независимые изменения time и pitch;
- есть formant correction/shift: важно для речи, потому что обычное повышение
  pitch делает голос «мультяшным», а понижение — неестественно гулким;
- автор заявляет тестирование с MSVC, то есть библиотека реалистична для
  Windows-сборки Tauri;
- нет LGPL/GPL-обязательств и динамической DLL-зависимости, как у SoundTouch и
  Rubber Band соответственно.

Источник: [Signalsmith Stretch](https://github.com/Signalsmith-Audio/signalsmith-stretch).

## Целевая семантика UI

Текущий slider «Скорость» должен стать **«Темп»**:

- `0` — исходная длительность;
- отрицательное значение — медленнее, положительное — быстрее;
- pitch голоса не изменяется.

Slider «Высота» сохраняет текущую шкалу `-100..100 → -12..+12 semitones`.
При наличии formant-режима добавить отдельный переключатель «Сохранять тембр
голоса» с включённым по умолчанию состоянием. Его не следует смешивать с
DeepFilterNet: шумоподавление и коррекция формант — независимые эффекты.

Нужно принять продуктовый диапазон tempo отдельно. Значения 0.25–4.0x из
текущего UI слишком экстремальны для естественной речи; стартовый безопасный
диапазон для оценки — 0.75–1.50x. Если сохраняется существующий slider,
экстремальные точки должны быть явно помечены как «может ухудшить разборчивость».

## Предлагаемая архитектура

```text
decode PCM (interleaved f32)
  → DeepFilterNet (только если включён)
  → Signalsmith Stretch (tempo + pitch + formant)
  → encode WAV
  → existing playback
```

- DeepFilterNet остаётся первым: это нейросетевое шумоподавление, не движок
  time/pitch manipulation.
- Signalsmith получает весь decoded clip офлайн. Это проще и надёжнее, чем
  realtime-поток: TTS уже сгенерирован до начала обработки.
- Один экземпляр stretch создаётся на одну фразу и корректно закрывается по
  API библиотеки; выход вычитывается до конца. Не добавлять heuristic
  `trim_silence` после него.
- PCM layout должен быть определён единообразно. Сейчас декодер добавляет
  Symphonia-каналы последовательно (planar), а дальнейший код считает буфер
  interleaved. В рамках миграции декодер обязан возвращать interleaved frames
  либо новый wrapper должен явным образом конвертировать planar ↔ interleaved.

## Интеграция с Rust

Signalsmith Stretch header-only, поэтому нужен маленький локальный C++ bridge,
а не большой внешний runtime:

1. Зафиксировать upstream commit/tag и положить заголовки в контролируемую
   vendor-зависимость либо получать их воспроизводимо в `build.rs`.
2. Добавить C ABI bridge (`extern "C"`) с минимальными операциями:
   создать процессор; настроить sample rate/channels/tempo/pitch/formant;
   подать frames; получить готовые frames; уничтожить процессор.
3. Компилировать bridge через `cc`/C++ в `src-tauri/build.rs`; явно включить
   оптимизацию для этого translation unit и MSVC-настройки.
4. Сделать безопасный Rust wrapper с RAII (`Drop`) и `Result`, не передавать
   указатели библиотеки по Tauri boundary.
5. В `effects.rs` заменить `apply_speed` и `apply_pitch` единой функцией
   `apply_stretch`; временно сохранить старый путь за feature flag для
   A/B-сравнения и отката.
6. Сохранить WAV encoder и `AudioEffects` DTO. Переименование `speed → tempo`
   в storage/API проводить только отдельной миграцией настроек с backward
   compatibility; на первом этапе можно оставить поле `speed`, изменив его
   документированную семантику.

## Риски и решения

| Риск | Как контролировать |
|---|---|
| C++ усложняет Windows build | Прототипировать `cargo check` и debug/release build на чистом Windows; bridge ограничить C ABI |
| Неестественный тембр при сильном pitch | Formant correction включён по умолчанию; ограничить рекомендуемый UI-диапазон; слушательские тесты на русской речи |
| Tail/latency | Офлайн-обработка с полным drain результата; тест с импульсом на последних 5–20 ms; никаких amplitude-based trim |
| Регрессия stereo | Тесты interleaved stereo и явное преобразование layout |
| Зависимость от upstream | Зафиксировать ревизию, хранить license notice, выполнить проверку обновлений вручную |

## Критерии приёмки

- В обработанном аудио сохраняется последний фонем/шёпот/дыхание; выход не
  короче ожидаемой длительности из-за внутреннего tail.
- `tempo` меняет длительность, но при `pitch = 0` не меняет измеряемую высоту
  синусоидального test tone.
- `pitch` на ±3/±6/±12 semitones не меняет целевую длительность более чем на
  ±1 frame (или зафиксированную библиотекой погрешность).
- Положения slider'ов `-100..100`, sample rate 16/24/44.1/48 kHz, mono/stereo
  дают валидный WAV и не вызывают panic/утечку native object.
- Прослушивание минимум 10 русских TTS-фраз на tempo 0.75/1.0/1.25/1.5 и pitch
  -6/0/+6: нет исчезновения окончания, заметной смены тембра при включённой
  коррекции формант и нет click на границах.
- `cargo check --manifest-path src-tauri/Cargo.toml`, `npx vue-tsc --noEmit`
  и `scripts/build.ps1 -Mode debug` проходят успешно.
