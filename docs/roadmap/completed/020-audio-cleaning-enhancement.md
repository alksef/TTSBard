# ROADMAP-020 — Очистка и улучшение аудио

**Дата:** 2026-07-11  
**Статус:** `completed` — DeepFilterNet интегрирован в аудиопайплайн
**Источник:** Запрос пользователя, обсуждение производительности  

---

## 1. Цель и требования

Добавить поддержку **DeepFilterNet** (нейросетевого подавления шума и улучшения речи) в аудиопайплайн TTSBard, чтобы улучшить качество синтезируемого Silero TTS звука (который приходит в MP3 и содержит легкую «металличность»/артефакты кодека).

### Ключевые требования:
1. **Отсутствие задержек (Latency):** Инициализация модели и компиляция графа ONNX движком `tract` занимает 150-400 мс. Запуск бинарника на каждую короткую фразу создаст неприемлемые лаги перед воспроизведением. Модель должна загружаться **один раз в память** при старте или при первом включении эффекта, а инференс должен проходить мгновенно (in-process, за 10-30 мс).
2. **Нативная интеграция:** Использование Rust-крейта `deep_filter` напрямую из git-репозитория как зависимости `src-tauri` без внешнего Python-окружения на компьютере пользователя.
3. **Настройки в UI:** Интеграция в существующий `AudioEffectsPanel.vue` (внутри `TtsPanel.vue`) как дополнительный переключатель «Очистка аудио (DeepFilterNet)» со слайдером уровня подавления (от 0% до 100%, мапящимся на `atten_lim_db` от 100 до 0 dB).

## 2. Предлагаемая архитектура

### 2.1 Backend (Rust)

1. **`src-tauri/Cargo.toml`**
   - Добавить Git-зависимость `deep_filter` с подкаталогом `libDF` и включенной фичей `tract` для нативного инференса ONNX.
   - Добавить `tract-onnx` / `tract-pulse` при необходимости (если они не реэкспортируются).

2. **Модели (Assets)**
   - Встроить или динамически скачивать/подгружать модель `DeepFilterNet3_onnx.tar.gz` (весит ~3.5 МБ). Приоритет — встраивание в бинарник через Cargo features `default-model` в `deep_filter`, либо ручной `include_bytes!` модели.

3. **`src-tauri/src/audio/effects.rs`**
   - Расширить структуру настроек эффектов `AudioEffects`:
     ```rust
     pub struct AudioEffects {
         pub pitch: i16,
         pub speed: i16,
         pub volume: i16,
         pub enhance_enabled: bool,
         pub enhance_atten_db: f32, // limit of attenuation, e.g., 10.0-15.0 dB for mild enhancement
     }
     ```
   - Добавить инициализацию глобального стейта инференса DeepFilterNet (например, `Mutex<Option<DfTract>>` в `AppState` или лениво инициализировать в `apply_effects` с сохранением стейта).
   - Встроить шаг обработки в `apply_effects()` перед финальным энкодингом в WAV:
     ```rust
     // PCM f32 samples -> DeepFilterNet processing frame-by-frame -> PCM f32 enhanced
     ```

4. **`src-tauri/src/commands/audio.rs`** (или новые команды)
   - Tauri команды для изменения настроек шумоподавления и их сохранения в `AppSettings`.

---

## 2.2 Frontend (Vue / TypeScript)

1. **`src/types/settings.ts`**
   - Обновить `AudioEffects` интерфейс, добавив поля:
     - `enhance_enabled: boolean`
     - `enhance_atten_db: number`

2. **`src/components/tts/AudioEffectsPanel.vue`**
   - Добавить секцию настроек улучшения голоса с помощью DeepFilterNet:
     - Переключатель (Toggle) для включения/выключения очистки.
     - Слайдер «Мягкость / Глубина очистки» (0–100%). Рекомендуемое дефолтное значение — 30-50% (что соответствует 10-15 dB лимиту подавления, давая натуральное звучание TTS без металлических искажений).

---

## 3. Критерии приемки

1. Проект успешно собирается (`cargo check` + `npx vue-tsc --noEmit`).
2. В секции эффектов TTS панели появляется новый блок настройки DeepFilterNet.
3. При генерации голоса и включенном режиме очистки аудио проходит обработку через встроенную модель DeepFilterNet в памяти.
4. Отсутствует заметный лаг (более 100 мс) перед стартом воспроизведения при включенном шумодаве.
5. Настройки сохраняются и считываются корректно при перезапуске приложения.
