# ROADMAP-033 — Воспроизведение фраз из аудиокеша

**Статус:** `completed` — повтор из аудиокеша реализован

## Цель

Добавить в историю фраз быстрый повтор уже синтезированного аудио без нового TTS-запроса.

## Зафиксированные решения

- Отдельный UI-индикатор наличия кеша не нужен.
- По нажатию кнопки backend пытается воспроизвести кеш.
- Если кеш отсутствует, UI показывает локальную ошибку/плашку.
- В истории сохраняются провайдер и голос; язык не сохраняется.
- Одинаковый текст может появляться несколькими записями, если различаются провайдер, голос или аудиоверсия.
- Нельзя полагаться на проверку через предварительный `ls` папки: проверяется конкретный кеш при воспроизведении.

## Текущее состояние

`PlaybackManager` уже держит memory-cache последних 20 фраз, но он живёт только до перезапуска и идентифицируется эфемерным playback UUID. Для истории нужен стабильный persistent cache key и сохранение готового аудио/метаданных.

## Риски

- Кеш должен учитывать параметры, влияющие на результат: текст после preprocessing, provider, voice и применённые audio effects.
- Старые `phrase_history.json` должны загружаться без миграционной ошибки.
- Ошибка cache miss не должна запускать новый TTS автоматически.
