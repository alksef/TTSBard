## Контекст
Stage 08 (офлайн-орфография) предлагает движок **`spellbook`** (нативный Rust-порт Nuspell от
helix-editor) + Hunspell-словари `ru.aff`/`ru.dic`. Но `spellbook` — **alpha-крейт**, и
критический вопрос — стабильность **`suggest()`** на русском. Если `suggest` слабый/багует,
весь подход A (бэкенд) меняется на fallback C (`nuspell-wasm` во фронте) — а это другая
архитектура.

**Этот spike — ответить «взлетит ли spellbook на русском suggest» ДО написания плана
офлайн-орфографии.** Spike = малая, изолированная, выбрасываемая после ответа задача.

## Цели spike
1. Заставить `spellbook` загрузить русский Hunspell-словарь (`ru.aff` + `ru.dic`).
2. Проверить `check()` (корректные слова = correct, опечатки = incorrect) на русском.
3. **Главное:** проверить `suggest()` — для типовых русских опечаток вернуть осмысленные
   варианты исправления.
4. Оценить: вес словаря в памяти, скорость проверки, стабильность (panic?).
5. **Решение:** подходит ли spellbook, или нужен fallback (nuspell-wasm / другой крейт).

## Что сделать (изолированно, НЕ в основном коде)

### 1. Изолированный cargo-проект (временный)
Создать временную директорию (НЕ в `src-tauri/` — spike не должен загрязнять основной проект):
```
spike-spellbook/
  Cargo.toml   # spellbook + зависимости
  src/main.rs  # демо
  dict/ru.aff  # русский Hunspell-словарь
  dict/ru.dic
```

### 2. Словари
Скачать `ru.aff` + `ru.dic` (пакет `hunspell-ru`, источник — Debian/Freedesktop). ~1-3 МБ.
Закладывает оценку веса словаря в бандле (риск из stage 08 + review-019).

### 3. Демо `src/main.rs`
```rust
use spellbook::*; // точный API — проверить по docs.rs/spellbook на момент spike

fn main() {
    // 1. Загрузка словаря
    let dict = Dictionary::load_aff("dict/ru.aff")
        .and_then(|aff| {
            let mut d = aff;
            d.load_dic("dict/ru.dic")?;
            Ok(d)
        })
        .expect("failed to load ru dictionary");

    // 2. check() — корректные слова
    for word in &["привет", "мир", "тестирование", "JavaScript"] {
        println!("check({word}) = {}", dict.check(word));
    }
    // 3. check() — опечатки (должны быть false)
    for word in &["првиет", "миор", "тстирование"] {
        println!("check({word}) = {}", dict.check(word));
    }

    // 4. suggest() — ГЛАВНОЕ
    for word in &["првиет", "миор", "тстирование", "програмирование"] {
        let sugg = dict.suggest(word);
        println!("suggest({word}) = {:?}", sugg);
    }
}
```
> Точный API `spellbook` (имена методов `check`/`suggest`, сигнатура загрузки) — **уточнить
> по docs.rs/spellbook на момент spike**, крейт alpha, API может отличаться от псевдокода выше.

### 4. Критерии прохода/провала
**SPIKE ПРОЙДЕН (→ подход A, бэкенд spellbook):**
- Словарь грузится без panic.
- `check()` корректно отличает русские слова от опечаток.
- **`suggest()` возвращает осмысленные варианты** для ≥3 из 4 тестовых опечаток
  (например, для «првиет» → «привет» в топе).
- Словарь в памяти ≤ ~50 МБ, проверка 100 слов < 100 мс.

**SPIKE ПРОВАЛЕН (→ fallback C, nuspell-wasm во фронте, или крейт zspell):**
- Panic при загрузке/проверке, ИЛИ
- `suggest()` возвращает мусор/пусто для большинства опечаток, ИЛИ
- Вес/скорость неприемлемы.
