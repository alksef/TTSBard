# Plan 73: Гибридное автодополнение продолжения (слои: без AI + AI)

**Дата:** 2026-06-28
**Тип:** Feature plan (общий)
**Статус:** Draft (код пишет DeepSeek)
**Stage-исследование:** `docs/stage/03-text-completion-without-ai.md`
**Зависит от:** `72-local-history-autocomplete` (слой 0 — Trie слов)
**Связано:** `PROBLEMS.md:57,74` (AI-автодополнение, Claude API)

## Цель
Поверх автокомплита слов (план 72) добавить **продолжение фразы**: базовый слой без AI
(n-граммы по личной истории) + опциональный AI-слой (Claude API, через существующий
`ai` модуль / `AiProvider`). Гибрид — по рекомендации stage-исследования.

## Решено пользователем
- Архитектура: **гибрид со слоями**.
- Без-AI слой: **n-граммы/Марков** из личной истории пользователя (без внешнего корпуса).
- AI-слой: опционально, через существующую AI-инфраструктуру.
- Слои объединяются в одном autocomplete-попапе CM6.

## Архитектура слоёв
- **Слой 0 — Trie слов** (план 72): всегда включён, оффлайн.
- **Слой 1 — n-граммы/Марков** (этот план): «следующее слово» / короткое продолжение для
  типовых реплик. Строится из истории ввода; оффлайн, без AI.
- **Слой 2 — AI** (этот план): продолжение произвольного текста через Claude API, по
  отдельному триггеру/настройке (`editor.ai_completion`).

## Область изменений (общая)

### Backend (Rust)
1. **n-граммная модель:** модуль, накапливающий биграммы/триграммы из истории
   (`record_history` из плана 72 расширяется).
2. **Команды:** `get_phrase_completion(prefix/context) -> Vec<Suggestion>` (слой 1).
3. **AI-продолжение:** через существующий `ai` модуль (`AiProvider`, `AppState.ai_client`) —
   команда `get_ai_completion(context)` с настройкой `editor.ai_completion` и проверкой
   наличия API-ключа (как в `docs/plans/55-...ai-correction`).
4. **Настройки:** расширить `EditorSettings` (`editor.ai_completion: bool`).

### Frontend (Vue/TS)
1. **Composable:** `useTextCompletion.ts` — слой 1 (n-граммы) + слой 2 (AI) с debounce и
   индикатором загрузки для AI.
2. **Autocomplete CM6:** попап объединяет слои — точные слова (слой 0), n-граммные
   продолжения (слой 1), AI-варианты (слой 2, отдельная группа/иконка).
3. **Настройки UI:** тумблер AI-продолжения в панели настроек (по образцу AI-коррекции).

## Критерии приёмки
- После ввода частых фраз n-граммный слой предлагает правдоподобное следующее слово/фразу.
- AI-слой работает при включённой настройке и наличии ключа; без ключа graceful-fallback
  на слои 0–1.
- Один попап, понятная визуальная разница между слоями.
- Оффлайн-режим (слой 1) не требует сети.

## Риски / заметки
- Качество без-AI продолжения на русском ограничено морфологией (см. stage 03) — ожидаемо
  для коротких типовых реплик.
- Не дублировать логику с AI-коррекцией (`docs/plans/55-...`) — переиспользовать `ai` модуль.
