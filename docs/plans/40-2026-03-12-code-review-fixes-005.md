## Задачи

### Критичные (CRITICAL)

#### 1. Неограниченный рост памяти в потоках аудиоплеера
- **Файл:** `src-tauri/src/audio/player.rs:69-91`
- **Проблема:** `play_mp3_async_dual` создаёт потоки без join — при интенсивном использовании TTS количество потоков растёт неограниченно
- **Решение:**
  - Сохранять JoinHandles в Vec и присоединять старые потоки перед запуском новых
  - Или использовать thread pool

#### 2. Блокирующие операции в async контексте
- **Файл:** `src-tauri/src/event_loop.rs:137-144`
- **Проблема:** `process_text_ready` создаёт новый tokio Runtime на каждое событие
- **Решение:**
  - Использовать общий Runtime созданный при запуске приложения
  - Или spawn задачу через `tokio::spawn` на существующем runtime

#### 3. unwrap() на слайсе API ключа
- **Файл:** `src-tauri/src/tts/openai.rs:104`
- **Проблема:** Срез API ключа без проверки границ — паника если ключ короче 11 символов
- **Решение:**
  - Использовать `get()` для безопасного среза: `&self.api_key[..7.min(self.api_key.len())]`

### Некритичные (MINOR)

#### 4. Избыточные клоны в управлении состоянием
- **Файл:** `src-tauri/src/state.rs:318-328`
- **Проблема:** Методы клонируют всю конфигурацию вместо чтения конкретных полей
- **Решение:**
  - Читать конкретные поля внутри области блокировки RwLock

#### 5. Несогласованный паттерн обработки ошибок
- **Файл:** `src-tauri/src/commands/mod.rs:157`
- **Проблема:** Некоторые команды возвращают `Result<(), String>`, другие используют `AppError`
- **Решение:**
  - Стандартизировать на `AppError` во всех командах

#### 6. Потенциальная паника в операциях с каналами
- **Файл:** `src-tauri/src/state.rs:146-161`
- **Проблема:** `emit_event` игнорирует ошибки отправки без логирования
- **Решение:**
  - Добавить warning логи при неудачной отправке в канал

#### 7. Потенциальная утечка памяти в event listeners (Web)
- **Файл:** `src/components/TtsPanel.vue:227`
- **Проблема:** `listen()` создаёт listener, cleanup может не сработать при ошибке
- **Решение:**
  - Сохранять cleanup функцию и гарантировать её вызов

#### 8. Тип возвращаемого значения setTimeout (Web)
- **Файл:** `src/components/TtsPanel.vue:65`
- **Проблема:** setTimeout приводится к unknown as number
- **Решение:**
  - Использовать `ReturnType<typeof setTimeout>`

### Оптимизации (OPTIMIZE)
