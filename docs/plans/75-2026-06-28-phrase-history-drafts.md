# Plan 75: История фраз как черновики (извлечение в редактор)

**Дата:** 2026-06-28
**Тип:** Feature plan (общий)
**Статус:** Draft (код пишет DeepSeek)
**Stage-исследование:** `docs/stage/05-phrase-history.md`
**Зависит от:** `71-codemirror-editor` (редактор), `72-local-history-autocomplete` (`HistoryManager`)
**Связано:** `74-playback-control-window` (там — мгновенный повтор 5 фраз; здесь — извлечение в редактор)

## Цель
Дать пользователю **достать ранее введённую фразу целиком** из журнала и загрузить её в
редактор для правки перед отправкой — чтобы не вводить текст заново.

> Отличие от плана 74: там «5 недавних» для **мгновенного повтора** (озвучить как есть) в окне
> управления воспроизведением. Здесь — **извлечение в редактор** для правки/переиспользования.

## Решено пользователем
- **Единое хранилище всех фраз** (`phrase_history.json`, persistent) — единственный источник
  правды. «5 последних» (план 74) и полный список (этот план) — это **views из одного
  хранилища**, не два журнала.
- **UX:** **Вариант 2c** — список фраз внутри InputPanel (боковая панель/выпадение), клик →
  загрузить фразу в редактор.
- **Дедупликация:** схлопывать одинаковые (incr count, обновлять `lastUsed`).
- **Поиск по подстроке** в списке — обязательно.
- **Триггер записи:** `TextSentToTts` (отправка).

## Область изменений (общая)

### Backend (Rust)
1. **Единое хранилище фраз** — расширить `HistoryManager` (`src-tauri/src/history.rs`): slice
   `Vec<PhraseEntry { id, text, count, lastUsed/timestamp }>`, persistence в
   `phrase_history.json` (по образцу `input_history.json`/`ngrams.json`). **Это единственный
   журнал фраз в приложении** — план 74 берёт отсюда «5 последних».
   - Методы: `record_phrase(text)` (с дедупликацией), `get_phrases(filter: Option<String>,
     limit)` (фильтр по подстроке, сортировка по `lastUsed` desc; **тот же метод** отдаёт
     «5 последних» при `limit=5` без фильтра), `delete_phrase(id)`, `clear_phrases()`.
   - Все блокировки `parking_lot`, ошибки `AppError`/`Result<T,String>`, **без `.expect()`**
     (уроки ревью 71-73); запись на диск off-thread (`spawn`/debounce, см. паттерн плана 72).
2. **Команды Tauri** (`commands/history.rs` или новый `commands/phrase_history.rs`):
   `get_phrase_history(filter, limit)` (используется и планом 74 с `limit=5`),
   `delete_phrase(id)`, `clear_phrase_history()`. (`record_phrase` вызывается из бэкенда — п.3.)
   Регистрация в `invoke_handler`.
3. **Триггер записи:** при `TextSentToTts` (отправка фразы в TTS, `commands/mod.rs`) вызывать
   `record_phrase(text)` в единое хранилище.

### Frontend (Vue/TS)
1. **Компонент `PhraseHistoryList.vue`** внутри `InputPanel` (рядом с `TtsEditor`):
   - свёрнут/развёрнут по кнопке (иконка) или хоткею;
   - текстовый фильтр по подстроке (debounce);
   - список: превью текста + timestamp/счётчик; клик → загрузить в редактор;
   - опционально: удаление одной фразы, «очистить всё».
2. **Загрузка в редактор:** клик по фразе → `TtsEditor` заменяет документ. Если в редакторе
   есть несохранённый текст — подтверждение (не затирать молча).
3. **Composable** `usePhraseHistory.ts` — обёртка над invoke (get/delete/clear + debounce
   фильтра), по образцу `useInputHistory.ts`.

## Критерии приёмки
- Отправленная фраза появляется в списке (с дедупликацией).
- Поиск по подстроке фильтрует список.
- Клик по фразе загружает её в редактор (с подтверждением при затирании).
- История переживает перезапуск.
- Список сворачивается/разворачивается; не мешает основному вводу.
- `vue-tsc` / `cargo check` проходят (0 warnings).

## Не входит
- Мгновенный повтор (replay) — план 74.
- Autocomplete-слой «recent» (2a) / command-palette (2b) — отдельная тема, если понадобится.
- Хранение голоса/провайдера с фразой — пока не нужно (повтор тем же голосом — план 74).

## Риски / заметки
- **Единый источник фраз:** `phrase_history.json` — единственный журнал. План 74 НЕ заводит
  свой; он запрашивает «5 последних» через `get_phrases(limit=5)`. Если в реализации плана 74
  уже создан отдельный journal 5 недавних — **объединить** в это хранилище (см. ревью плана 74).
- Фильтр/сортировка на стороне Rust (как в плане 72 для слов), чтобы не тянуть весь журнал на фронт.
