## Приоритет 1: Критичные проблемы (CRITICAL)

### 1.1. Избыточная блокировка мьютексов в SettingsManager
**Проблема**: Каждый метод `SettingsManager` вызывает `load()`, который блокирует, читает файл, парсит JSON. При частых вызовах это создает избыточную нагрузку.
**Файлы**: `src-tauri/src/config/settings.rs:312-528`
**Задачи**:
- [ ] Добавить кэширование настроек в памяти с инвалидацией при изменении
- [ ] Использовать `RwLock` для read-heavy workload
- [ ] Оптимизировать getter методы для чтения из кэша

### 1.2. Потенциальная гонка при сохранении настроек
**Проблема**: Методы `save_twitch_settings` и `save_webview_settings` сначала сохраняют в `AppState`, затем в файл. Между этими операциями другой поток может прочитать устаревшие данные.
**Файлы**: `src-tauri/src/commands/twitch.rs:37-46`, `src-tauri/src/commands/webview.rs:32-47`
**Задачи**:
- [ ] Использовать транзакционный подход - atomic update
- [ ] Сначала сохранять в файл, потом обновлять память
- [ ] Обеспечить согласованность данных между источниками
