# Design: Telegram Авторизация для Silero TTS

## Обзор

Добавляем интеграцию с Telegram ботом @silero_voice_bot для генерации речи через Silero TTS провайдер. Пользователь авторизуется через телефон+код, сессия сохраняется локально в SQLite.

## Требования

1. **UI Компоненты**
   - Модальное окно авторизации (3 состояния)
   - Кнопка в настройках Silero TTS
   - Красная плашка при ошибках подключения
   - Инфо-текст в настройках о формате голосовых

2. **Поток авторизации**
   - Пользователь вводит: телефон, API ID, API Hash
   - Запрос кода подтверждения
   - Ввод кода → подключение
   - Сессия сохраняется в %APPDATA%

3. **Хранение данных**
   - SQLite сессия (grammers-session)
   - Путь: `%APPDATA%\app-tts-v2\telegram.session`

## UI Компоненты

### Модальное окно авторизации (TelegramAuthModal)

**Состояние 1 - Форма входа:**
- Поле: Номер телефона (placeholder: "+79991234567")
- Поле: API ID (число)
- Поле: API Hash (строка)
- Ссылка: "Как получить API данные?" → https://my.telegram.org/apps
- Кнопка: "Получить код"

**Состояние 2 - Ввод кода:**
- Текст: "Код отправлен на {номер}"
- Поле: Код из SMS/Telegram
- Кнопка: "Подтвердить"

**Состояние 3 - Подключено:**
- Аватар пользователя и имя
- Текст: "Подключено как @username"
- Кнопка: "Отключить"

### Плашка Silero TTS

**Обычное состояние:** Серый/белый фон

**Состояние ошибки:** Красный фон + текст проблемы + кнопка "Исправить"

### Настройки

Инфо-текст: "Для работы Silero TTS убедитесь, что в боте включены формат голосовых"

## Tauri Commands

- `telegram_init(api_id, api_hash, phone)` - инициализация
- `telegram_request_code()` - запрос кода
- `telegram_sign_in(code)` - ввод кода
- `telegram_sign_out()` - выход
- `telegram_get_status()` - статус подключения
- `telegram_get_user()` - данные пользователя

## Обработка ошибок

При ошибке подключения:
- Красная плашка Silero TTS
- Модальное окно с описанием проблемы
- Кнопки действий: "Попробовать снова", "Отключить"
