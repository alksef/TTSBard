## Приоритет 1: Критичные проблемы (CRITICAL)

### 1.1. Небезопасная статическая мутабельная переменная в SoundPanel hook ✅
**Проблема**: `static mut SP_HOOK_STATE` используется в `soundpanel/hook.rs:24` без синхронизации. Создаёт риск data races и undefined behavior.
**Файлы**: `src-tauri/src/soundpanel/hook.rs:24, 104`
**Риск**: HIGH - undefined behavior, data races
**Задачи**:
- [x] Заменить `static mut SP_HOOK_STATE` на `std::sync::OnceLock<Arc<SoundPanelState>>`
- [x] Проверить все unsafe блоки на наличие race conditions
- [ ] Добавить тесты для конкурентного доступа к состоянию хука
- [x] Убедиться, что Windows хуки выполняются корректно с новой синхронизацией

### 1.2. Race condition при обновлении настроек в SoundPanelState ✅
**Проблема**: Методы `set_floating_opacity` и `set_floating_bg_color` блокируют mutex, отправляют событие (которое может попытаться заблокировать тот же mutex), создают потенциал для deadlock.
**Файлы**: `src-tauri/src/soundpanel/state.rs:152-161, 169-178`
**Риск**: MEDIUM - возможный deadlock при быстром изменении настроек
**Задачи**:
- [x] Клонировать значения перед отправкой события
- [x] Отправлять события после освобождения mutex (drop guard)
- [ ] Добавить тесты для быстрого последовательного изменения настроек
- [ ] Рассмотреть использование async channel для событий
