# Plan 77: Вкладки для параллельного ввода разных текстов (Editor Tabs)

**Дата:** 2026-06-30
**Статус:** draft (для реализации через DeepSeek по WORKFLOW)
**Связано:** stage `docs/stage/06-editor-tabs-multiple-texts.md`,
арх-ревью `docs/reviews/review-019-2026-06-30.md`, план 75 (drafts — отдельная сущность),
план 71 (CodeMirror).

## Контекст
Фронтенд-only фича: строка табов над редактором, каждый таб — независимый текст.
Переключение мгновенное, без потери содержимого. Persist — **только в памяти сессии**
(бэкенд НЕ трогается; осознанное решение, см. stage 06 KEY_DECISIONS). Это **отдельная
сущность от drafts** (план 75 = журнал произнесённого, persistent; табы = рабочие черновики,
in-memory). Без merge.

## Цели
1. Несколько независимых текстов, переключение табами без потери содержимого.
2. CodeMirror **не пересоздаётся** при переключении — используется существующий
   `watch(modelValue)` в `TtsEditor.vue`, который заменяет документ.
3. Тема — только CSS-vars (light/dark автоматически, как везде).
4. Корректная работа в minimal-mode.

## Архитектура

### 1. Composable `src/composables/useEditorTabs.ts` (НОВЫЙ)
```ts
import { ref, computed } from 'vue'

export interface EditorTab {
  id: string
  title: string
  text: string
}

function genId(): string {
  // crypto.randomUUID доступен в Tauri WebView; fallback на счётчик для безопасности.
  if (typeof crypto !== 'undefined' && 'randomUUID' in crypto) {
    return crypto.randomUUID()
  }
  return `tab-${Date.now()}-${Math.random().toString(36).slice(2, 8)}`
}

export function useEditorTabs() {
  const tabs = ref<EditorTab[]>([{ id: genId(), title: 'Текст 1', text: '' }])
  const activeId = ref<string>(tabs.value[0].id)

  const active = computed<EditorTab>({
    get: () => tabs.value.find(t => t.id === activeId.value) ?? tabs.value[0],
    set: (v) => {
      const t = tabs.value.find(t => t.id === activeId.value)
      if (t) Object.assign(t, v)
    },
  })

  function create(): string {
    const n = tabs.value.length + 1
    const tab: EditorTab = { id: genId(), title: `Текст ${n}`, text: '' }
    tabs.value.push(tab)
    activeId.value = tab.id
    return tab.id
  }

  function close(id: string) {
    const idx = tabs.value.findIndex(t => t.id === id)
    if (idx === -1) return
    tabs.value.splice(idx, 1)
    if (tabs.value.length === 0) {
      // Не оставлять пустое состояние — всегда ≥1 таб.
      const tab: EditorTab = { id: genId(), title: 'Текст 1', text: '' }
      tabs.value.push(tab)
      activeId.value = tab.id
      return
    }
    if (id === activeId.value) {
      // Выбрать соседа (предпочтительно предыдущий, иначе следующий).
      const next = tabs.value[Math.min(idx, tabs.value.length - 1)]
      activeId.value = next.id
    }
  }

  function select(id: string) {
    if (tabs.value.some(t => t.id === id)) activeId.value = id
  }

  function rename(id: string, title: string) {
    const t = tabs.value.find(t => t.id === id)
    if (t) t.title = title
  }

  return { tabs, activeId, active, create, close, select, rename }
}
```
**Синглтон vs инстанс:** `useEditorTabs()` вызывается один раз в `InputPanel.vue` (один
InputPanel в приложении, см. `App.vue`). Локальный инстанс достаточен; module-level singleton
НЕ нужен (избегаем глобального состояния, как в `useErrorHandler` — там singleton оправдан
shared-toast, здесь нет).

### 2. Компонент `src/components/editor/EditorTabs.vue` (НОВЫЙ)
- Строка табов над `.textarea-wrapper` в `InputPanel.vue`.
- Props: `tabs`, `activeId`. Emits: `create`, `close`, `select`, `rename`.
- Каждый таб: `title` (дабл-клик → inline `<input>` для переименования) + кнопка `×`.
- Активный таб подсвечен `--color-accent`.
- Кнопка `+` — создать таб (emit `create`).
- Тема — только CSS-vars (`--color-bg-elevated`, `--color-border`, `--color-accent`,
  `--color-text-primary`, `--color-text-muted`).
- Minimal-mode: `overflow-x: auto` на строке табов; сама строка видна (в отличие от
  `correct-button`, который скрыт `v-if="!isMinimalMode"` — табы нужны и в minimal, это
  ключевая фича). Узкие табы скроллятся по горизонтали.

### 3. Связка с `InputPanel.vue`
```ts
import { useEditorTabs } from '../composables/useEditorTabs'
import { ref as vueRef, nextTick } from 'vue'

const { tabs, activeId, active, create: createTab, close: closeTab, select: selectTab, rename: renameTab } = useEditorTabs()

// text становится computed-прокси к активному табу.
// TtsEditor v-model="text" остаётся БЕЗ ИЗМЕНЕНИЙ.
const text = computed<string>({
  get: () => active.value.text,
  set: (v) => { active.value.text = v },
})

const editorRef = vueRef<InstanceType<typeof TtsEditor> | null>(null)

async function onCreate() {
  createTab()
  await nextTick()
  editorRef.value?.focus()
}
async function onSelect(id: string) {
  selectTab(id)
  await nextTick()
  editorRef.value?.focus()
}
```
В шаблон `InputPanel.vue`, **над** `.textarea-wrapper` (внутри `.input-group`):
```vue
<EditorTabs
  v-if="!isMinimalMode || true"
  :tabs="tabs"
  :active-id="activeId"
  @create="onCreate"
  @close="closeTab"
  @select="onSelect"
  @rename="renameTab"
/>
<div class="textarea-wrapper" :class="{ 'minimal-wrapper': isMinimalMode }">
  <TtsEditor ref="editorRef" v-model="text" ... />
  ...
</div>
```
(`v-if` всегда true — табы видны везде; условие оставлено как заметка, что minimal НЕ скрывает табы.)

### 4. Фокус-менеджмент
- `select`/`create` → `nextTick(() => editorRef.value?.focus())`. `TtsEditor` уже экспонирует
  `focus()` (`TtsEditor.vue:341`).
- Нужна `ref` на `TtsEditor` (`editorRef`).

## Риски и решения (из stage 06 + review-019)
1. **Сброс курсора/скролла при переключении таба** — `TtsEditor.watch` заменяет весь документ,
   курсор уходит в начало. **Решение для этого плана:** принять как ожидаемое поведение
   (зафиксировать в комментарии). Сохранение `cursorPos`/`scrollTop` в `EditorTab` —
   **отдельный будущий этап** (не в этот план, избегаем переусложнения).
2. **Minimal-mode** — узкое окно. **Решение:** `overflow-x: auto` на строке табов; табы видны
   в minimal (в отличие от correct-button). Узкие табы скроллятся.
3. **Закрытие последнего таба** — всегда ≥1 таб (см. `close()` выше: при пустоте создаёт новый).
4. **Закрытие таба с несохранённым текстом** — **БЕЗ подтверждения** (in-memory, текст не
   критичен; соответствует «только в памяти сессии»).
5. **Quick-editor mode** (`handleEnter` очищает `text.value`) — с computed-прокси очистит
   активный таб. **Корректно**, зафиксировать в комментарии `handleEnter`.
6. **Дублирование с drafts (план 75)** — разные сущности, без merge. Визуально разделены:
   табы сверху, drafts снизу (`PhraseHistoryList`). Tooltip на строке табов: «Рабочие черновики
   (не сохраняются)» — чтобы пользователь не путал с persistent-историей.

## Критерии готовности
- `useEditorTabs.ts`: CRUD (create/close/select/rename), всегда ≥1 таб.
- `EditorTabs.vue`: строка табов, переименование (дабл-клик), закрытие, кнопка `+`, тема
  CSS-vars, minimal-mode (overflow-x).
- `InputPanel.vue`: computed-прокси `text`, ref на editor, фокус на select/create; табы
  монтируются над `.textarea-wrapper`.
- Создание/переключение/закрытие табов не теряет текст активного таба; CodeMirror не
  пересоздаётся (используется существующий `watch(modelValue)`).
- `npx vue-tsc --noEmit` и `cargo check` — 0 ошибок, 0 warnings. (Бэкенд НЕ трогается —
  `cargo check` изменений не потребует, но проверить, что фронт-правки ничего не сломали
  косвенно.)
- Тема: light/dark корректны через CSS-vars.
