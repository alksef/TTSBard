# Plan 118: Тестирование авторизации WebView роутера (loopback, private и public IP)

**Дата:** 2026-07-11  
**Статус:** Запланировано к реализации  
**Источник:** architecture-review-2026-07-11 (High — WebView)

---

## 1. Проблема
В `src-tauri/src/webview/server.rs` реализована логика авторизации для страницы `/`, эндпоинта `/auth` и SSE `/sse`:
- Для локальных сетей (loopback и private IP) доступ разрешён без токена.
- Для внешних IP-адресов требуется токен (из cookie `webview_auth` или параметров запроса).

В коде нет интеграционных тестов, проверяющих это поведение. Нужны тесты на уровне роутера Axum, которые будут эмулировать запросы с разных `SocketAddr` и заголовков и проверять код ответа (`200 OK` или `401 Unauthorized`).

---

## 2. Предлагаемое решение
Добавить модуль тестов `#[cfg(test)] mod tests` в конец `src-tauri/src/webview/server.rs`.
В тестах мы будем:
1. Создавать тестовый экземпляр `ServerState` с настроенным токеном доступа (например, `Some("test-token".to_string())`).
2. Собирать роутер Axum:
   ```rust
   let app = Router::new()
       .route("/", get(index))
       .route("/auth", get(auth_handler))
       .route("/sse", get(sse_handler))
       .with_state(state);
   ```
3. Использовать `tower::ServiceExt` для отправки запросов напрямую в роутер с помощью метода `.oneshot()`.
4. Эмулировать IP-адрес клиента с помощью расширения `ConnectInfo(SocketAddr)`.

Мы напишем следующие тестовые сценарии:
- **Локальный доступ (Loopback IP / 127.0.0.1)**: запрос на `/` без токена должен вернуть `200 OK`.
- **Локальный доступ (Private IP / 192.168.1.50)**: запрос на `/` без токена должен вернуть `200 OK`.
- **Внешний доступ (Public IP / 8.8.8.8) без токена**: запрос на `/` должен вернуть `401 Unauthorized`.
- **Внешний доступ (Public IP / 8.8.8.8) с правильным токеном в Query**: запрос на `/?token=test-token` должен вернуть `200 OK` и выдать cookie `webview_auth`.
- **Внешний доступ (Public IP / 8.8.8.8) с правильным токеном в Cookie**: запрос на `/` с заголовком `Cookie: webview_auth=test-token` должен вернуть `200 OK`.
- **Внешний доступ (Public IP / 8.8.8.8) с неверным токеном**: запрос должен вернуть `401 Unauthorized`.

---

## 3. Критерии готовности
- Все тесты проходят успешно на целевой платформе при запуске `cargo test --package ttsbard --lib webview::server::tests` (или эквивалентном).
- Логика авторизации роутера полностью покрыта проверками.
