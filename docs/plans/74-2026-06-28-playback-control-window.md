# Plan 74: Очередь воспроизведения и плавающее окно управления

**Дата:** 2026-06-28
**Тип:** Feature plan (общий)
**Статус:** Draft (код пишет DeepSeek)
**Stage-исследование:** `docs/stage/04-playback-control-floating-window.md`
**Зависит от:** `72-local-history-autocomplete` (phrase_history для 5 недавних фраз — частично;
  см. также `docs/stage/05-phrase-history.md`)
**Из research:** `docs/research/01-...` → идея **A** (Speech Queue + Interrupt)

## Цель
Добавить **очередь фраз** и **плавающее окно управления воспроизведением**, всегда видимое
поверх всех окон, с паузой/остановкой, повтором и списком последних фраз.

## Решено пользователем
- **Вариант 1** — PlaybackManager (Rust) + новое плавающее окно `playback-control`.
- Окно показывается **всегда**, пока приложение запущено.
- **5 последних фраз** в окне; клик → повторно озвучить.
- **Оба replay:** 🔁 текущей фразы с начала + клик по фразе в списке.
- **Пауза = настоящая пауза** (`Sink::pause()`, позиция помнится) → **Resume** продолжает
  **с той же фразы и позиции**, доигрывает её, затем очередь идёт дальше.
- **Стоп = остановить текущую фразу** (`Sink::stop()`), **очередь держать** (не очищать).
- **Очередь видна:** если после текущей фразы что-то ожидает — показывать очередь.

## Область изменений (общая)

### Backend (Rust)
1. **PlaybackManager** — персистентный модуль `playback/`, замена локального `AudioPlayer::new()`
   в `speak_text_internal` (`commands/mod.rs:245`). Ключевое отличие от текущего `AudioPlayer`:
   **Sink сохраняется** (не пересоздаётся), чтобы поддержать настоящий pause/resume. Хранит:
   - текущий `rodio::Sink` (для pause/play/seek/stop) + статус (Idle/Playing/Paused/Stopped);
   - очередь `VecDeque<QueuedPhrase>` (фраза + кеш аудио + метаданные);
   - **кеш последних аудио** для replay фраз из истории (без пересинтеза).
2. **События** (расширить `AppEvent`): `PlaybackStarted{text}`, `PlaybackFinished`,
   `PlaybackPaused`, `PlaybackResumed`, `PlaybackStopped`, `QueueChanged{items}`. Сейчас конца
   воспроизведения нет — добавить.
3. **Команды Tauri:**
   - `playback_pause()` → `Sink::pause()` (позиция помнится, статус Paused);
   - `playback_resume()` → `Sink::play()` → доиграть текущую до конца → затем очередь дальше;
   - `playback_stop()` → остановить текущую (`Sink::stop()`), **очередь держать**;
   - `playback_repeat()` → 🔁 текущей с начала (`try_seek(0)` на живом Sink, либо перевыгрузка);
   - `replay_phrase(id)` → из 5 недавних (через кеш аудио, без пересинтеза);
   - `get_playback_state()` → статус + текущая + очередь + 5 недавних фраз.
4. **5 недавних фраз** — брать из phrase-history (план `05`/`72`); если её persistent-части
   ещё нет — добавить минимальный journal отправленных фраз (`phrase_history.json`).
5. **Hotkeys:** добавить в `HotkeySettings` `playback_pause` (toggle pause/resume),
   `playback_stop`, `playback_repeat` (по образцу `sound_panel`).

### Frontend
1. **Новое окно `playback-control`** — запись в `tauri.conf.json` (label `playback-control`,
   как soundpanel: decorations:false, alwaysOnTop:true, skipTaskbar:true, transparent:true,
   visible:**true** по умолчанию, exclude-from-capture).
2. **Отдельный Vue-app `src-playback/`** (entry index.html + main.ts + App.vue, по образцу
   `src-soundpanel/`).
3. **UI окна:** текущая фраза + кнопки (⏸ пауза/⏹ стоп/🔁 repeat) + список очереди (если есть)
   + список 5 недавних фраз (клик → replay).
4. События: слушать `playback-*`, обновлять состояние реактивно.

## Поведение очереди (ключевая семантика)
- Новая фраза при `Playing`/`Paused` → в конец очереди (не прерывает).
- `PlaybackFinished` (естественный конец фразы) → если очередь непуста, играть следующую.
- **Пауза** → `Sink::pause()` (позиция помнится). **Resume** → `Sink::play()` → фраза
  доигрывает до конца → срабатывает `PlaybackFinished` → очередь идёт дальше. Т.е. пауза
  не «съедает» текущую фразу и не двигает очередь вручную.
- **Стоп** → `Sink::stop()` (текущая снята, необратимо для этого Sink), **очередь держать**.
  Продолжение — со следующей фразы очереди по явному действию (resume/next) или новой отправкой.

## Критерии приёмки
- Окно всегда видно поверх всех окон; тема/прозрачность по системе тем.
- **Пауза** замирает речь на позиции; **Resume** доигрывает ту же фразу и идёт к очереди.
- **Стоп** снимает текущую фразу, очередь остаётся.
- 🔁 проигрывает текущую фразу с начала.
- Клик по фразе в «5 недавних» → повторно озвучить (без пересинтеза — из кеша).
- Очередь видна в окне, когда есть ожидающие фразы.
- Hotkeys pause(toggle)/stop/repeat работают глобально.
- `vue-tsc` / `cargo check` проходят.

## Риски / заметки
- rodio `stop()` необратим для Sink — после стопа текущей нужен **новый Sink** для следующей
  фразы (PlaybackManager пересоздаёт Sink при запуске следующей). Пауза/resume работают только
  на живом Sink.
- Кеш аудио: ограничить размер (LRU, последние N), чтобы не раздувать память.
- Взаимное исключение окон: `ActiveWindow` сейчас `None`/`SoundPanel` — решить, нужно ли
  `ActiveWindow::Playback` (вероятно нет, т.к. окно не перехватывает клавиатуру).

## Не входит
- Persistent phrase-history как отдельная фича панели — план `05` (здесь только минимальный
  journal для «5 недавних», если ещё нет).
