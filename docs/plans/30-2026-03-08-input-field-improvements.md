# План: Доработка поля ввода

**Дата:** 2026-03-08
**Задача:** Улучшение UX поля ввода - убрать кнопку, добавить горячие клавиши, live replacement

**Статус:** ⏳ В РАБОТЕ

## Требования

1. **Убрать кнопку "Озвучить"** из UI
2. **Enter** → отправка на TTS
3. **Shift+Enter** → перенос строки (дефолтное поведение)
4. **Очистка поля** сразу после отправки (не дожидаясь проигрывания)
5. **Live replacement** препроцессора сразу после пробела (видимая замена)

## Выбранный подход

- Замена на стороне UI (без backend вызовов при вводе)
- Загрузка replacements.txt и usernames.txt при монтировании компонента
- Использование тех же файлов, что и в режиме перехвата (синхронизация)
- Курсор прыгает в конец при замене (простое решение)

### Шаг 4: InputPanel.vue - обработчик Enter

### Шаг 6: Немедленная очистка
- [ ] Изменить `speak()`: убрать `await` перед очисткой

## Ключевые файлы для изменения

- `src/components/InputPanel.vue` - основные изменения

## Пример работы live replacement

```
Ввод: \name [пробел]
Результат: Алекс [пробел]

Ввод: %john [пробел]
Результат: Джон Смит [пробел]

Если паттерн не найден - остаётся как есть
```

## Примечания

1. Курсор прыгает в конец textarea при замене (accept)
2. Замена видима для пользователя в реальном времени
3. Используются те же файлы, что и в режиме перехвата (синхронизация)
4. Enter без модификаторов = отправка, Shift+Enter = новая строка
