# Аудиоэффекты и постобработка звука

TTSBard применяет аудиоэффекты перед воспроизведением и позволяет проверить результат на WAV- или MP3-файле. Обработка выполняется в бэкенде на PCM-данных с использованием Signalsmith Stretch и DeepFilterNet.

## 1. Поддерживаемые эффекты

### 1.1 Высота тона

* Диапазон: `-100..100`, то есть до `±12` полутонов.
* Signalsmith Stretch меняет высоту тона независимо от темпа.

### 1.2 Темп

* Диапазон: `-100..100`, что соответствует `0.75×..1.50×`.
* Темп меняется без изменения высоты тона.
* Подписи шкалы: `0.75×`, `0.80×`, `0.90×`, `1.00×`, `1.10×`, `1.25×`, `1.30×`, `1.50×`.

### 1.3 Громкость

* Диапазон: `0..200%`, где `100%` — исходный уровень.
* Громкость применяется отдельно во время воспроизведения.

### 1.4 Сохранение тембра

Signalsmith Stretch поддерживает отдельный переключатель сохранения тембра. Он помогает менять высоту тона и темп без заметного эффекта «бурундука» и чрезмерного искажения голоса.

## 2. Нейросетевое улучшение речи (DeepFilterNet)

В приложение встроен нативный Rust-инференс DeepFilterNet 3. Модель работает в памяти процесса и используется для шумоподавления и смягчения артефактов синтеза.

Глубина очистки задаётся в диапазоне `5..30 dB`:

* `5..12 dB` — мягкая очистка, рекомендуется для естественного звучания;
* `15..20 dB` — более сильное подавление шума;
* `25..30 dB` — максимальная очистка, которая может добавить неестественные артефакты.

DeepFilterNet применяется первым, до изменения темпа и высоты тона. При ошибке во время обычного TTS используется исходный звук без очистки; при ошибке предпрослушивания показывается сообщение.

## 3. Проверка эффектов до сохранения настроек

В разделе **Аудио → Эффекты** можно выбрать WAV- или MP3-файл и прослушать два варианта:

* **Оригинал** — файл без аудиоэффектов;
* **С эффектами** — текущие значения ползунков и DeepFilterNet, включая ещё не сохранённые изменения.

Предпрослушивание выполняется через PCM-путь воспроизведения. Его можно остановить, заменить файл или очистить выбор. Кнопка **Отменить** возвращает последние сохранённые значения, а **Сохранить** применяет настройки к последующим TTS-фразам.

## 4. Актуальный pipeline

```text
входные аудиоданные TTS или выбранный WAV/MP3-файл
  → декодирование в interleaved f32 PCM
  → DeepFilterNet (если включён)
  → Signalsmith Stretch (tempo + pitch + formant correction)
  → передача готового PCM в playback pipeline
```

Поле `speed` в storage/API сохранено для совместимости, но пользовательская семантика — **tempo**. Регулятор громкости в этот backend pipeline не входит и применяется отдельно при воспроизведении.

Экспорт из редактора текста — отдельный режим: он сохраняет исходный аудиофайл, полученный от TTS-провайдера, без DeepFilterNet, темпа, высоты тона и громкости. Это позволяет получить исходник и отдельно проверять на нём разные варианты обработки.
