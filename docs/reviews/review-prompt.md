# Code Review Prompt

## Контекст
Разработчик работает с **Rust** (backend, Tauri) и **Web** (Vue.js, TypeScript, CSS).

## Критичные моменты для проверки

### Rust
- **Память и безопасность**: `.unwrap()`, `.expect()`, утечки памяти, unsafe код
- **Конкурентность**: мьютексы, deadlock, race conditions, `Arc<Mutex<>>`
- **Обработка ошибок**: `Result`, `?` оператор, propagating errors
- **Производительность**: избыточные клоны, аллокации, блокирующие операции

### Web
- **Reactivity**: корректное использование `ref`, `reactive`, `computed`
- **Lifecycle утечки**: `onUnmounted`, отписки от событий
- **Type Safety**: `any` типы, отсутствующие типы
- **State management**: мутации состояния, race conditions

### Интеграция Tauri
- **IPC**: правильная передача данных между Rust и Frontend
- **Сериализация**: ошибки `serde`
- **Блокирующие операции**: async/await в командах

### Общее
- **Security**: инъекции, валидация входных данных, secrets
- **Error handling**: пользовательские сообщения, логирование
- **Code duplication**: повторяющийся код
- **Единообразие**: соответствие подходов существующему коду, паттернам проекта

## Игнорировать
- **Хранение токенов**: Хранение токенов (Twitch, Telegram, OpenAI) в открытом виде допустимо
- **Логирование токенов**: Логирование API ключей (даже замаскированных) допустимо
- **Валидация токенов**: Валидация формата токенов на клиенте достаточна
