# DECISION-008 — JSON persistence

**Статус:** `accepted`

## Контекст

Объём настроек, вкладок и истории не требует базы данных, а файлы должны быть
переносимыми и доступными для диагностики.

## Решение

Хранить состояние в version-tolerant JSON под системным config directory
TTSBard. Записи выполняются под lock и заменяются атомарно; старые schemas
читаются через serde defaults/migrations.

## Последствия

SQLite не вводится без доказанной потребности. Corrupt config восстанавливается
через backup/defaults; чувствительные значения не логируются.
