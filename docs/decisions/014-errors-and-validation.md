# DECISION-014 — Ошибки и проверка

**Статус:** `accepted`

## Контекст

Внутренней диагностике нужен контекст, но frontend должен получать безопасные
ошибки. Не все Windows/audio/UI сценарии автоматизируются.

## Решение

В Rust используются contextual errors и structured `tracing`; IPC boundary
преобразует их в безопасное сообщение. Автотесты покрывают чистую логику и
persistence, runtime/UI дополняются ручными сценариями.

## Последствия

Checklist агента не считается верификацией. Проверки выбираются по риску;
ошибки не игнорируются без явного fallback и диагностического события.
