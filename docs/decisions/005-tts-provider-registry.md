# DECISION-005 — Registry TTS-провайдеров

**Статус:** `accepted`

## Контекст

Поддерживаются удалённые и локальные TTS-провайдеры, включая динамически
обнаруживаемые Piper-модели.

## Решение

Провайдеры реализуют общий контракт синтеза и регистрируются в
`TtsProviderRegistry` по стабильному id. Выбор активного provider отделён от
его реализации.

## Последствия

UI получает descriptors из registry. Новый provider не добавляет ветвление по
всему pipeline; warmup и availability остаются явными операциями.
