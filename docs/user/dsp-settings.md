# Практическая настройка DSP-постобработки

DSP в TTSBard — это лёгкая постобработка уже синтезированной речи. Она не восстанавливает достоверные детали, которых нет в исходнике, и не заменяет DeepFilterNet. Основная цель DSP — сделать голос чище по тональному балансу, ровнее по громкости и безопаснее по пикам.

Порядок обработки:

```text
DeepFilterNet → pitch/tempo → EQ → Compressor → Limiter
```

Проверять результат нужно кнопкой **Все эффекты** в панели «Проверка эффектов». Она использует текущие черновые значения, поэтому настройки можно прослушать до сохранения.

## Как настраивать

1. Выбрать короткий, но показательный файл: фраза с шипящими (`с`, `ш`, `ч`), взрывными согласными (`п`, `т`, `к`) и длинными гласными.
2. Сначала нажать **Оригинал**, затем **Все эффекты**.
3. Включать только один блок за раз.
4. После каждого изменения сравнивать не только громкость, но и дикцию, окончания слов и паузы.
5. Сохранить настройки только после A/B-сравнения на нескольких голосах.

Не стоит одновременно поднимать EQ, makeup compressor и громкость устройства: так легко принять более громкий сигнал за более качественный.

## EQ

EQ меняет частотный баланс. Он не должен заметно менять длительность или высоту голоса.

### Low Cut

Убирает низкочастотный гул, лишний бас и вибрации.

Что пробовать:

| Ситуация | Частота |
|---|---:|
| Чистый TTS без гула | выключить |
| Голос немного «бубнит» | 60–80 Hz |
| Сильная мутность или гул | 90–120 Hz |

Если голос стал тонким, сухим или потерял тело — частота слишком высокая.

Поле **Крутизна** задаёт желаемую крутизну среза. В текущей реализации DSP используется фиксированный мягкий low-cut второго порядка; поэтому это поле пока является заделом для будущей точной настройки и не должно использоваться как главный регулятор результата.

### Parametric bands

У каждой из трёх полос есть:

- **Частота** — центр воздействия;
- **Усиление** — boost/cut в dB;
- **Q** — ширина полосы.

Практические области:

| Область | Что слышно | Что попробовать |
|---|---|---:|
| 150–300 Hz | мутность, «бочка» | `-1…-3 dB`, Q `0.7–1.2` |
| 500–1000 Hz | носовой/коробочный оттенок | `-1…-2 dB`, Q `1.0–1.5` |
| 2–4 kHz | разборчивость согласных | `+1…+2 dB`, Q `0.7–1.2` |
| 5–8 kHz | резкость, сибилянты | `-1…-2 dB`, Q `1.5–3` |
| 8–12 kHz | «воздух» и яркость | осторожный boost `+0.5…+1.5 dB` |

Начинать лучше с ослабления проблемной области, а не с усиления. Большой boost быстро приводит к резкости и клиппингу. Если исходник 16 kHz, частоты выше примерно 8 kHz недоступны в полном объёме; high-shelf не восстановит их правдоподобно.

**Q:** маленькое значение воздействует широко и естественно, большое — узко и хирургически. Для речи обычно достаточно `0.7–1.5`.

### High Shelf

Меняет всё выше выбранной частоты:

- `+0.5…+1.5 dB` около 8–10 kHz — немного больше яркости;
- `-1…-2 dB` около 6–8 kHz — если голос режет слух.

Если появились свистящие «с» и «ш», сначала уменьшить high-shelf, а затем проверить область 5–8 kHz parametric-полосой.

## Compressor

Компрессор уменьшает разницу между тихими и громкими участками. Он делает речь плотнее, но слишком сильная компрессия поднимает шум и съедает естественную динамику.

| Параметр | Влияние | Безопасный старт |
|---|---|---:|
| Threshold | уровень, выше которого начинается сжатие | `-18 dB` |
| Ratio | сила сжатия | `2:1` |
| Attack | как быстро ловятся пики | `5–10 ms` |
| Release | как быстро компрессор отпускает сигнал | `80–150 ms` |
| Knee | мягкость входа в компрессию | `4–8 dB` |
| Makeup | компенсация потерянной громкости | `0…+2 dB` |

Что пробовать:

- голос скачет по громкости — threshold `-20…-16 dB`, ratio `2:1`;
- согласные стали слишком резкими — увеличить attack до `10–20 ms`;
- слышно «дыхание» или громкость заметно пульсирует — увеличить release;
- звук стал плоским и зажатым — поднять threshold, уменьшить ratio или отключить makeup;
- после compressor стало просто громче, но не лучше — вернуть makeup к `0 dB` и сравнить на одинаковой громкости.

Для TTS обычно не нужен ratio выше `3:1` и makeup выше `+3 dB`.

## Limiter

Limiter — последний защитный блок. Он не должен формировать характер голоса; его задача — не допустить опасных пиков после EQ и makeup compressor.

Рекомендуемые значения:

- **Ceiling:** `-1 dB`;
- **Release:** `50–100 ms`.

Если limiter постоянно сильно уменьшает сигнал, причина обычно в слишком большом EQ boost или makeup compressor. Сначала ослабить предыдущие блоки.

В текущей реализации limiter использует peak envelope, плавное отпускание и жёсткое ограничение потолком. Lookahead пока нет, поэтому слишком быстрый одиночный пик может быть ограничен жёстче, чем при студийном lookahead-limiter.

## Готовые стартовые варианты

### Естественный голос

```text
EQ: включить, только parametric band -1 dB в области 200–300 Hz
Compressor: threshold -18 dB, ratio 2:1, attack 8 ms, release 120 ms, makeup 0 dB
Limiter: ceiling -1 dB, release 70 ms
```

### Более разборчивый голос

```text
EQ: +1 dB около 3 kHz, при необходимости +0.5 dB high-shelf около 9 kHz
Compressor: threshold -18 dB, ratio 2:1, makeup 0…+1 dB
Limiter: ceiling -1 dB
```

### Тёмный/глухой голос

```text
EQ: +1…+1.5 dB около 3 kHz и +0.5…+1 dB high-shelf
Компрессор: мягкий, ratio 2:1
Лимитер: оставить защитным, без попытки сделать громкость
```

### Резкий голос

```text
EQ: -1…-2 dB в области 5–8 kHz, high-shelf выключить
Compressor: attack 10–20 ms, ratio около 2:1
Limiter: ceiling -1 dB
```

## Симптомы и быстрые решения

| Симптом | Что проверить |
|---|---|
| Голос стал тонким | уменьшить Low Cut или вернуть его в bypass |
| Появилась резкость | уменьшить 3–8 kHz или high-shelf |
| Слова стали менее разборчивыми | ослабить denoise, вернуть EQ boost к 0 dB, увеличить compressor attack |
| Громкость «дышит» | уменьшить compressor ratio, увеличить release |
| Появились щелчки/жёсткие пики | уменьшить makeup/EQ boost, оставить limiter включённым |
| Разницы почти не слышно | это нормально для чистого TTS; не усиливать всё подряд |

Главное правило: лучше небольшая коррекция `±1…2 dB`, чем агрессивная цепочка из нескольких усилителей. Если исходный голос уже чистый, DSP должен исправлять конкретный недостаток, а не обязательно быть включённым целиком.
