# Stage 44 — VTube Studio: lifecycle подключения в UI

## Проблема

Панель VTube Studio использует кнопку Play как «проверку подключения» и
показывает локальный результат последней проверки. В отличие от Twitch,
пользователь не может явно запустить, остановить или перезапустить runtime
подключение, а настройки автозапуска нет.

Простого вызова существующего `disconnect()` недостаточно: следующий
`typing=true` сейчас снова лениво откроет WebSocket. Поэтому ручной Stop должен
фиксировать отдельное runtime-намерение «не подключаться».

## Целевая модель

- `start_on_boot` — сохранённая настройка, по умолчанию `false`.
- Runtime `desired_running` не является пользовательской настройкой:
  - Start/Restart устанавливают его в `true`;
  - Stop устанавливает в `false`;
  - при запуске приложения оно устанавливается из `start_on_boot`.
- `typing=true` может восстанавливать оборванный WebSocket только при
  `desired_running=true`.
- Старое поле `enabled` остаётся читаемым для обратной совместимости
  `settings.json`, но больше не представляется отдельной галкой в панели.
- Runtime status: `Disconnected`, `Connecting`, `Connected`, `Error`.
  Frontend получает начальное состояние командой и дальнейшие изменения через
  `vtube-studio-status-changed`.

## UI

Источник визуального паттерна — `src/components/TwitchPanel.vue`:

- Connected: Restart + Stop.
- Disconnected/Error: Start + disabled Stop.
- Connecting: disabled Start + disabled Stop.
- Под header: `Запускать при старте приложения`, сохраняется сразу.
- Порт и общая кнопка Save сохраняются.
- **Тест параметра** — отдельная карточка с заголовком `Тест параметра`,
  доступная только когда runtime status = `Connected`. Содержит:
  - `Таймаут, мс` (числовой, от 100 до 5000, по умолчанию 800);
  - `Повторы` (числовой, от 1 до 10, по умолчанию 1);
  - кнопка `Проверить`, отключена если статус не Connected, выполняется
    другая операция, или значения невалидны.
  - Подсказка: каждый повтор отправляет `TTSBardTyping=1`, ждёт таймаут,
    затем отправляет `0`. Между повторами — пауза той же длительности.
  - Команда: `test_vtube_studio_typing(timeout_ms, repeat_count)`.
  - Требует `desired_running=true`, статус Connected, живой WebSocket.
  - Не вызывает `connect_ws`, аутентификацию, создание параметра или
    сохранение настроек.
  - Не меняет `desired_running` во время или после теста.
  - При ошибке транспорта/inject: сокет сбрасывается, auth очищается,
    статус → Error, событие эмитируется перед возвратом ошибки.
  - Старая команда `test_vtube_studio_connection` и кнопка `Тест подключения`
    удалены из UI. Команда сохранена для обратной совместимости.

Иконки обязаны иметь `title` и `aria-label`. Сохраняются размеры, отступы,
цвета, hover/disabled-состояния Twitch-панели и адаптивность без
горизонтального overflow.

## Совместимость

- Старые настройки без `start_on_boot` десериализуются с `false`.
- Token не попадает в safe DTO.
- GitHub/release сборка не получает локальных espeak/CRT patch-файлов.
- Существующая передача `TTSBardTyping` и WebView typing не меняется.
