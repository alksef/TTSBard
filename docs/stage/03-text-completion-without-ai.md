# Stage: Автодополнение с предложением продолжения — БЕЗ AI (возможно ли?)

**Дата:** 2026-06-28
**Статус:** research / ✅ РЕШЕНО
**Решение:** **Гибрид со слоями** (по рекомендации) — базовый слой без AI + опциональный AI-слой.
**Связано:** `01-monaco-vs-codemirror-editor-research.md`, `02-local-history-autocomplete.md`

---

## Вопрос

Реализовать автодополнение, которое предлагает **продолжение предложения/фразы**
(не одно слово, а кусок текста), **без AI** (без Claude API и нейросетей). Возможно ли?

## Короткий ответ

**Да, технически возможно**, но **качество будет заметно ниже** AI, особенно для русского
языка. Без нейросетей получаются «глухие» предсказания: только то, что уже видел, без
понимания смысла и без адаптации к морфологии.

---

## Варианты реализации без AI

### Вариант A — n-граммная модель / цепь Маркова
- Строим статистику: «после последовательности из N слов чаще всего идёт слово X».
- Например биграммы/триграммы: `["привет", "как"] → "дела"`.
- Источник данных: **тексты самого пользователя** (его история ввода) + опционально готовый
  корпус.
- **Плюсы:** очень быстрые (микросекунды, без GPU), малый объём памяти, прозрачная логика,
    нет проблем с приватностью (всё локально, на своих данных).
- **Минусы:** предсказывает **только уже виденные** комбинации; плохо для русского
    (богатая морфология — падежи, рода, спряжения; n-грамма «видит» только 1–2
    предыдущих слова и часто ошибается в форме); жёсткие, негибкие предложения.

### Вариант B — взвешенный Trie по фразам
- Как в `02-...`, но храним целые часто используемые фразы целиком (шаблоны приветствий,
  типовые ответы).
- По префиксу подставляем целиком заготовленную фразу.
- **Плюсы:** идеально для повторяющихся коротких фраз (TTS-кейс — часто одни и те же реплики).
- **Минусы:** не генерирует ничего нового, только точные совпадения/префиксы.

### Вариант C — T9-стиль
- Предсказание следующего слова по набранным символам + частоте.
- **Минусы для русского:** ещё хуже из-за неоднозначности и морфологии; современный софт
    (SwiftKey, Gboard) от этого давно ушёл в нейросети.

### Гибрид (практичный для нашего кейса)
- **Trie/частотник по словам** (Вариант B + `02-...`) для быстрых подсказок отдельных слов.
- **n-граммы по истории пользователя** для подсказки «следующего слова» в типовых фразах.
- **Шаблоны/сниппеты** для часто повторяемых целиком фраз.
- Это покрывает бóльшую часть TTS-сценария (короткие повторяющиеся реплики) без AI.

## Качество: без AI vs с AI

- Нейросетевые модели на русском дают **на ~30–50% выше точность** за счёт attention
  (дальние зависимости, морфология).
- Не-AI модели часто выдают **грамматически неверные** или контекстно неуместные продолжения.
- Без AI жизнеспособно **только для коротких высокочастотных фраз** (приветствия, типовые
  глаголы/ответы). Для развёрнутых предложений качество будет плохим.

## Когда без AI — нормально
- Короткие повторяющиеся реплики (осн. кейс приложения).
- Когда критичны приватность и оффлайн (всё локально, без отправки текста в Claude API).
- Когда устраивает подстановка заготовок, а не «творческое» продолжение.

## Когда нужен AI
- Развёрнутые, осмысленные продолжения произвольного текста.
- Это и есть то, что уже заявлено в `PROBLEMS.md:57,74` (Anthropic Claude API).

## Рекомендация
- **Базовый слой — без AI** (Trie + n-граммы по личной истории + шаблоны фраз): закрывает
  повторяющиеся реплики, работает оффлайн и приватно.
- **Опциональный слой — AI** (Claude API): включается пользователем для сложных продолжений.
- Этажи не конфликтуют — можно показывать локальные подсказки по умолчанию и AI — по
  отдельному триггеру/настройке.

## Архитектура слоёв (гибрид, внутри CodeMirror autocomplete)
- **Слой 0 — Trie слов из истории** (`02-...`): подсказки отдельных слов по подстроке,
  сортировка по частоте. Всегда включён, оффлайн.
- **Слой 1 — n-граммы / Марков** по личной истории: «следующее слово» / короткое продолжение
  фразы для типовых реплик. Оффлайн, без AI.
- **Слой 2 — AI** (Claude API, `PROBLEMS.md:57,74`, уже есть `ai` модуль и `AiProvider`):
  опционально, по отдельному триггеру/настройке `editor.ai_completion`.
- Попап автокомплита CM6 объединяет слои: сначала точные Trie-совпадения, затем n-граммные
  продолжения, AI — отдельным пунктом/по явному вызову.
