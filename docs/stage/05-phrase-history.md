# Stage: История отправленных фраз (Persistent Phrase History)

**Дата:** 2026-06-28
**Статус:** research / option comparison
**Из research:** `docs/research/01-...` → идея **C** (Persistent Phrase History)
**Связано:** `04-playback-control-floating-window.md`, `docs/plans/71..73`

## Цель (запрос пользователя)
- **Persistent** журнал отправленных фраз с быстрым повтором (повторно озвучить).
- Подумать: как сделать **удобно**? Поскольку теперь есть редактор (CodeMirror, план 71) —
  можно ли реализовать через него?

## Контекст кода (что уже есть после 71–73)
- `src-tauri/src/history.rs` — `HistoryManager` (Trie слов + n-граммы), persistence в
  `input_history.json` / `ngrams.json`.
- `record_text` уже вызывается из фронтенда при отправке (план 72).
- Редактор `TtsEditor.vue` (CodeMirror) + autocomplete-источники (слой 0/1/2, план 73).
- События: `TextSentToTts` — момент отправки фразы (хороший триггер для записи в журнал).
- Воспроизведение: `rodio::Sink` (план 04 добавит `PlaybackFinished`).

> **Различие:** план 72 = история **слов** (для автокомплита). Здесь = история **целых фраз**
> (для повтора/переиспользования). Разные модели/файлы, но могут жить в одном `HistoryManager`.

## Варианты хранения
### A. Отдельный файл `phrase_history.json` (рекомендуемый)
- `Vec<PhraseEntry { text, timestamp, count, lastUsed }>`; лимит N (например 100–500),
  кольцевой буфер (старые вытесняются).
- Сортировка: по `lastUsed desc` (недавние сверху) и/или по `count` (частые).
- **Плюсы:** изолировано, просто бэкапить, не мешает Trie/словам.
### B. Часть существующего `input_history.json` (тот же HistoryManager)
- Добавить slice `phrases` рядом со словами и n-граммами.
- **Плюсы:** один менеджер/файл. **Минусы:** смешение ответственностей.
### C. SQLite / embedded DB
- Избыточно для сотен фраз; не соответствует существующей file-based стратегии (`KEY_DECISIONS` #15).

**Рекомендация:** **A** (отдельный `phrase_history.json`), управляется тем же
`HistoryManager` (расширить) или новым `PhraseHistoryManager`.

## Варианты UX (как показывать и повторять)

### Вариант 1 — Список в плавающем окне управления воспроизведением (план 04)
- В окне `playback-control` секция «Последние 5 фраз» → клик = повторно озвучить.
- **Плюсы:** окна управления + история в одном месте; закрывает стримерский кейс.
- **Минусы:** только 5; не для редактирования.

### Вариант 2 — Через редактор CodeMirror (идея пользователя) ⭐
Несколько способов задействовать редактор:
- **2a. Autocomplete-источник «недавние фразы»:** при вводе показать целые недавние фразы
  как варианты продолжения/замены (новый слой в попапе плана 73, `type: "recent"`). Принять
  Tab → вставить фразу в редактор, потом править и отправить.
- **2b. Command Palette / быстрый выбор:** хоткей внутри редактора (например `Ctrl+Space`
  на пустой строке или `/`) открывает список недавних фраз; выбор загружает фразу в редактор.
- **2c. Режим «история как черновики»:** боковая панель/выпадение в InputPanel со списком
  фраз; двойной клик → загрузить в редактор для правки перед отправкой.
- **Плюсы:** повтор + **правка** фразы перед повтором (важно — текст мог устареть);
  переиспользует инфраструктуру автокомплита (план 73).
- **Минусы:** 2a может засорять попап; 2b/2c — дополнительный UI.

### Вариант 3 — Отдельная панель в основном окне (как SoundPanel/Preprocessor)
- Отдельный раздел «История фраз» в sidebar со списком, поиском, повтором, удалением.
- **Плюсы:** полнофункциональное управление (поиск, очистка, избранное). **Минусы:**
  больше UI; не «мгновенный» доступ.

### Вариант 4 — Гибрид (рекомендация)
- **Быстрый повтор (без правки):** 5 недавних фраз в плавающем окне управления (план 04) —
  один клик → озвучить.
- **Правка перед повтором:** через редактор — autocomplete-источник «недавние фразы» (2a)
  ИЛИ command-palette (2b) загружает фразу в редактор.
- **Полное управление:** панель в основном окне (3) — поиск/чистка/избранное.
- **Горячая клавиша** «повторить последнюю фразу» (replay last) — мгновенно.

## Решения по UX (предложить пользователю)
1. **Сколько хранить:** N фраз (100? 500?) и_policy вытеснения (по времени vs по частоте).
2. **Повтор = озвучить как есть** vs **загрузить в редактор для правки**: нужны оба?
   (Гибрид закрывает оба.)
3. **Дедупликация:** одинаковые фразы схлопывать (incr count) или хранить каждую отправку?
4. **Где кнопка/доступ:** плавающее окно (план 04) + редактор (autocomplete/palette) + панель?
5. **Метаданные:** хранить ли с фразой голос/провайдера (чтобы повторить тем же голосом)?

## Рекомендация
- Хранение: **A** (`phrase_history.json`, тот же `HistoryManager`).
- UX: **Вариант 4 (гибрид)** — быстрый повтор в плавающем окне (план 04) + загрузка в
  редактор через autocomplete-слой «recent»/command-palette (переиспользует план 73) +
  панель для полного управления. Горячая клавиша «replay last».
- Это закрывает и «повторить как было», и «поправить и повторить» — два разных реальных кейса.
