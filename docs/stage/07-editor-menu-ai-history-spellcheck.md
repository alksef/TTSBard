# Stage: Меню редактора (AI, история, онлайн-проверка орфографии)

**Дата:** 2026-06-30
**Статус:** research / оценка (план — отдельным файлом)
**Решение (предварительное):** **меню-кнопка (popover/dropdown)** в InputPanel, открывающая
панель действий редактора: AI-коррекция, история фраз (drafts, plan 75), онлайн-проверка
орфографии. НЕ табы — это **альтернативный/дополнительный** UX к `06-editor-tabs`.
**Связано:** `01-monaco-vs-codemirror-editor-research.md`, `05-phrase-history.md`,
`06-editor-tabs-multiple-texts.md`, планы `71..75`

## Цель (запрос пользователя)
- Дать **единое меню** для доступа к функциям редактора, которые сейчас разбросаны:
  - **AI** — сейчас кнопка `correct-button` («AI», `InputPanel.vue:201-211`), корректирует весь текст.
  - **История** — сейчас только через autocomplete (`TtsEditor.vue` слой 0/1: `get_history_suggestions`,
    `get_phrase_completion`) и drafts (plan 75, ещё не реализован как UI списка).
  - **Онлайн-проверка орфографии** — НЕТ ВООБЩЕ (нет команды, нет UI).
- Вопрос пользователя: «через меню вызывать AI, историю? проверка орфографии онлайн».

## Контекст кода (что уже есть)
- `InputPanel.vue:130-141` — `correctText()` → `invoke('correct_text')` (AI-коррекция, OpenAI/Z.ai).
- `InputPanel.vue:109-115` — `recordHistory()` → `invoke('record_history')` (запись фразы).
- `TtsEditor.vue` — autocomplete-источники: история слов, фразы, AI-completion (план 73).
- `TelegramAuthModal.vue`, `FishAudioModelPicker.vue` — готовый паттерн **modal** (`modal-overlay` /
  `modal-container`, CSS-vars `--modal-overlay`). Но для «меню действий» modal тяжеловат — лучше
  **popover/dropdown** (легче, не блокирует контент).
- Тема: везде CSS-vars (`--color-bg-elevated`, `--color-border-strong`, `--shadow-soft` и т.д.).
- Онлайн-проверки орфографии в коде **нет** — потребуется новая бэкенд-команда (HTTP к API) +
  UI-подсветка ошибок в CodeMirror.

## Варианты меню (что рассматривали)

### 1. Dropdown/popover из кнопки ⭐ рекомендуется
Кнопка (например `⋮` или `Sparkles`) в строке `InputPanel` рядом с `correct-button`. Клик →
выпадающий список действий. Переиспользует CSS-vars тему, не перекрывает редактор.
- **Плюсы:** лёгкий, знакомый UX, масштабируется (добавлять пункты = добавлять строки).
- **Минусы:** надо делать click-outside-to-close + позиционирование (но это мелочи, ~30 строк).

### 2. Контекстное меню (правый клик в редакторе)
CodeMirror поддерживает свой `contextmenu`; можно перехватить и показать своё меню на выделении.
- **Плюсы:** «правильное» место для правки выделенного фрагмента (AI/орфография по выделению).
- **Минусы:** менее discoverable (пользователь должен знать, что есть ПКМ); на Tauri/WebView
  есть нюансы с нативным меню. Хорош как **дополнение** к dropdown, не замена.

### 3. Боковая панель (drawer) — список истории/ошибок слева
Отдельная раскрывающаяся панель со списком drafts / найденных ошибок орфографии.
- **Плюсы:** много места для списков (история фраз, список ошибок с заменами).
- **Минусы:** перетягивает на «табы/меню текстов» из `06` — другая задача; layout в minimal-mode
  ломается сильнее.

> **Рекомендация: Dropdown (1) как основа.** Контекстное меню (2) — опционально для
> «AI/орфография по выделению» на следующем этапе. Drawer (3) — отдельная задача (если история
> вырастет в полноценный список drafts-UI из plan 75).

## Онлайн-проверка орфографии — отдельная оценка (самая дорогая часть)

Это **новая фича**, которой нет ни в бэкенде, ни во фронте. Компоненты:

1. **Бэкенд-команда** `check_spelling(text) -> Vec<SpellError { from, to, suggestions[] }>`:
   - HTTP-запрос к внешнему API (LanguageTool — open-source, есть бесплатный API; или Yandex.Speller).
   - Прокси: учитывать существующий network-прокси (`/tts/network/proxy`) — как в OpenAI/AI-запросах.
   - Кэшировать/дебаунсить (не дёргать API на каждое нажатие).
2. **Фронт: подсветка в CodeMirror** — `Decoration`/`StateField` с подчёркиванием ошибок
   (Linter-подобно; в `@codemirror/language` есть `linter` — он идеально подходит).
3. **Фронт: варианты замены** — tooltip/меню на ошибке со списком suggestions.

**API-варианты:**
- **LanguageTool** — русский поддержан, есть self-hosted и облачный API; квоты. Рекомендуется.
- **Yandex.Speller** — бесплатный, русский хорошо; простой JSON API.
- **Local (hunspell/nuspell)** — без сети, но нужен биндок в Rust + словари в бандле; тяжело.

**Оценка:** онлайн-орфография = **средняя/крупная** задача (бэкенд-команда + прокси + кэш +
CodeMirror linter-интеграция + tooltip замен). Больше, чем само меню.

## Риски / нюансы (для будущего плана)
1. **Дублирование с autocomplete** — AI-коррекция/дописать уже частично в попапе автокомплита
   (план 73). Меню = **ручной/явный** вызов тех же бэкенд-функций. Зафиксировать: не дублируем
   логику, переиспользуем команды.
2. **Конфликт с drafts-UI (plan 75)** — «История фраз» в меню зависит от того, как plan 75 даст
   список фраз. Меню не должно проектировать свой формат данных — ждём/опираемся на plan 75.
3. **Онлайн-зависимость** — орфография не работает офлайн; нужен fallback (warning, как в
   TTS-провайдерах без ключа).
4. **Приватность** — текст уходит на внешний API. Подсветить пользователю (как с AI-ключами);
   возможно, настройка вкл/выкл (новое поле в `settings.json` → `#[serde(default)]`, иначе грабли
   миграции как с `playback_pause`).
5. **Click-outside / escape** — popover должен закрываться (есть паттерн ESC из `TtsEditor`).
6. **Minimal-mode** — меню-кнопка должна жить в узком окне (как `correct-button` сейчас).

## Оценка трудозатрат (по этапам)

| Этап | Объём | Зависимости |
|---|---|---|
| Dropdown-каркас `EditorMenu.vue` (popover, тема, click-outside) | малый-средний | нет |
| Пункты «AI: корректировать / дописать» (обёртки над существующими командами) | малый | ✅ бэкенд есть |
| Пункт «История фраз» (список drafts в меню) | средний | ⚠️ UI plan 75 |
| Онлайн-проверка орфографии (бэкенд + CodeMirror linter + tooltip) | средняя-крупная | ❌ новая |

**Итого:** меню-каркас + AI-пункты = небольшая задача (фронтенд-only, бэкенд уже есть).
История и орфография — отдельные, более крупные подзадачи со своими планами.

## KEY_DECISIONS (предварительные)
- **Меню = dropdown/popover** из кнопки в InputPanel (не modal, не drawer).
- **Переиспользуем существующие команды** (`correct_text`, `get_ai_completion`) — не дублируем.
- **Онлайн-орфография — отдельная крупная подзадача** (LanguageTool/Yandex API, CodeMirror linter),
  со своим планом; в меню попадает последней.
- **Тема через CSS-vars** (light/dark), как во всём приложении.
- **Поэтапно:** меню-каркас + AI → история (после plan 75) → орфография.
- **Конфликт с `06-editor-tabs`:** это **два независимых UX** (табы = несколько текстов;
  меню = действия над одним). Могут сосуществовать (табы сверху, меню-кнопка в строке ввода).
