# Stage 36: Доработки Telegram-подключения

**Дата:** 2026-07-17  
**Статус:** реализовано

## Цель

Сделать подключение Telegram устойчивее и аккуратнее по UX: корректно переживать временный сброс серверного auth-процесса и не показывать лишние/чувствительные данные в модальном окне.

## Реализованные доработки

### 1. Повтор запроса кода при `AUTH_RESTART`

Telegram может вернуть `500 AUTH_RESTART` на первом `auth.sendCode`. Backend теперь автоматически повторяет запрос максимум два раза с задержками 250 и 500 мс.

Retry выполняется только для структурированной ошибки `InvocationError::Rpc` с `code == 500` и `name == "AUTH_RESTART"`. Другие ошибки и таймауты не повторяются.

Файл:

- `src-tauri/src/telegram/client.rs`

### 2. Скрытие данных авторизации

В `TelegramAuthModal.vue` добавлены переключатели видимости для:

- номера телефона;
- API ID;
- API Hash;
- пароля 2FA.

По умолчанию все эти значения скрыты.

### 3. Упрощение шага ввода кода

Удалена лишняя строка вида `На номер: +...` из шага ввода кода Telegram. Номер уже введён на предыдущем шаге и не должен дублироваться.

### 4. Явное закрытие модального окна

Клик по затемнённой области за пределами окна больше не закрывает подключение. Окно закрывается только по кнопке `×` или явным действием внутри модального окна, что предотвращает случайную потерю текущего auth-flow.

Файл:

- `src/components/TelegramAuthModal.vue`

## Ограничения

Автоматический retry не скрывает ошибку после трёх неудачных ответов Telegram: в этом случае пользователю показывается исходная ошибка запроса кода.
