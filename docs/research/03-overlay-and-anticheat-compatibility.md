# Research: оверлей, саундпад и совместимость с античитом

**Дата:** 2026-07-11  
**Статус:** исследование реализации; не является гарантией совместимости с конкретной игрой

## Краткий вывод

Визуальный оверлей TTSBard не внедряется в процесс игры. Это отдельные прозрачные окна Tauri/WebView, поэтому сам по себе он относится к низкорисковому классу внешних оверлеев. Однако приложение также устанавливает глобальный low-level keyboard hook (`WH_KEYBOARD_LL`) для режима Intercept. Именно этот механизм, а не отрисовка, может конфликтовать с античитом или правилами конкретной игры.

Нельзя обещать, что античит «не отреагирует»: решение всегда принимает античит и издатель конкретной игры. BattlEye указывает, что нечитерские оверлеи обычно поддерживаются, если разработчик игры не решил иначе; Riot отдельно отделяет допустимые overlays/API-инструменты от внешнего чтения памяти, которое не разрешено. См. [BattlEye FAQ](https://www.battleye.com/support/faq/) и [Vanguard Developer Relations](https://support-developer.riotgames.com/hc/en-us/articles/28021427366163-Vanguard).

## Как отрисовываются окна

В `src-tauri/tauri.conf.json` объявлены два независимых окна:

| Label | Размер по умолчанию | Свойства | Интерфейс |
|---|---:|---|---|
| `soundpanel` | 450×225 | без рамки, прозрачное, `alwaysOnTop`, не в taskbar, изначально скрыто | `src-soundpanel/SoundPanelApp.vue` |
| `playback-control` | 350×400 | те же свойства | `src-playback/PlaybackControlApp.vue` |

Окна создаёт Tauri, а содержимое рисует WebView через Vue и CSS. Прозрачность и цвет — это `backgroundColor: rgba(...)` на корневом HTML-элементе; они не используют DirectX/OpenGL hook и не рисуются в swapchain игры. Положение, цвет и opacity хранятся в `%APPDATA%/ttsbard/windows.json` через `WindowsManager`.

В проверенных модулях нет чтения памяти игры, DLL-инъекции, hook-а графического API или драйвера.

## Окно управления воспроизведением

`playback-control` показывает текущую фразу, статус, очередь и последние фразы. Кнопки выполняют pause/resume, stop и repeat через Tauri-команды к `PlaybackManager`; событие playback обновляет Vue-состояние окна.

При показе окно восстанавливает позицию, вызывается `show()` и **`set_focus()`**. Следовательно, это не полностью пассивный игровой HUD: его открытие отберёт фокус у игры. Окно также можно открыть глобальной комбинацией, при старте приложения или переключателем из главного UI.

## Саундпад: ввод и известные ограничения

- A–Z обрабатываются в самом WebView через `keydown` и `event.code` (`KeyA`…`KeyZ`), поэтому окно должно быть в фокусе.
- При открытии саундпад также вызывает `set_focus()`. Нажатие A–Z запускает привязанный аудиофайл и, если `stay_visible` выключен, скрывает окно.
- `click-through` логически противоречит этому пути: окно без фокуса не получит DOM `keydown`, поэтому A–Z обычно не сработают. Это не способ запускать звуки поверх игры без переключения фокуса.
- Существующий `Intercept` работает только для NumPad и F-клавиш и поддерживает открыть окно, а также pause/stop/repeat playback. Он **не запускает** A–Z sound bindings.
- Переключение click-through сохраняет настройку, но команда `sp_set_floating_clickthrough` не вызывает `set_ignore_cursor_events` для уже открытого нативного окна. Гарантированное применение происходит при следующем показе `soundpanel`; CSS немедленно отключает pointer events лишь у области контента.

## Античит: оценка риска

| Механизм | Реализация в проекте | Оценка |
|---|---|---|
| Внешние topmost-окна | штатные Tauri/WebView окна | низкий риск, но возможны запрет политикой игры или невидимость в exclusive fullscreen |
| Глобальные hotkeys | `tauri-plugin-global-shortcut` | обычно штатный механизм ОС; возможны конфликты сочетаний |
| Intercept | `SetWindowsHookExW(WH_KEYBOARD_LL, …)` | повышенный риск совместимости: хук глобально видит ввод и при активной привязке поглощает клавишу |
| Защита от захвата | WinAPI `SetWindowDisplayAffinity(WDA_EXCLUDEFROMCAPTURE)` | не взаимодействует с процессом игры, но может скрывать окно из OBS/захвата и быть запрещена правилами трансляции |

Важно: hook устанавливается при запуске приложения. Выключение настройки Intercept лишь переводит callback в pass-through; сам `WH_KEYBOARD_LL` не снимается до завершения приложения.

## Рекомендация для защищённых игр

1. Не считать совместимость гарантированной без названия игры, античита и её правил.
2. Для минимального риска использовать только визуальное окно и обычные global shortcuts; не включать Intercept.
3. Для настоящего режима `gaming-safe` изменить жизненный цикл: создавать `WH_KEYBOARD_LL` только после явного включения Intercept и снимать hook сразу при выключении.
4. Не рассчитывать на саундпад A–Z как на бесфокусный игровой саундборд: текущая архитектура требует фокуса окна.
5. Проверять совместимость только штатным запуском и через официальную поддержку/документацию игры; не пытаться обходить античит.

## Основные файлы

- `src-tauri/tauri.conf.json` — свойства окон.
- `src-tauri/src/soundpanel_window.rs` и `src-tauri/src/playback_window.rs` — показ, позиция, фокус.
- `src-soundpanel/SoundPanelApp.vue` и `src-playback/PlaybackControlApp.vue` — Vue/CSS-отрисовка и ввод.
- `src-tauri/src/soundpanel/hook.rs` — глобальный `WH_KEYBOARD_LL`.
- `src-tauri/src/soundpanel/bindings.rs` — команды саундпада.
- `src-tauri/src/hotkeys.rs` — registered global shortcuts.
- `src-tauri/src/window.rs` — `SetWindowDisplayAffinity`.
