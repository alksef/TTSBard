# TTSBard

Быстрый синтез речи для общения и трансляций — превратите мысль в голос с минимумом отвлечения от основного занятия.

TTSBard — это незаметное desktop-приложение, которое работает в фоне и доступно мгновенно по горячей клавише в любом окне. Оно объединяет ввод текста, озвучку, звуковую панель и интеграции (OBS, Twitch) в одном месте — чтобы не переключаться между приложениями.

## Для кого

Приложение создавалось для двух аудиторий с общим знаменателем — «быстро превратить мысль в голос»:

- **Люди с ограничениями речи**, использующие синтез речи как основной способ общения в мессенджерах, браузере и повседневных задачах. Для них критичны скорость и удобство ввода: автодополнение, история фраз, проверка орфографии, минимум когнитивной нагрузки.
- **Стримеры и контент-мейкеры**, использующие озвучку и звуковую панель на трансляциях (Twitch, YouTube). Для них важны озвучка в реальном времени, наборы звуков, OBS-интеграция и скорость реакции прямо во время игры.

Большинство возможностей работает для обеих аудиторий — акценты просто смещаются.

## История проекта

Всё началось с [интервью на YouTube](https://www.youtube.com/watch?v=O6StzI8CCx0) с участием стримеров [megubite](https://www.twitch.tv/megubite) и [alexxa69419](https://www.twitch.tv/alexxa69419).

Наблюдая за трансляциями [alexxa69419](https://www.twitch.tv/alexxa69419), удалось глубже погрузиться в тему TTS и именно через неё узнать о существовании целого комьюнити стримеров, общающихся с аудиторией исключительно с помощью синтеза речи: [pirunka_tsd](https://www.twitch.tv/pirunka_tsd), [weissstille](https://www.twitch.tv/weissstille) и [safutara_kaino](https://www.twitch.tv/safutara_kaino).

Стримеры используют программу **TTS Voice Wizard** в связке с Python-скриптом [TTS-Wizard-2-Silero-Telegram-Bot](https://github.com/alexxa69420/TTS-Wizard-2-Silero-Telegram-Bot) (написанным `alexxa69419`) для интеграции с Telegram-ботом [@silero_voice_bot](https://t.me/silero_voice_bot). Также `alexxa69419` разработала вспомогательную утилиту [randompad](https://github.com/alexxa69420/randompad) — написанный с помощью нейросетей саундпад, который при нажатии одной горячей клавиши воспроизводит случайный звук из заранее заданного набора. А у [pirunka_tsd](https://www.twitch.tv/pirunka_tsd) была подсмотрена идея вывода произносимого текста прямо в OBS Studio для удобства зрителей.

Хотя TTS Voice Wizard — мощный инструмент, он перегружен лишними функциями (так как изначально создавался под VRChat) и не имеет встроенного вывода текста в OBS. Но самым главный недостаток - необходимость переключаться на окно программы через `Alt+Tab` для ввода каждой новой фразы.

В эпоху современных ИИ-агентов («вайб-кодинга») появилась возможность быстро создать единое, удобное десктопное приложение, специально спроектированное под TTS-общение и оснащенное всеми нужными инструментами (глобальный перехват ввода, саундпад, OBS/Twitch-интеграция). Так и родился **TTSBard**.

## Решение

**TTSBard** — это незаметное TTS-приложение, которое работает в фоновом режиме и доступно мгновенно по горячей клавише в любом приложении.

*Подробная презентация всех 11 функциональных блоков:* [docs/user/presentation.md](docs/user/presentation.md).

### Основной сценарий использования

1. Нажали `Ctrl+Shift+F3` — главное окно с редактором мгновенно появляется поверх всех окон и получает фокус.
2. Ввели текст (можно использовать вкладки для параллельной работы или автодополнение по истории).
3. Нажали `Enter` (или кликнули кнопку отправки) — текст отправляется в TTS и озвучивается.
4. Нажали `Escape` — окно скрывается в трей, возвращая фокус в предыдущее активное приложение (например, в игру).

### Главное окно и системный трей

**Идея:** Приложение всегда доступно, но не мешает работе.

**Как это работает:**
- При закрытии главного окна оно **не закрывается**, а сворачивается в **системный трей** (область уведомлений Windows)
- Двойной клик по иконке разворачивает окно
- Правый клик открывает контекстное меню (Выход)

**Преимущества:**
- ✅ Приложение всегда под рукой
- ✅ Не загромождает панель задач
- ✅ Быстрый доступ без Alt+Tab
- ✅ **Защита от захвата экрана:** возможность сделать окна приложения (настройки, саундпад, плеер) невидимыми для OBS, Discord и других программ записи, предотвращая утечку API-ключей и токенов в эфир.

### Продвинутый редактор текста

Встроенный текстовый редактор на базе **CodeMirror 6** специально разработан для быстрого и комфортного набора текста без отвлечения от основного занятия.

**Возможности:**
- **Независимые вкладки (Tabs):** Поддержка создания нескольких параллельных вкладок с возможностью быстрого переключения между ними (по аналогии с вкладками в браузере или IDE). Содержимое вкладок, их порядок и активное состояние автоматически сохраняются между перезапусками приложения.
- **Офлайн-проверка орфографии:** Локальная проверка орфографии в реальном времени (на базе Hunspell) без обращения к внешним онлайн-сервисам.
- **История слов и автодополнение:** Приложение собирает историю введенных слов в персональный словарь (сохраняется в `input_history.json` между сессиями) и предлагает умные подсказки по подстроке во время набора, ускоряя общение.
- **Журнал истории фраз:** Быстрый поиск и вставка ранее отправленных реплик (хранятся в `phrase_history.json`) в двух режимах: **добавление** в конец текущей строки или полная **замена** текста в редакторе (с подтверждением для защиты от перезаписи).
- **Макросы и быстрая вставка:** Поддержка пресетов быстрой замены фраз (`\name` и `%username`), которые автоматически преобразуются в текст при нажатии пробела в процессе набора.
- **Числовой препроцессор:** Автоматическая конвертация чисел в слова с правильным грамматическим согласованием рода и множественного числа прямо в редакторе.
- **Маршрутизация вывода:** Префиксы в начале текста (`!` — отправить в OBS/WebView, `!!` — отправить только в TTS, без префикса — отправить везде) позволяют гибко управлять выходом фразы в Twitch и WebView.
- **Удобная отправка:** Клавиша `Enter` мгновенно отправляет текст на озвучку, а `Escape` скрывает окно редактора и возвращает фокус в предыдущее активное приложение. Режим `F6` позволяет отправлять сообщения по `Enter`, оставляя окно редактора открытым.

### AI-коррекция текста

Интеллектуальное исправление ваших фраз «на лету» перед озвучиванием.

**Возможности:**
- Поддержка провайдеров **OpenAI** (GPT-4o/GPT-3.5), **DeepSeek** (direct API) и **Z.ai**.
- **Кастомный системный промпт:** Настройка инструкций для ИИ (например, для исправления опечаток, пунктуации или изменения стиля фразы).
- **Тумблер в редакторе:** Быстрое включение/выключение AI-коррекции прямо в окне набора текста.

### Звуковая панель (Саунпад)

**Идея:** Мгновенное воспроизведение звуковых эффектов по клавише.

**Как это работает:**
1. Назначьте аудиофайл на букву (A-Z)
2. Нажмите `Ctrl+Shift+F2` — появится панель с клавиатурой
3. Нажмите любую букву — звук воспроизводится мгновенно

**Возможности:**
- **Наборы звуков (Sound Sets):** Возможность создания неограниченного числа независимых профилей (наборов) звуков под разные игры или темы трансляций с быстрым переключением между ними.
- **Клавиатурная сетка:** До 26 одновременных привязок на один набор (клавиши A-Z).
- **Поддерживаемые форматы:** MP3, WAV, OGG, FLAC.
- **Встроенные горячие клавиши:** Быстрое открытие панели и воспроизведение без кликов мыши.
- **Настройка в разделе «Звуковая панель»**

**Глобальные горячие клавиши:**
- `Ctrl+Shift+F2` — показать/скрыть панель
- `A-Z` — воспроизвести назначенный звук (работает globally!)

### WebView сервер (OBS интеграция)

**Идея:** Отображение текста в реальном времени в OBS Studio или браузере через Server-Sent Events (SSE).

**Как это работает:**
1. Включите WebView сервер в настройках (по умолчанию порт 10100)
2. Добавьте Browser Source в OBS: `http://localhost:10100`
3. Отправьте текст в TTS — текст появится в OBS мгновенно

**Возможности:**
- SSE (Server-Sent Events) для real-time обновлений
- Настраиваемый порт и bind address (0.0.0.0 / 127.0.0.1)
- Полная кастомизация HTML/CSS шаблонов (папка в config)
- Токен-аутентификация для внешних подключений
- UPnP — автоматический проброс портов на роутере
- Автозапуск при старте приложения

**Настройка в OBS:**
1. Источники → Добавить → Browser Source
2. URL: `http://localhost:10100`

**Внешнее подключение (для коллабораций):** вы можете открыть доступ к своему WebView-серверу из интернета (включив UPnP в настройках или настроив port forwarding на роутере вручную). Это позволяет передавать поток озвучиваемых сообщений на удаленные системы. Например, во время совместных стримов другой стример может добавить вашу ссылку (защищенную токеном авторизации) в качестве Browser Source в свой OBS, чтобы субтитры ваших реплик выводились прямо у него на трансляции в реальном времени.

### Панель управления воспроизведением

Плавающее окно управления воспроизведением (Playback Control) предоставляет контроль над озвучиваемым текстом в реальном времени.

**Возможности:**
- **Управление речью:** Кнопки паузы, полной остановки (Stop) и воспроизведения фразы сначала.

### Экспериментальный перехват клавиш (Numpad Control)

Для максимальной скорости и удобства во время динамичного геймплея предусмотрен экспериментальный режим, превращающий клавиши цифрового блока (Numpad) в панель прямого управления функциями приложения (воспроизведение, переключение режимов, вызовы).

### Как работает глобальный перехват (WH_KEYBOARD_LL)

TTSBard использует **Windows Low-Level Keyboard Hook** (`WH_KEYBOARD_LL`) для обработки глобальных горячих клавиш и экспериментального режима Numpad Control. Этот механизм Windows API позволяет приложению регистрировать нажатия клавиш без постоянного удержания фокуса на главном окне.

**Технические детали:**

| Аспект | Описание |
|--------|----------|
| **Тип хука** | `WH_KEYBOARD_LL` (Low-Level) |
| **Уровень** | User-mode (не требует драйвера ядра) |
| **Область действия** | Глобальная (все приложения системы) |
| **Функция API** | `SetWindowsHookExW` |

**Как это работает:**

1. **Регистрация хука** — приложение вызывает `SetWindowsHookExW` с типом `WH_KEYBOARD_LL`.
2. **Фильтрация событий** — в отличие от полной блокировки ввода, хук перехватывает и поглощает только назначенные функциональные клавиши (например, блок Numpad в режиме перехвата) для вызова действий приложения.
3. **Пропуск трафика** — все остальные клавиши передаются операционной системе без задержек и изменений через `CallNextHookEx`.
4. **Удаление хука** — при закрытии приложения хук автоматически удаляется.

**Преимущества подхода:**
- ✅ Не требует драйвера ядра (работает в user-mode).
- ✅ Точечно фильтрует только зарегистрированные клавиши управления, не влияя на обычный набор текста.
- ✅ Минимальное влияние на производительность системы.

**Ограничения:**
- ❌ Не перехватывает клавиши, когда фокус находится в приложениях с правами Администратора.
- ❌ Не может блокировать системные комбинации (например, `Ctrl+Alt+Del`).
- ❌ Доступно только для Windows 10/11.
