# TTSBard

TTS-приложение для общения с минимальным отвлечением от работы за компьютером.

---

## Проблема

Подробнее: [PROBLEMS.md](PROBLEMS.md)

Люди, использующие синтез речи (TTS) для общения, сталкиваются с тремя основными проблемами:

| Проблема | Последствия |
|----------|-------------|
| **Низкая скорость ввода** | Медленное общение, frustration |
| **Отсутствие интеграции** | Постоянное переключение между окнами, потеря контекста |

Существующие TTS-решения работают изолированно — нужно переключаться между мессенджером/браузером и приложением TTS, запускать воспроизведение. Это прерывает workflow и увеличивает время на общение.

## Решение

**TTSBard** работает незаметно в фоновом режиме и доступен мгновенно по горячей клавише в любом приложении.

Нажали `Ctrl+Shift+F1` — появляется плавающее окно для перехвата текста. Ввели текст — он озвучивается. Нажали `Escape` — окно исчезает, и вы возвращаетесь к работе.

**Где это работает:**
- Мессенджеры (Telegram, WhatsApp, Discord)
- Браузер
- Любое приложение на Windows
- Внутри самого TTSBard

## Возможности

### Глобальные горячие клавиши

| Комбинация | Действие |
|------------|----------|
| `Ctrl+Shift+F1` | Режим перехвата текста / Показать плавающее окно |
| `Ctrl+Shift+F2` | Показать звуковую панель |
| `Ctrl+Shift+F3` | Показать главное окно (поверх всех, для фокуса) |

> **Взаимное исключение:** Только одно плавающее окно может быть активно одновременно. Если открыто плавающее окно перехвата, звуковая панель не откроется, и наоборот.

### Режим перехвата текста

Режим перехвата — это основной способ ввода текста для озвучивания. При нажатии `Ctrl+Shift+F1` появляется плавающее окно, которое начинает **перехватывать все нажатия клавиш** с клавиатуры, независимо от того, в каком приложении вы работаете.

**Как это работает:**
1. Нажмите `Ctrl+Shift+F1` — появится плавающее окно в верхней части экрана
2. Начните печатать — все клавиши перехватываются и отображаются в окне
3. Нажмите `Enter` — текст отправляется в TTS для озвучивания
4. Окно можно перемещать мышкой в любое место экрана

**Особенности:**
- Перехват работает в любом приложении (браузер, мессенджеры, игры)
- Клавиатура блокируется для других приложений во время перехвата
- Автоматическая конвертация раскладки RU/EN (F8 для переключения)
- Пресеты позволяют быстро вводить часто используемые фразы
- Режим F6 позволяет отправлять несколько сообщений без закрытия окна

**Горячие клавиши в режиме перехвата:**

| Клавиша | Действие |
|---------|----------|
| `F8` | Переключить раскладку клавиатуры (EN ↔ RU) |
| `F6` | Переключить режим: Enter не закрывает окно / Enter закрывает окно |
| `Enter` | Отправить текст в TTS (закрыть окно, если режим F6 выключен) |
| `Escape` | Очистить текст, отключить перехват, скрыть окно |
| `Backspace` | Удалить последний символ |
| `Space` | Добавить пробел (с автозаменой пресетов) |

**Режим F6:** Когда включён, Enter отправляет текст в TTS, но окно остаётся открытым для продолжения ввода. Удобно для последовательных сообщений.

### Препроцессор текста

Пресеты позволяют автоматически заменять короткие коды на полный текст при вводе. Замена происходит мгновенно после нажатия пробела.

| Синтаксис | Описание | Пример |
|-----------|----------|--------|
| `\ключ` | Замена текста по коду | `\name` + пробел → Александр |
| `%юзернейм` | Замена имени пользователя | `%admin` + пробел → Администратор |

### Звуковая панель (Саунпад)

Позволяет привязать аудиофайлы к клавишам для мгновенного воспроизведения.

- До 26 привязок (по одной на каждую букву A-Z)
- Поддерживаемые форматы: MP3, WAV, OGG, FLAC
- Настройка в разделе «Звуковая панель»

### TTS провайдеры

**OpenAI TTS**
- 13 естественных голосов: alloy, ash, ballad, coral, echo, fable, nova, onyx, sage, shimmer, verse, marin, cedar
- Настройка скорости (0.25-4.0)
- Поддержка прокси
- Требуется API ключ

**Silero Bot**
- Интеграция с Telegram ботом
- Авторизация через телефон
- Работает без API ключа

### Аудио вывод

**Dual output** — одновременное воспроизведение на два устройства:
- **Динамики** — выбор устройства, громкость, вкл/выкл
- **Виртуальный микрофон** — VB-Cable, VoiceMeeter и др.

## Технологии

- **Tauri 2.0** — desktop фреймворк
- **Vue.js 3** — UI с Pinia для state management
- **Rust** — backend, low-level keyboard hook
- **Windows API** — WH_KEYBOARD_LL для перехвата клавиш
- **Rodio** — асинхронное MP3 воспроизведение

### Как работает перехват клавиш (WH_KEYBOARD_LL)

TTSBard использует **Windows Low-Level Keyboard Hook** (`WH_KEYBOARD_LL`) для глобального перехвата клавиш. Это механизм Windows API, который позволяет приложению перехватывать все нажатия клавиш в системе.

**Технические детали:**

| Аспект | Описание |
|--------|----------|
| **Тип хука** | `WH_KEYBOARD_LL` (Low-Level) |
| **Уровень** | User-mode (не требует драйвера ядра) |
| **Область действия** | Глобальная (все приложения системы) |
| **Функция API** | `SetWindowsHookExW` |

**Как это работает:**

1. **Регистрация хука** — приложение вызывает `SetWindowsHookExW` с типом `WH_KEYBOARD_LL`
2. **Перехват событий** — Windows вызывает процедуру хука при каждом нажатии/отпускании клавиши
3. **Обработка** — приложение может заблокировать клавишу (вернуть `1`) или передать дальше (`CallNextHookEx`)
4. **Удаление хука** — при закрытии приложения хук автоматически удаляется

**Преимущества подхода:**
- ✅ Не требует драйвера ядра
- ✅ Работает в user-mode
- ✅ Легко устанавливается и удаляется
- ✅ Минимальное влияние на производительность

**Ограничения:**
- ❌ Не перехватывает клавиши от приложений с правами Administrator
- ❌ Не может блокировать `Ctrl+Alt+Del` (Secure Desktop)
- ❌ Только Windows 10/11
